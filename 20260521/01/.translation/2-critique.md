# 第二步：找问题

对照直译稿，按风格定义（懂行的朋友在解释、表达清楚、不绕口、保留作者声音）找出问题。

## 术语生硬，直译腔重

- **"编码代理"（coding agents）/"代理"（agent）**：全文核心词，"代理"在中文里指代不清、生硬。应统一为"编码智能体""智能体"，或在语境清楚时用"它"。
- **"线程"（thread）**：在 Codex 语境里指一段对话/会话，直译"线程"会让人想到操作系统线程。但这是 Codex 的产品术语，保留"线程"可接受，需首次出现时让语境带出"对话"的意味。
- **"循环中"（in the loop）/"在循环里"**："仍在循环中""贴近它"——"in the loop"是"还参与其中、还在场"，直译"循环"会误读为程序循环。需意译为"还在场""还盯着""人还在场的时候"。
- **"中介完成"（mediated by code）**："由代码中介完成"非常翻译腔。改为"通过代码来完成""要靠代码"。
- **"想法倾倒"（thought dump）**：生硬。改为"把想法一股脑倒出来"。
- **"散文"（prose）**："打磨过的散文"——这里 prose 指"成形的文字/书面表达"，不是文学散文。改为"打磨好的文字"。
- **"产物"（artifact）**：可接受，但有些地方"呈现产物"读着别扭，可按语境换"成果""产出物"。
- **"幕僚长"（Chief of Staff）**：偏书面，但是常见译法，可保留；或用"大管家"更贴"懂行朋友"的语气。保留"幕僚长"并无大碍。
- **"心跳式循环唤醒"（heartbeat-style recurring wake-up）**："心跳式"对普通读者不友好。需补一句解释或改为"像心跳一样按节奏定时唤醒"。
- **"触及范围"（reach）**：标题"工具和触及范围"生硬。改为"工具与能触及的范围"或"工具与延伸"。

## 长句、难句，需要拆

- 开头第二段"随着这些界面对 Codex 可用，它开始感觉不那么像狭义上的编码助手，而更像一个完成计算机工作的系统。"——"对……可用""狭义上"翻译腔，要拆短、口语化。
- 语音输入首句"因为它在一个想法被压缩成打磨过的散文之前，捕捉到了它粗糙的版本"——被动+长定语，绕。需重写为主动短句。
- 目标节"有一条终点线、代理可以随时间持续努力达成的较长期运行的 Codex 任务"——定语堆叠，读不顺。
- 共享记忆首句"长期运行的线程在它们共享任何单个对话之外的记忆时会变得更有用"——"在……时"嵌套，绕口。

## 用词/标点

- 全文为面向中文读者的正文，标点应全角；代码块（vault 树、AGENTS.md）内保持原样。直译稿基本已用全角，但需复查 `$browser`、`@chrome` 等行内代码与正文衔接处。
- "diff""拉取请求"——pull request 已是熟词，可保留"拉取请求"或用"PR"，面向"会写点代码"的读者，"拉取请求（PR）"更自然。"diff"保留英文即可。
- "界面"（surface/interface）：surface 在原文里多指"可操作的表面/界面"，有的地方译"界面"OK，有的地方（web surfaces）指网页本身，需按语境调整。
- "那"开头的句子多（That changes... → "那改变了"），中文里连续用"那"做主语生硬，改为"这"或重组。

## 语气/感染力丢失

- 结尾"Ambition matters, but without verification it's just a wish." → "雄心很重要，但没有验证，它只是一个愿望。"基本到位，可再凝练为"光有雄心不够，没有验证，那就只是空想。"
- "Repositories hold code. The vault holds rolling context." → 原文是工整对仗的短句，直译"仓库装代码。库装滚动的上下文。"对仗在，但"滚动的上下文"生硬，改"不断累积的上下文"。
- 整体作者声音：原文短句多、节奏明快、爱用对照（now/next、code/context）。意译要保住这种"一句一个点"的节奏，避免合并成长句。
