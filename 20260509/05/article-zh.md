# 如何用 Claude 写出真正有你自己风格的 YouTube 脚本

**作者：** Bryan Ng ([@boomerrbryan](https://x.com/boomerrbryan))  
**日期：** 2026年5月8日  
**来源：** [How to Use Claude to Write YouTube Scripts That Actually Sound Like You](https://x.com/Zephyr_hg/status/2052718260097876413)

大多数人让 AI 写完脚本，立刻就觉得不对劲。听起来不像自己。太生硬。太通用。像个聊天机器人在硬撑人味儿。

问题不在 AI。问题在于你给它的东西太少了。

用 Claude 有两种具体的方法，能彻底改变输出质量。一种解决风格问题，一种解决编辑问题。两者结合，能把 Claude 变成一个真正让你的脚本变好的写作搭档——不只是写得更快。

## 第一步：先把你的风格喂给 Claude

在写第一个字之前，你得先教 Claude 你是怎么说话的。

方法出奇地简单。去你的 YouTube 频道，找两三个视频，把文字稿复制出来粘进 Claude。如果你还没有视频，用你写的简报、PDF，或者任何听起来像你自己的文字都行。

然后让 Claude 做一件事：把你的写作风格提炼成一份 JSON 档案。

JSON 档案就是一个结构化的数据文档。你让 Claude 分析你的写作，把它固定下来——你的语气、句子长度、标志性用语、讲故事的方式、建立可信度的手段。全部以可以随时粘回任何新对话的格式保存。

出来的东西长这样：你的性格标签，比如"像老师但不装"、"随意但有料"；你常用的过渡表达；你习惯的句式结构。这是你的写作指纹，存进了一个文档。

有了这份档案，下次打开 Claude 新对话，先把它粘进去，然后说"用这份 JSON 风格档案，给我写一篇关于[主题]的脚本"。输出结果会跟什么都不给就直接让 AI 写截然不同。Claude 基于一个关于 YouTube 增长的脚本风格档案，生成了这样一个开场钩子："你花好几周规划视频，拍了好几个小时，剪到眼睛出血，终于点了上传——47 次播放，三个赞，唯一的评论是你妈妈发的。"这种开场不是靠通用提示词出来的，是靠训练出来的。

这套方法对自己的频道和为客户写脚本都适用。把客户的写作风格喂给 Claude，提取风格档案，之后就能随时按他们的腔调出稿。

一个重要提示：不要试图一口气让 Claude 把整个脚本写出来。先让它给你几个开场钩子的变体，再写正文各个部分的变体，一段一段迭代，最后再拼在一起。

## 第二步：用 Claude 发现你自己看不到的问题

第二种用法是编辑——这个被严重低估了。

写完脚本之后，你离它太近了。薄弱的衔接、前后矛盾的信息、反复出现的内容、没落地的引导语——这些你很难自己发现。但 Claude 能发现。

关键是给 Claude 一套框架，而不是直接说"帮我改好"。先把你的编辑标准喂给它。可以是一个 PDF，一份幻灯片，也可以是你讲自己编辑思路的视频文字稿。先让 Claude 理解规则，再让它按规则来。

然后粘进你的草稿，给 Claude 一个角色定位——"你是一个播放量过百万的 YouTube 脚本作者兼资深文案"——然后让它每次只批评一个具体部分。不是整篇脚本，是一个点、一个段落。

反馈会很具体。举个真实的例子：一个关于"重新看待你的朝九晚五"的脚本点，被返回了这样的批评：过渡太弱、前后信息矛盾、例子用烂了、内容重复、结构不清晰。Claude 随后给出了改写版，删掉了废话，论点变得更锋利——这些问题，作者自己过了一遍完全没看出来。

改写不是完美的，但明显更好，而且就几秒钟的事。

说句实话：底层的文字还是得你来写。思考这件事没办法外包。你能做的是，用 Claude 给你的写作照一面镜子，让你看见自己看不到的地方。但怎么提问、喂什么进去、聚焦哪个部分——这个判断，还是你的事。

## 接下来做什么

从风格档案开始。把你现有视频的两三份文字稿（如果还没有视频就用你写得最好的文章）粘进 Claude，让它提炼一份 JSON 写作风格档案，保存好。

下次坐下来写脚本，先把这份档案粘进去，看看出来的结果有什么不同。

就这一个改变——在让 Claude 用你的风格写之前，先把你的风格给它——就能让 AI 辅助写出的脚本真正像你，而不是大多数人最后拿到的那种通用货。
