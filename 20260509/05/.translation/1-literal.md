# 如何使用 Claude 来编写真正听起来像你的 YouTube 脚本

**作者：** Bryan Ng ([@boomerrbryan](https://x.com/boomerrbryan))  
**日期：** 2026年5月8日  
**来源：** [How to Use Claude to Write YouTube Scripts That Actually Sound Like You](https://x.com/Zephyr_hg/status/2052718260097876413)

大多数人用 AI 写脚本后会立刻讨厌结果。听起来不像他们自己。很生硬。很通用。听起来像一个聊天机器人试图假装成人类。

问题不在 AI。问题在于你给它的信息不够多。

有两种特定的方式使用 Claude，可以完全改变输出质量。一种解决声音问题。另一种解决编辑问题。合在一起，它们把 Claude 变成一个真正让你的脚本变得更好的写作伙伴——不只是更快。

## 首先在 Claude 上训练你的品牌声音

在你写下第一个字之前，你需要教 Claude 你是怎么说话的。

这样做的方法出奇地简单。去你的 YouTube 频道，从你现有的两三个视频中提取文字稿，然后粘贴到 Claude 里。如果你还没有视频，可以用新闻简报、PDF，或者任何听起来像你的写作。

然后让 Claude 做一件特定的事：提取你品牌声音的 JSON 档案。

JSON 档案只是一个结构化的数据文档。你是在让 Claude 分析你的写作并捕捉它——你的语气、你的句子长度、你的标志性短语、你的讲故事风格、你的可信度建立方式。所有这些，都以一种你可以粘贴回任何未来对话的格式保存。

这是会出现的内容：像"教育导师"和"随意权威"这样的性格特征，你使用的特定过渡短语，你偏向的句子结构。这是你的写作指纹，被保存在一个文档中。

一旦你有了那个档案，把它粘贴到任何新的 Claude 对话中，然后说"用这个 JSON 档案给我写一篇关于[主题]的脚本"。结果与冷启动的 AI 草稿截然不同。Claude 从一个关于 YouTube 增长的脚本中生成了这个示例钩子："你花好几周规划你的视频，你拍摄几个小时，你编辑直到眼睛出血，你点击上传——47次观看，三个赞，而唯一的评论是来自你妈妈的。"那种钩子不是来自通用提示词。它来自训练。

这种方法无论是为自己的频道写作还是为客户写脚本都有效。在客户的品牌声音上训练 Claude，提取 JSON 档案，你就可以按需以他们的声音生产脚本。

一个重要提示：不要试图一次性完成脚本。先要求提供钩子变体。然后是正文部分变体。在你尝试把整件事组合在一起之前，让 Claude 对各个部分进行迭代。

## 使用 Claude 来捕捉你在编辑时遗漏的内容

第二个用例是编辑——而这个被低估了。

当你写好一个脚本后，你离它太近了。你会错过薄弱的过渡、相互矛盾的信息、重复性的内容，以及没有落地的行动号召。这些正是 Claude 可以发现的那种问题。

关键是给 Claude 一个编辑框架。不要只是粘贴你的脚本然后说"让这个变得更好"。先上传你的编辑指南。这可以是一个 PDF，一套幻灯片，甚至是你解释编辑原则的视频的文字稿。在让 Claude 应用这些规则之前，先喂给它这些规则。

然后粘贴你的草稿脚本，给 Claude 一个角色——"你是一个拥有数百万次观看的 YouTube 脚本作者和熟练的文案撰写人"——然后让它每次批评一个特定的部分。不是整个脚本。一个点，一个部分。

反馈是具体的。这里有一个真实的例子：一个关于重新框架你的朝九晚五工作的脚本点被返回，批评包括"薄弱的过渡"、"相互矛盾的信息"、"过度使用的例子"、"重复性内容"和"不清晰的结构"。Claude 然后生成了一个改写版本，删去了废话，磨锐了论点——这是作者在自己的审查中完全忽略的东西。

改写不完美。但实质上更好，而且只花了几秒钟。

对此的诚实评价是：基础文案仍然需要来自你。你无法完全外包思考。你能做的是使用 Claude 来举起一面镜子照向你的写作，让你看到你看不到的东西。提示本身——决定喂给 Claude 什么，如何构建请求，关注哪个部分——那是你仍然需要做的思考。

## 你的下一步

从品牌声音练习开始。从你现有的视频中提取两三个文字稿（或者如果你还没有视频，就用你最好的写作）并粘贴到 Claude 里。让它提取一个 JSON 品牌声音档案。保存那个文档。

然后下次你坐下来写脚本时，先粘贴那个档案，看看会出来什么。

那一个改变——在要求 Claude 用你的声音写作之前，先给 Claude 你的声音——就是将真正听起来像你的 AI 辅助脚本与大多数人最终得到的通用输出区分开来的东西。
