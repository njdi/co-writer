# 如何用 Claude 写出真正有你自己风格的 YouTube 脚本

**作者：** Bryan Ng ([@boomerrbryan](https://x.com/boomerrbryan))  
**日期：** 2026年5月8日  
**来源：** [How to Use Claude to Write YouTube Scripts That Actually Sound Like You](https://x.com/Zephyr_hg/status/2052718260097876413)

很多人第一次让 AI 帮自己写脚本，写完看一眼，觉得哪儿不对。

说不出来哪里，但就是不像自己。太规整，太稳，像一个人在很努力地假装自然。

问题不在 AI。问题在于，你没给它足够的东西去模仿。

用 Claude 写脚本，有两件事值得认真对待。一件是风格，一件是编辑。这两件事做对了，Claude 就不再只是"更快"，而是真的能让你的脚本变好。

## 先把你的写作风格存下来

你要做的第一件事，不是打开 Claude 然后说"帮我写个脚本"。

你要做的第一件事，是教 Claude 你是怎么说话的。

方法很直接。去你的 YouTube 频道，随便找两三个视频，把字幕或文字稿复制出来，粘进 Claude。如果你还没有视频，用你写过的简报、文章，任何带着你自己气质的文字都行。

然后让 Claude 做一件事：把你的写作风格总结成一份 JSON 档案。

JSON 档案听起来技术性很强，其实就是一个结构化的文字文件。你在让 Claude 分析你写东西的习惯——你的语气、你的句子有多长、你爱用哪些词、你怎么讲故事、你怎么让人信你。全部固定下来，存成一份可以反复用的格式。

出来的东西大概长这样：你的性格标签——"像老师，但不装"，"随意，但有料"；你常用的过渡句；你偏好的句式节奏。一份属于你的写作指纹。

有了这份档案，以后每次打开 Claude，先把它粘进去，然后说："用这份风格档案，给我写一篇关于[主题]的脚本。"

效果会很不一样。

举个例子：Claude 基于某人的写作风格档案，为一篇关于 YouTube 增长的脚本写了这样一个开场——

"你花好几周规划视频，拍了好几个小时，剪到眼睛出血，终于点了上传——47 次播放，三个赞，唯一的评论是你妈妈发的。"

这种开场，靠随便写个提示词是出不来的。

另外，如果你是帮客户写脚本，这套方法同样管用。把他们的文章、视频字幕喂给 Claude，提炼出风格档案，之后就能随时用他们的腔调出稿。

还有一点：不要让 Claude 一口气把整个脚本写完。先要几个开场变体，挑一个满意的；再写正文各部分；最后才拼在一起。分段迭代，比一次性写完效果好得多。

## 写完之后，让 Claude 帮你找你自己看不到的问题

第二件事是编辑，而且这件事经常被低估。

写完一篇东西，你太熟悉它了。里面哪里衔接奇怪、哪里前后打架、哪里同样的话说了两遍、哪里想引导读者但没落地——这些，你自己往往看不出来。

Claude 能看出来。

但有个前提：你不能直接把脚本扔给它说"帮我改好"。你得先给它一套标准。

把你的编辑原则整理出来，喂给 Claude——可以是一个 PDF，一份幻灯片，甚至是你某次讲自己怎么写脚本的视频文字稿。先让它理解规则，再让它用规则来审你的稿子。

然后给 Claude 一个具体的角色："你是一个视频累计播放量过百万的 YouTube 脚本作者，也是资深文案。"接着粘进你的草稿，让它每次只看一个点、一个段落，给出批评。

反馈会是具体的。

一个真实的例子：有人把一段关于"重新看待朝九晚五"的脚本发给 Claude，返回的批评是：过渡太弱、前后逻辑矛盾、这个例子大家见腻了、内容重复、结构说不清楚。然后 Claude 给出了改写版——废话少了，论点更紧了。而这些问题，这位作者自己过了一遍，一个都没发现。

改写不是完美的。但比原来好，而且用了没几秒钟。

说句实话：脚本最底层的东西还是得你来想。思考，没办法外包。Claude 能做的是给你的写作照一面镜子，让你看见自己看不到的地方。但该问什么问题、该把哪一段交给它看——这个判断，还是你的。

## 今天就可以开始做的事

找你已有的两三份视频文字稿（没有视频就用你写得最好的文章），粘进 Claude，让它提炼出一份 JSON 写作风格档案，存好。

下次坐下来写脚本，先把这份档案粘进 Claude，再开始写。

就这一步的区别——在让 Claude 用你的风格写之前，先把你的风格给它——写出来的东西会不一样。
