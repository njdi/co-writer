# 如何用 Claude 写出真正有你自己风格的 YouTube 脚本

**作者：** Bryan Ng ([@boomerrbryan](https://x.com/boomerrbryan))  
**日期：** 2026年5月8日  
**来源：** [How to Use Claude to Write YouTube Scripts That Actually Sound Like You](https://x.com/Zephyr_hg/status/2052718260097876413)

接触过不少做自媒体的人。

有一件事几乎是共同的困境：让 AI 帮写脚本，写出来，看两眼，觉得不对。不是质量差，是不像自己。读起来像一份通知，而不是一个人在说话。

我理解这种感觉。

问题不在 AI。不在工具。在于你给它的东西太少了。

---

说具体一点。

用 Claude 写脚本，真正重要的事只有两件。

一件是风格。一件是编辑。

这两件事做对了，Claude 才是搭档，而不只是个加速工具。

---

**先说风格。**

很多人一上来就说："帮我写个关于 XX 的脚本。"

这是最大的问题。

你什么都没给，凭什么它能写出你的感觉？

正确的顺序是反过来的。先把你的写作风格喂给它，再让它写。

怎么喂？

去你的频道，随便找两三个视频，把文字稿复制出来，粘进 Claude。如果你还没有视频，用你写的文章、简报，任何听起来像你自己的文字都行。

然后让 Claude 做一件事：把你的写作风格提炼成一份 JSON 档案。

所谓 JSON 档案，就是一个结构化文件。你的语气是什么、句子长不长、你惯用什么过渡词、你怎么讲故事、你怎么让人信服。全部固定下来，存成可以反复粘贴的格式。

存好之后，以后每次打开 Claude 新对话，先把这份档案粘进去，再说你要写什么。

效果是不一样的。

举个实际的例子。Claude 基于某人的风格档案，生成了这样一个开场钩子：

"你花好几周规划视频，拍了好几个小时，剪到眼睛出血，终于点了上传——47 次播放，三个赞，唯一的评论是你妈妈发的。"

这不是通用提示词写出来的。是训练出来的。

这是本质区别。

---

好吧，还有一个细节。

不要让 Claude 一口气把整个脚本写完。

先要开场变体，挑一个满意的。再写正文各部分的变体。最后才拼。

分段迭代，远比一次性交付出来的东西好。

---

**再说编辑。**

写完一篇东西，你离它太近了。

哪里衔接奇怪，哪里前后打架，哪里同样的话说了两遍——这些，你自己往往真的看不出来。

Claude 能看出来。

但有个前提：你不能直接把稿子扔给它说"帮我改好"。

你得先给它标准。

把你的编辑原则整理出来，可以是一个 PDF、一份幻灯片，或者你哪次讲自己怎么写脚本的视频文字稿。先让 Claude 理解规则，再让它用这个规则来审你的稿子。

然后给它一个角色定位：你是一个播放量过百万的 YouTube 脚本作者，也是资深文案。接着粘进你的草稿，让它每次只看一个点、一个段落。

不是整篇。是一个段落。

反馈会很具体。

一个真实的案例：有人把一段关于"重新看待朝九晚五"的脚本交给 Claude，返回的批评是：过渡太弱、前后信息矛盾、这个例子用滥了、内容重复、结构不清楚。然后 Claude 给出改写版，废话少了，论点更紧了。这些问题，作者自己过了一遍，一个都没发现。

改写不完美。但明显更好。就几秒钟的事。

---

说实话，底层的判断还是得你来做。

Claude 能做的是给你的写作照一面镜子。

但问什么问题，喂什么进去，聚焦哪个段落——

这个，没办法外包。

---

**总结就这一句话：**

在让 Claude 用你的风格写之前，先把你的风格给它。

其他的，跟着做就行。
