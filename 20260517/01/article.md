# Claude Code 的远程控制和消息推送是咋回事？

你有没有遇到过这种情况：在公司用 Claude Code 跑了一个比较耗时的任务，结果要出门了，只能盯着终端干等，或者直接放弃走人。

现在不用这样了。

Claude Code 有两个功能可以解决这个问题：**远程控制（Remote Control）** 和 **消息推送（Push Notifications）**。

一个让你离开电脑后还能继续操控会话，另一个在任务完成时主动通知你。

---

## 远程控制：离开电脑，但不离开 Claude

### 它到底是个什么东西

说白了，远程控制就是用手机或浏览器来控制你自己机器上的 Claude Code。

Claude 还在你本地跑，你的文件、MCP 服务器、项目配置全都在，手机或浏览器只是一个窗口，发出去的指令会实时同步回你的机器执行。

这和 Claude Code on the Web 不一样——后者是真的跑在 Anthropic 的云上，没有你的本地环境。

### 怎么开启

有两种方式。

**第一种：直接从项目目录启动一个新的远程会话**

在项目目录下跑这条命令：

```bash
claude remote-control
```

命令跑起来之后，终端会一直挂着等待远程连接，同时显示一个 session URL；按空格键还能弹出二维码，用手机扫一下直接打开。

想给这个 session 起个名字方便找？加个参数：

```bash
claude remote-control --name "我的项目"
```

**第二种：在已有会话中途开启**

如果你已经在跑一个 Claude Code 会话，不用退出重开，直接在会话里执行斜杠命令：

```
/remote-control
```

它会把当前这个会话立刻变成可远程连接的状态，你之前的对话上下文全都还在。

同样可以指定名称，在命令后面带上就行：

```
/remote-control 我的项目
```

### 怎么从其他设备连上去

有三种方式，任选一种：

**方式一：打开 Session URL**

终端启动后会显示一个链接，在任意浏览器里打开就能进入远程会话。

**方式二：扫二维码**

在终端按空格键会显示一个二维码，它和 Session URL 是同一个东西。用手机扫一下，跳转打开对应的链接，进入会话。

**方式三：在 Claude 手机 App 的 Code 界面里直接找（最推荐）**

打开 Claude 手机 App，点导航栏的 **Code** 标签，这里会列出你所有的远程会话，直接按名字找就行。

---

连上之后，你在手机上发的消息、在终端发的消息，全都同步在同一个会话里，可以混着用。

### 让每次启动都自动开启远程控制

如果你希望每次开 Claude Code 都自动开启远程控制，不用每次手动跑命令，可以在 Claude Code 里执行 `/config`，找到 **Enable Remote Control for all sessions**，设置为 `true`。

---

## 消息推送：任务跑完了，手机主动告诉你

### 它是用来干嘛的

有了远程控制，Claude 在跑任务，你可以去干别的事情。但你怎么知道任务跑完了？总不能一直盯着手机屏幕吧。

消息推送就是为了解决这个问题。当 Claude 完成一个耗时任务，或者遇到需要你决策的情况，它会主动给你的手机发一条推送通知。

你也可以直接在 prompt 里告诉它：

```
跑完测试之后通知我
```

Claude 会在任务结束后主动推一条通知过来。

### 怎么配置

四步搞定：

1. **安装 Claude 手机 App**（iOS 或 Android 都有）
2. **用和 Claude Code 终端一样的账号登录**
3. **允许通知权限**（系统弹窗，点允许）
4. **在终端里执行 `/config`**，找到 **Push when Claude decides**，开启它

配置好之后，远程控制会话活跃期间，推送就自动生效了。

---

## 小结

远程控制和消息推送组合起来，用法很自然：在家或公司启动一个 Claude Code 任务，出门之前跑一句 `claude remote-control`，然后告诉 Claude"跑完通知我"，就可以放心去干别的了。

任务完成，手机收到推送，掏出来看结果，或者继续给指令——全程不需要守着电脑。

目前这两个功能都在 **research preview** 阶段，需要 Claude Code v2.1.51 及以上版本，用 `claude --version` 确认一下你的版本。
