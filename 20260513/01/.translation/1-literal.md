 # Airtap 如何工作。

 三层结构。AI 在云端，双手触达每台设备，可接入任何能读取 SKILLS.md 的 agent。

 [立即试用](https://airtap.ai/app) [查看实际效果 →](https://airtap.ai/see-it-in-action.html)

 架构
 ## 大脑 · 双手 · 设备

 大脑负责决策。AutoPilot 负责执行。操作落地在真实或云端手机上。

 三层结构
 ## 每一层的作用

 1 · 大脑 Airtap AI 云
 记忆、例程与决策。跨会话持久存在，随时可达。或者——接入你自己的 agent。

 2 · 双手 AutoPilot
 将意图转化为实际的点击操作。驱动真实应用——点击、滚动、输入、导航，完全像人一样操作。

 3 · 设备 云手机或你的手机
 AutoPilot 运行在专属云手机（始终开启）或你的实体设备上——你的应用，你的账户。或者两者兼用。

 两条设备路径
 ## 云手机还是你自己的手机

 云端 云手机
 每位 Airtap 用户都有一台专属云端手机。应用始终保持登录状态。任务全天候运行——不占用你的设备，不依赖你的电池或网络。最适合监控、定时任务和夜间自动化。

 实体 你的手机 + AutoPilot
 在 Android、iOS、TV 或 PC 上安装 AutoPilot。Airtap 随后可以驱动你已经在用的应用，在你已经携带的设备上运行。最适合与特定账户、位置或硬件绑定的操作。

 两者皆可。或者两者都用。同一个大脑，同一套例程。

 Agent 集成
 ## 接入 Claude、Codex 或 OpenClaw

 一个 SKILLS.md 文件教会你的 agent 如何操作手机。你的 agent 不再只是回答，而是开始行动。

 1 获取技能文件
 下载 Airtap 的 SKILLS.md——一个定义 agent 如何操作手机的单一文件。

 2 加载到你的 agent
 将文件放入 Claude、Codex、OpenClaw 或任何兼容 SKILLS.md 的运行时。

 3 分配一台手机
 启动云手机或连接实体设备。你的 agent 已准备好行动。

 可实现的功能
 ## 一旦 agent 拥有手机

 消息 发送与回复
 跨 WhatsApp、Telegram、iMessage、Slack——agent 读取，agent 回复。

 社交 发布与监控
 TikTok、Instagram、X——发布内容，追踪互动，回复评论。

 预订 预约与购买
 OpenTable、ClassPass、Amazon——在名额刚开放的那一刻抢到餐桌、课程、优惠。

 监控 追踪动态
 追踪价格下降、领取状态、豁免清除——并在触发条件命中时立即行动。

 工作流 多步骤任务
 端到端串联应用——收集、决策、执行、确认——不丢失上下文。

 多 agent 手机网络
 每个 agent 有自己的手机、身份和持久应用会话。在多个 agent 之间协调。
