# 直译问题批注

## 1. "例程"（routines）
- 问题：生硬，中文里"例程"是偏程序员的说法，普通读者感觉陌生
- 建议：改为"任务"或"常规任务"更自然

## 2. "跨会话持久存在，随时可达"
- 问题：逐字翻译"Persistent across sessions, always reachable"，读起来像机器翻译
- 建议：改为"记忆不会消失，随时都能接上"或"关了重开，它还记得；任何时候，它都在线"

## 3. "或者——接入你自己的 agent"
- 问题：破折号用法来自英文原文结构，中文里显得突兀
- 建议：改为"当然，你也可以接入自己的 agent"

## 4. "两者皆可。或者两者都用。同一个大脑，同一套例程。"
- 问题："两者皆可"偏文言，"同一套例程"生硬
- 建议："两种都行，也可以同时用。背后是同一个大脑，跑的是同一套任务。"

## 5. "豁免清除"（waiver clears）
- 问题：这是非常生硬的直译，中文读者完全不知道什么意思
- 建议：改为"等待资格放开""排队资格到了"，或干脆概括为"某个门槛解除"

## 6. "端到端串联应用"
- 问题：技术感太重，读者需要停下来想一下是什么意思
- 建议：改为"把多个应用串起来"

## 7. "在多个 agent 之间协调"
- 问题：缺乏主语，语气偏平
- 建议：改为"多个 agent 各司其职，互相协作"

## 8. "大脑在云端。双手在每台手机上。"
- 问题：直译保留了原文的节奏感，但"双手"这个比喻在最后单独出现有点突兀
- 建议：保留原文节奏，轻微调整："大脑在云端，手在每台手机上。"

## 9. 整体语气
- 原文的语气简洁、有力、带点自信，像产品文案
- 直译整体偏平，少了一些"说服力"
- 意译时要在保持信息准确的前提下，让每一段读起来更有节奏感
