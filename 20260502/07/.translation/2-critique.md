# 直译稿问题梳理

## 一、语气不够自然，翻译腔残留

**问题1：** "每天浪费 47 分钟在向一个本应了解我的工具重复解释自己"
→ 句子太长太绕，中文读起来费劲。
→ 改成：每天有 47 分钟在干同一件事——向一个本该已经了解我的工具，重复解释我自己。

**问题2：** "这才是真正变得严肃的地方"
→ 原文 "this is where it gets serious" 翻成这样很奇怪，没有中文味道。
→ 改成：从这里开始，事情才算真正上台阶了。

**问题3：** "这个很厉害"（对应原文 "this one is wild"）
→ "wild" 有那种让人眼前一亮的意思，"很厉害"太平。
→ 改成：这个有点意思。或：这一个真的有点东西。

**问题4：** "我用三个工具协同工作来解决了这个问题"
→ 翻译腔，中文更自然的说法是：我用三个工具配合着解决了这个问题。或：靠三个工具配合，我把这个问题解决掉了。

**问题5：** "以下才是真正有效的做法："
→ "以下才是" 有点书面，原文 "here's what actually works" 口气更随意。
→ 改成：这样写才有用：

**问题6：** "这是你一次性永久告诉 claude 你是谁、你在构建什么，以及你希望事情如何完成的方式"
→ 太长，结构绕。
→ 改成：它就是你告诉 claude "我是谁、在做什么、希望怎么做事"的地方——一次写好，永久生效。

## 二、语气感染力流失

**问题7：** "整套系统的运行成本是每月 $20 的订阅费"
→ 原文 "the whole thing runs on a $20/mo subscription" 语气是种惊喜感。
→ 改成：整套系统，只要每月 20 美元的订阅费就够了。

**问题8：** "但更重要的是，agents 现在在凌晨 3 点做我以前在下午 3 点做的工作"
→ 这句话有种轻描淡写的冲击力，译文基本准确，但可以更有节奏感。
→ 改成：更重要的是，那些我以前下午三点才能做的事，agents 现在在凌晨三点就做完了。

**问题9：** "你不需要今天就构建全部 6 层"
→ 原文是轻松的收尾，可以更口语。
→ 改成：不用今天就把 6 层全搭完。

## 三、结构问题

**问题10：** "以下是我一步步构建它的详细过程"
→ 原文 "here's exactly how i built it, step by step" 比较简短有力。
→ 改成：下面一步步拆给你看。

**问题11：** "ROI 完全不在一个量级上"
→ 有点生硬，原文 "the ROI is not even close" 是说效果差距悬殊。
→ 改成：回报差距根本不是一个级别的。

## 四、准确度微调

**问题12：** "claude code 是一个通才。它什么都能做，但没有什么做得特别出色"
→ 第二句 "nothing exceptionally well" 意思是没有什么领域特别擅长。译文基本准确，但可以更流畅。
→ 改成：claude code 开箱即用是个通才——什么都能做，但没有哪件事做得特别出色。

**问题13：** "头脑风暴 → 规格 → 计划 → TDD → 实现 → 审查"
→ "规格"不够口语，中文常说"需求文档"或直接留 spec。
→ 改成：头脑风暴 → 需求确认 → 计划 → TDD → 实现 → 审查
