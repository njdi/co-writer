# 直译稿问题分析

## 风格问题

### 1. 标题过于字面
- "如何构建小东西" → 中文里"构建"偏技术书面，"做"或"搭"更口语自然
- 同样"构建有观点的产品"里"构建"显生硬

### 2. 英文句式残留
- "构建某样东西也是最好的学习方式" → 典型的英文语序直译，中文应调整为"学东西，最好的办法就是去做"
- "如果你对所有人都是一切，你最终对任何人都什么都不是" → 绕口，中文不这样说
- "这很有趣" 结尾太平淡，原文"It's a lot of fun"语气更活泼

### 3. 语气丢失
- 原文有很强的直接、对话感，一些地方翻译后变得过于书面
- "抱歉这听起来很刺耳" → "这话可能不太好听" 更自然
- "反直觉的是" → 虽然正确，但略显翻译腔

### 4. 部分专有词汇处理
- "skill" 在原文中既是 Claude 的 skill 概念，也有通用技能的意思，直译为"技能"在部分上下文语义偏差
- "vibe coding" 保留原文更好，加括号简单解释即可
- "令牌" → "token" 更直接，目标读者懂互联网，不需要强行翻译

### 5. 重要语气句
- "Bring the problem, not the solution." → "带来问题，而不是解决方案" 失去了口号感，应更有力
- "Go tell a great story." → "去讲一个好故事" 可以，但结尾力度稍弱

### 6. 列表项目描述冗长
- 三个项目的描述翻译偏长，可以更简洁

## 需要改善的地方小结

1. "构建" 换成更口语的"做"或"搭"（视上下文）
2. 修复英文句式残留，用自然中文重组
3. 还原原文的直接、轻松对话语气
4. "skill" 根据上下文处理；"vibe coding" 保留英文
5. "token" 不翻译
6. 口号式短句要有力度
