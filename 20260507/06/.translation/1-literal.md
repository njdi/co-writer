# 如何构建一个每天都在变得更聪明、而你什么都不用做的 Obsidian 知识库

**作者：** CyrilXBT ([@cyrilXBT](https://x.com/cyrilXBT))  
**日期：** 2026年5月7日  
**来源：** [How to Build an Obsidian Knowledge Vault That Gets Smarter Every Day Without You Doing Anything](https://x.com/Zephyr_hg/status/2052235121416188114)

你读的每一篇文章。你保存的每一条推文。你录制的每一段语音备忘录。这一切都会自动流入系统。Claude 将这些点连接起来。你收获洞见。

大多数人把他们的 Obsidian 知识库当作档案柜来对待。他们把东西放进去。他们从不取出任何东西。六个月后，他们拥有一个整理得很漂亮的信息档案库，里面的东西他们完全忘记了，也从未付诸行动。

这份指南构建的是完全不同的东西。不是你向其添加内容的知识库，而是向你添加内容的知识库。一个你消费的每条信息都自动流入的系统，Claude 浮现出你错过的关联，而洞见每天都在复利增长，你无需做任何额外的工作。

第二大脑和死档案库之间的区别只有一件事：反馈。只进不出的信息不是知识系统。它是一个有着好文件夹的墓地。这份指南构建的反馈循环，会将你的知识库从存储系统转变为思维伙伴。

## 为什么大多数知识系统会失败

第二大脑的承诺是你永远不会再失去一个好主意。对大多数人来说，现实是他们花三个小时建立一个漂亮的文件夹结构，添加两周内容，然后完全停止打开这个应用，因为从来没有任何有用的东西回流出来。

失败模式总是一样的。系统是为输入而设计的。没有人为输出而设计。你捕获一切。你什么也检索不出来。知识库在增长。你的思维没有。

这种情况发生有三个具体原因，而且每一个都是可以修复的。

**第一：捕获摩擦。** 如果向你的知识库添加某样东西需要超过10秒的手动操作，在任何真正的认知负荷下你都会停止这样做。大多数人的捕获流程包括复制、粘贴、打标签、分类和总结——所有这些都在他们甚至还没处理刚读到的内容之前。当摩擦足够高时，习惯就会崩溃。

**第二：没有关联层。** 大多数知识库都是孤立笔记的集合。每篇文章都存在于自己的文件中。每个想法都坐落在自己的文件夹里。没有任何机制能纵观一切并说：你三月份保存的这件事与你今天正在处理的问题直接相关。没有这一层，知识库就是一个没有搜索功能的图书馆。

**第三：没有返回的理由。** 如果你的知识库不主动把洞见推送给你，你就必须记得主动去获取。没有人会记得。知识库变成了你偶尔添加内容、只有在主动搜索某些特定内容时才会打开的东西。那不是思维伙伴，那是书签工具。

一个从不说话的第二大脑不是第二大脑。它是一种非常有条理的忘记事物的方式。

这份指南构建的系统解决了所有这三个问题。捕获是自动的——你永远不需要手动添加任何东西。关联是 Claude 的工作——它读取所有内容并浮现出重要的东西。返回已内置于每日仪式中——知识库每天早上在你询问之前就向你简报。

## 架构：四个协同工作的层次

在触碰任何工具之前，先理解四层结构。这个系统中的每一款软件都只服务于一个功能。没有重叠。一切沿着一个方向流动。

**第一层是捕获。** 这是每一个无需你手动输入任何内容就能将信息引入系统的工具。Readwise 用于文章和高亮。Airr 用于播客片段。Whisper 用于语音备忘录。一个专用的 Telegram 机器人用于从你的手机快速保存。这一层中没有任何东西需要你分类、打标签或总结。原始信息进入。仅此而已。

**第二层是管道。** N8N 自动化监视每个捕获来源，并将新内容路由到你的 Obsidian 知识库中的正确位置。无需手动归档。无需复制粘贴。一个新的文章高亮出现在 Readwise 中，几分钟内它就以格式化的 markdown 文件形式出现在你的知识库中，来源、日期和内容都已自动结构化。

**第三层是 Obsidian。** 知识库本身。你本地机器上的一个 markdown 文件文件夹。这是永久存储层。一切都在这里。没有任何东西被删除。知识库是你曾经消费并认为值得保存的一切事物的真实来源。

**第四层是 Claude。** 这是贯穿其他所有层次运行的智能层。Claude 读取知识库。找到关联。浮现模式。撰写每日简报。回答关于你自己思维的问题。这是让知识库成为思维伙伴而非档案库的层次。

## 第一步：无摩擦的自动捕获

捕获层有一个任务：在不对你提出任何要求的情况下收集一切。捕获中的每个摩擦点都是你知识库中未来的空缺。设置一次。永远不要再碰它。

**文章和高亮：** Readwise 是书面内容捕获层的支柱。安装浏览器扩展。你读的每篇文章，高亮重要的句子。Readwise 自动存储它们。你不用做其他任何事情。不总结。不打标签。高亮然后继续。

Readwise 还连接到 Kindle、Twitter 书签、Instapaper 和 Pocket。你在这些平台上保存的任何内容都会自动流入 Readwise。一个工具聚合了一切。

**播客和音频：** Airr 让你可以通过摇晃手机来剪辑播客片段。片段的文字记录会自动保存。对于更长的内容——会议、讲座、语音备忘录——录制下来并通过 Whisper 处理。粘贴音频文件，几秒钟内就能得到干净的文字记录。

**从任何地方快速捕获：** 构建一个 Telegram 机器人，接受你发送给它的任何消息并将其路由到你的知识库。在车里突然闪现的想法。你想思考的推文。谈话中出现的问题。发送给机器人。它会自动落入你知识库的收件箱文件夹。用 Claude Code 和 N8N 构建需要30分钟。

**N8N 工作流 — Telegram 到 Obsidian：**

```
节点 1：Telegram 触发器 → 事件：消息 → chat_id：your_bot_id
节点 2：代码（格式化笔记）→ 文件名：inbox/{{date}}-quick-capture.md → 内容：# 快速捕获 / {{message}} / 来源：Telegram / 日期：{{date}}
节点 3：写文件到 Obsidian 知识库 → 路径：/your-vault/inbox/ → 操作：创建
```

## 第二步：可扩展的知识库结构

你的知识库文件夹结构决定了 Claude 导航它的能力。不要过度设计。五个文件夹。这就是整个结构。

**收件箱** — 所有东西首先落在这里。未处理。原始状态。所有自动捕获在归档之前的暂存区。

**笔记** — 处理过的高亮、文章、播客片段。每个来源一个文件。由 N8N 管道自动格式化。

**想法** — 你自己的思考。快速捕获。观察。文字转录的语音备忘录。你大脑的输出，而非他人的输入。

**项目** — 活跃的工作。每个项目一个文件夹。当你请求项目特定的上下文或决策时，Claude 读取这些内容。

**CLAUDE.md** — 指令层。Claude 在每个会话开始时首先读取的文件。告诉它你是谁、你在做什么，以及你想从知识库中得到什么。

这种简洁性是有意为之的。每一个复杂的文件夹结构最终都会在自身重量下崩溃，因为你不知道某样东西属于哪个文件夹，捕获摩擦上升直到系统崩溃。五个文件夹。一条规则：有疑问时，放入收件箱。

## 第三步：让一切运转起来的 CLAUDE.md 文件

这是整个系统中最重要的文件。没有它，Claude 每次会话都是冷启动——对你是谁、你在做什么，以及你想从知识库中得到什么一无所知。有了它，Claude 就是一个已经读了你几个月笔记的协作者。

CLAUDE.md 是你知识库根目录下的一个 markdown 文件。Claude 在每次会话开始时自动读取它。直接复制这个模板：

```markdown
# 我是谁
姓名：[你的名字]
工作：[你做什么——要具体]
专注：[你现在正在努力提升的一件事]
2026年目标：[你正在努力实现的3个具体成果]

# 当前项目
活跃：[你现在正在构建或处理的事情]
卡住了：[你最需要思维帮助的地方]
下一个里程碑：[当前冲刺完成后是什么样子]

# 这个知识库如何工作
收件箱：/inbox — 未处理的捕获，先在这里归档
笔记：/notes — 处理过的文章、高亮、研究
想法：/ideas — 我自己的思考和观察
项目：/projects — 活跃工作文件夹

# 我想从你那里得到什么
- 浮现我没有看到的关联
- 在同意我之前挑战我的假设
- 当我问该专注于什么时——从知识库上下文回答，而不是泛泛而谈
- 当我相信的某件事与我之前保存的某件事矛盾时提醒我

# 我正在阅读和思考的内容
[每周更新——当前的痴迷、活跃的问题、让你困惑的事情]
```

每周一早上更新"当前项目"和"我正在阅读"部分。五分钟。这一个习惯是保持 Claude 上下文随着你的工作演进而准确的关键。过时的 CLAUDE.md 会产生过时的答案。

## 第四步：自动运行的每日简报

每天早上在你打开任何一个应用之前，知识库就向你简报。昨夜发现的新关联。本周捕获内容中的模式。基于你一直在阅读的所有内容，今天值得思考的一个问题。

你不需要请求这份简报。它通过 N8N 按计划自动运行。当你坐下来工作时，它已经在你的收件箱文件夹里等着你了。

直接将这个提示词复制到你的 N8N Claude 节点中：

```
"你正在阅读我的 Obsidian 知识库。读取过去24小时内 /inbox 中的所有内容以及过去7天内 /notes 中的所有内容。

然后做三件事：

1. 关联 — 找到最近捕获的内容与我可能没有注意到的旧笔记之间最有趣的3个关联。要具体。引用相关段落。
2. 模式 — 识别我本周一直在阅读的内容中的一个模式。我的大脑显然在处理什么，即使我没有明确说出来？
3. 问题 — 根据你识别的模式，给我一个今天值得思考的问题。不是任务，是问题。

将其写成为 Obsidian 格式化的干净 markdown 文件。保存到 /inbox/brief-{{date}}.md"
```

将其设置为每个工作日早上6点运行。在打开其他任何东西之前阅读它。

## 第五步：每周综合

每周一次，进行更深层次的综合。十五分钟。与 Claude 坐下来谈论知识库一直在向什么方向构建。

复制这个提示词：

```
"阅读我整个 Obsidian 知识库。专注于过去7天添加的所有内容。

我想要四件事：

1. 新兴论点 — 我在没有明确说出的情况下正在向什么想法构建？我的思维中正在形成什么立场？
2. 矛盾 — 我最近保存了什么与我之前相信的东西相矛盾？从我自己的笔记中向我展示两面。
3. 知识空缺 — 根据我正在阅读和思考的内容，我显然没有阅读但应该阅读的内容是什么？缺少什么视角？
4. 一个行动 — 鉴于这个知识库中的所有内容，我本周可以做或思考的最高杠杆的一件事是什么？

直接说。挑战我。不要总结我已经知道的东西。"
```

综合会话是真正的复利发生的地方。每日简报浮现关联。每周综合构建论点。经过六个月的每周会话，你将拥有一份你的思维如何演进的记录——你持有并改变的每一个假设，每一个从小事开始并成长为你付诸行动的东西的想法。

## 没有人谈论的复利效应

一个月时，知识库感觉像是一个有用的工具。你保存得更多，失去的想法更少，每日简报偶尔会浮现出有趣的东西。

三个月时，感觉开始不同了。Claude 开始将第一个月的事情与第三个月的事情联系起来。你问它一个关于当前问题的问题，它找到了八周前你完全忘记的相关笔记。知识库知道关于你思维的事情，而你自己并不有意识地记得。

六个月时，它完全是另一回事了。你有了每一个你持有并改变的信念的记录。每一个你曾经坐下来思考的问题以及最终浮现的答案。每一个在你有意识地将其认定为痴迷之前就出现在你阅读中的模式。

你六个月后拥有的 AI 与你开始时的那个不同。它在你忙于生活时一直在阅读你的思维。

这是你自己思维的复利。大多数知识从不复利，因为它孤立地坐着。这个系统自动建立这些关联。你消费的每一个想法都加入了一个不断增长的想法网络，Claude 可以代表你导航这个网络。

你在你之后六个月开始这个系统的竞争对手不只是在设置上落后。他们落后了六个月的关联、模式和综合，这些使系统真正对你特定的思维方式具有智慧。那个差距不能通过更努力工作来弥合。它只能通过更早开始来弥合。

## 完整设置序列

**01** — 安装 Obsidian 并创建你的五文件夹结构：收件箱、笔记、想法、项目，以及根目录中的 CLAUDE.md。不要添加任何其他文件夹。从简单开始，让结构从实际使用中演进。

**02** — 将 Readwise 连接到你的知识库 Readwise 有原生的 Obsidian 集成。启用它。你在任何地方做的每一个高亮都会自动以格式化的 markdown 文件形式出现在你的笔记文件夹中。

**03** — 用 N8N 构建 Telegram 捕获机器人 使用上面的工作流。需要30分钟。一旦运行，它将处理你余生中来自手机的每一次快速捕获。

**04** — 写你的 CLAUDE.md 文件 使用上面的模板。要具体和诚实。Claude 输出的质量与你在这个文件中给它的上下文质量直接成比例。

**05** — 在 N8N 中设置每日简报自动化 将简报提示词安排在每个工作日早上6点运行。输出进入你的收件箱文件夹。在打开其他任何东西之前阅读它。

**06** — 每周一阻断15分钟用于每周综合 现在就把它放入你的日历。这是复利实际发生的会话。不要因为知识库感觉太空而在第二周跳过它。知识库永远不会太空而找不到值得思考的东西。

## 从五条笔记开始

人们从不构建这个系统的最常见原因是感觉一次要设置太多。

从更小的地方开始。今天，在 Obsidian 中放五条笔记。任何东西——五篇你一直想读的文章，五个一直在你脑子里的想法，五个你不断回想的问题。将 Claude 连接到那个文件夹。让它找到这五条笔记中的关联。

它会找到你错过的东西。它总是这样。那一刻——当 Claude 浮现两件你以为完全不相关的事情之间的关联时——是系统从一个概念变成你每天都想喂养的东西的时刻。

今晚从五条笔记开始。知识库会做剩下的事。
