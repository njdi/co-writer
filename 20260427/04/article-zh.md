# 关于 AI，我说几句实话

**作者：** Karri Saarinen ([@karrisaarinen](https://x.com/karrisaarinen))  
**日期：** 2026年4月26日  
**来源：** [Some Notes on AI](https://x.com/Zephyr_hg/status/2048267794924650791)

上一次模型编码能力大幅跃升，转眼快六个月了。六个月，往往是蜜月期的终点。之后，现实开始慢慢浮出水面。

我对 AI 的态度是谨慎乐观。大多数时候，我尽量活在当下的现实里，而不是活在某个遥远的未来——那个 AGI 与人类分道扬镳、建立起自己的"技术核心（TechnoCore）"社会的世界。

能力是真实的，局限也是真实的。有意思的问题不是"这会走向哪里"，而是"现在到底有什么不一样"。

我觉得当前市场上关于 AI 的叙事太过简单，说实话有点让人泄气。这场对话往往只有两端：要么乐观，要么末日，中间什么都没有。

我更感兴趣的是那个中间地带：什么在真正发生改变，什么是真正有用的，什么只是炒作，什么才是真正的风险，哪些我们还根本搞不明白。

---

## 计划没用，但做规划至关重要

最近，规划好像越来越不流行了。

我衡量 AI 影响的方式，通常是问：这件事在 AI 出现之前能不能做？就我所知，从来没有人规定必须搞长周期规划。普遍的认知一直都是要更敏捷，然而很多公司最后还是搞起了年度或半年度规划。

所以问题来了：为什么？那些规划周期解决的是什么问题？AI 真的解决了吗？

我没有完整的答案，但我常常觉得，规划的价值不在于产出（那张计划），而在于它是一次对齐和承诺的演练。它逼着整个组织聚在一起，辩论什么才是真正重要的，确定优先级，创造共识，再把这个方向传递给所有人。

有时候规划也是为了提前理清组织内部的复杂关系和边界，让各个部门能在同一张地图上导航。计划本身，其实只是让这一切会议和讨论真正发生的触发器。

所以，AI 或许能改变时间线、提升吞吐量，但我还不太看得出它怎么解决组织一直在处理的那些更深层问题。选择的必要性仍然存在。某种程度上，随着构建成本降低，选择甚至可能变得更重要。做更多东西变容易了，做错的东西也会变得更容易。

当然，这种转变还是有道理的。能力变化很快，市场偏好也在波动。超长的规划周期现在风险更高，因为计划还没完成，市场可能已经移了。

实验也有更充分的理由去做。AI 能力在持续提升，我们对如何使用它们的理解还不完整。有价值的用例、工具和工作流，往往得从实际尝试中涌现出来，不可能全都提前规划好。

这也是 Linear 从一开始的运作方式。我们有至少六个月的计划，确定整体方向，但保留每月甚至每周调整计划和优先级的能力。计划本质上是我们当时认为值得去做的判断。随着我们学到新东西、发现变化，就去调整。现在我们也在鼓励更多实验，留出更多空间去发现涌现的用例，而不是单靠计划驱动。

但如果完全没有计划、没有规划、没有方向，或者路线图短得可怜，组织也会失去方向，群龙无首。风险就是——为了错误的客户，因为错误的理由，做了错误的东西。

没有思考、没有方向、没有想清楚自己真正想实现什么，你可能最终只是做那些顺手的事。

不做规划，你也就永远不会"计划失败"——因为你根本没立过承诺、没定过目标、没给自己设过时间线。你只是随便造点什么，看看接下来发生什么，顺势让 AI 把你带向最容易的路，而不是最重要的路。

## 你在引导 AI，AI 也在引导你

这触及我之前写过的一个问题：工具总是会引导和影响工作流程，它总是在给你指路。

AI 工具不同于以往，因为它是思维工具，而不只是机械工具。它引导你、影响你工作的能力，比任何以前的工具都要强。

有一种有效的工作方式，就是不去对抗 AI，而是让它拉着走。这基本上就是 vibe coding 做好时的样子：顺着工具的纹理走，快速推进，接受模型有它自己的方向，借势而行。

这样做可以很有效，但不一定会把你带到你原本想去的地方。好不好，就看情况了。

## 专业知识的悖论

AI 在你最不熟悉的领域，往往看起来最厉害。我觉得这是市场上认知撕裂的最大根源。在外行眼里，AI 的能力是无限的。

在你深耕的领域，你能看到那些缝隙。输出结果很接近，但就是差那么点。缺失了上下文，捏造了细节，走了最显眼的路，或者需要大量干预才能变得有用。

在你不熟的领域，同样的输出感觉像变魔术，因为你没有判断力去发现缺少了什么。这是一种放大版的盖尔-曼失忆效应与邓宁-克鲁格效应的叠加——你不懂的领域越多，AI 看起来越神。

悖论在于：专业知识越深，AI 越难用，但如果你知道怎么用，它也越有价值。专家能看出烂活，但也知道怎么引导、约束和评估模型。

所以 AI 并不会消解专业知识的价值。它只是让专业知识更多地指向方向感、判断力，以及"知道什么叫好"。

## 编程

跟行业里的客户和公司聊下来，Agent 编程已经相当普遍了。但深浅不一。几乎没有人私下说他们的 agent 写了 100% 的代码，也没听说有多少真实的公司在跑大规模的独立 agent 集群。

工程师还是深度参与在引导 agent，通常也就能同时跑几个，再加后台几个处理简单任务的云 agent。再多，就活进了《哈里森·伯杰隆》的世界——什么东西都在不停地 ping 你，彻底打碎你的思路。

Linear 也是一样。我们大多数付费工作区现在都装了编程 agent，使用量在几个月内涨了超过 5 倍。各家公司内部具体怎么用云 agent 我不太清楚，但方向很明确：这些工具已经是日常开发工作流的一部分了。

在 Linear，工程师用了各种 agent，我们的交付速度也确实快了。仅靠我们自己的云编程 agent，现在每个月能修复超过 1000 个问题，还在快速增长。最大的变化是带宽变宽了——我们能处理更多以前觉得太小、太烦、太耗时的问题。

但难的问题还是难。Agent 编程并没有像叙事里说的那样大幅提速。它在边缘处有帮助：搭架子、重构、排查问题、处理小修小补。真正困难的部分，仍然需要你去理解系统、做出权衡、搞清楚该做什么。

在有经验的工程师手里，AI 的价值不是"AI 写代码，你接受"，而是围绕工作本身的加速。它可以探索方案、搭实现框架、调试报错、重构代码、写测试或者跑大规模迁移、处理小修复。

品味、约束和最终判断，还是得工程师来给。

## 设计

新的图像生成和设计能力好多了，但我还是觉得难用。

图像生成在迭代多了之后容易崩。你很难让 AI 以一种特定的方式只改一个地方，它往往同时改很多东西。每次迭代也像是在整张图上加了一层滤镜，到最后你不得不开一个新对话，重新描述，重新上传文件，从头来过。

我希望有更好的工具，能把 AI 限定在特定区域操作，而不是每次都让它把整个结果重新解释一遍。写作上也有同样的问题：要求改一处，模型往往把整篇都重塑了。我多希望图像生成上能有更好的控制手段和工作流。

## 设计与代码

我一直在思考一个问题：让设计工具直接作用于生产代码库，价值到底在哪里？

很多设计+代码工具会给你看一个 React 组件，比如一个按钮，然后让你用设计工具修改它。

但我在设计的时候，脑子里已经能想象那个按钮怎么运作了。在那个阶段，它是矢量图形还是 React 组件，对我来说几乎没有区别。

我做的很多设计工作不是生产设计，我不是在实现最终版本，也不是在测试所有边界情况。大多数设计工作是在做决策、理解问题、找到契合点。这个过程会产生很多版本和乱糟糟的想法。

我不希望每次修改都要承担生产代码的速度或 token 成本，不希望因为提示没跑完就报错，也不想主要通过文字或代码来设计。设计是视觉领域，对我来说，在基于画布的可视化工具里才最顺手。

先把"UI 还有没有必要"这个争论搁在一边。设计仍然是塑造问题和解决方案的一种方式。大多数设计工作是在搞清楚一个功能如何嵌入现有系统、如何与其他概念互动、它的边界在哪、它优化的是什么。

就算产品是通过 MCP、CLI 或 API 使用的，背后的概念和工作流也得讲得通才行。

我觉得更有用的，是更先进的视觉工具，或者语义化的 UI 设计工具——能让你更直接地定义布局和模式。画的不是一个矩形，而是一个模态框，它可以继承系统里的某些属性。AI 可以帮你填充界面、生成变体、探索方向。价值在于那些设计模式本身，而不在于这些模式有没有真实的生产代码在背后撑着。

然后，当我真的想把某个东西做成可以跑的原型时，应该有一套清晰的工作流，把设计文件整理好交给 AI，转成代码。

在 AI 的世界里，设计对我来说就像规划。那是 `/gstack` 在跑，不断地问你问题——但那些问题其实是你自己的大脑在问，答案浮现在屏幕上。

## 领域很重要

每次听到有人说某家公司是怎么用 AI 的，我都会先问：这家公司是谁？他们做的是什么方向？处于什么阶段？产品是什么？有没有客户？客户是谁？

我一直以来的观点是，每个产品需要不同程度的稳定性、信任感、安全性，连产品设计也一样——每个产品需要不同层次和类型的设计投入。

像邮件这种高频、每天都要碰的工具，需要大量 UX 打磨，因为用户能感受到每一处小划伤。用得多，小问题就会不断叠加。

后端服务就不一样了。价值在于后端逻辑。UI 可以更克制，甚至更粗糙，产品依然有价值。

我觉得很多 AI 公司更像后端公司在运作。能力就是模型本身，上面那层工具和框架在持续迭代，但大多时候还在幕后。这让他们能跑得快，因为每个新功能都可以是系统里新加的一个工具，不需要多少视觉或概念上的额外负担。

感觉更接近经典的 UNIX 系统——程序是文本化的，可以独立创建，在运行时自由组合。操作系统的可组合性，远比现代 macOS 这样的系统容易。

---

最后那个论点总是："现在是它们最差的时候。"

我同意。但我还是尽量贴着当下的能力走，因为我不知道它们会变成什么样。我的谨慎乐观就来自这里。做末日派没用，一厢情愿也没用。

更有用的姿态是：仔细观察，大胆去试，把判断力留住。AI 显然在改变很多东西，但我不觉得光靠"相信得更用力"就能建出什么。我们得更脚踏实地——看清楚现在什么是可能的，决定什么值得做，然后随着现实变化不断调整。

就像那句老话说的：如果潜力能换成营收，这些 CapEx 早就变成利润了。
