# 关于 AI 的一些笔记

**作者：** Karri Saarinen ([@karrisaarinen](https://x.com/karrisaarinen))  
**日期：** 2026年4月26日  
**来源：** [Some Notes on AI](https://x.com/Zephyr_hg/status/2048267794924650791)

我们正接近上一次模型编程能力大幅跃升后的第六个月。六个月通常是蜜月期。之后，一些现实开始浮现。

我对 AI 倾向于谨慎乐观。大多数时候，我试图活在当下的现实中。而不是活在某个遥远的未来，AGI 从人类中分裂出来，形成它自己的"技术核心（TechnoCore）"社会。

能力是真实的，但局限性也是真实的。有趣的问题不是"这会走向何方"，而是"现在实际上有什么不同"。

我发现当前市场上关于 AI 的叙述过于简单化，老实说有点令人沮丧。讨论往往在乐观主义和末日主义之间没有留下任何空间。

我更感兴趣的是中间地带：什么在真正改变，什么是有用的，什么被过度炒作，什么携带真实风险，以及什么是我们仍然不理解的。

---

## 计划毫无价值，但规划至关重要

最近，规划似乎正在过时。

我通常思考 AI 影响的方式是询问：同样的决定或活动在 AI 之前是否能做出。据我所知，从来没有强制要求做长期规划周期。如果说有什么的话，常识一直是要更敏捷，但许多公司最终还是采用了年度或半年度规划周期。

所以问题是：为什么？那些规划周期解决的是什么问题，AI 真的解决了那个问题吗？

我没有完整的答案，但我常常认为规划不是关于输出（计划），而是关于对齐和承诺的练习。它迫使组织聚集在一起，辩论真正重要的是什么，决定优先级，创造意义，然后与所有人分享那个方向。

有时也是为了预先处理组织的复杂性或边界，计划创建了各个部门可以在其上导航的共享地形。计划实际上只是让所有这些会议和讨论发生的强制函数。

因此，虽然 AI 可能改变时间线并增加带宽，但我还没有完全看出它如何解决组织一直在解决的其他问题。选择的需要仍然存在。在某些方面，随着构建成本下降，这些可能变得更加重要。如果做更多东西变得更容易，做错误的东西也会变得更容易。

这种转变还是有一些道理的。能力正在迅速变化。市场偏好处于动荡之中。非常长的规划周期现在承担更多风险，因为在你完成计划之前市场可能已经移动了。

也有明确的理由支持更多实验。AI 能力持续增加，我们对如何使用它们的理解仍然不完整。有价值的用例、工具和工作流程往往必须从尝试中涌现出来。它们不能总是提前规划。

这是我们从一开始在 Linear 的运营方式。我们有至少六个月的计划，设定整体方向。但我们保留每个月或每周更改计划或优先级的能力。计划总是对我们认为值得的事情的信念。当我们学习或注意到变化时，我们会调整它。我们现在也在鼓励更多实验。更多空间去发现涌现的用例，而不仅仅依赖计划。

但没有计划、没有规划、没有方向，或者只有非常短的路线图，也可能使组织失去方向感。风险变成了为错误的客户、因错误的原因构建错误的东西。

没有思考、方向，或者对你真正想要实现的目标的计划，你可能最终做任何容易做到的事情。

当你不做规划练习时，你也从不失败你的思考、承诺、计划或时间线，因为你从不做它们。你可以只是构建任何东西，看看接下来发生什么，并可能让 AI 引导你走向最容易的事情，而不是重要的事情。

## 你引导 AI，AI 引导你

这涉及到我之前写过的事情：工具总是引导和影响工作流程。它们总是向你展示路径。

AI 工具不同，因为它们是思维工具，而不仅仅是机械工具。它们引导你和你的工作的能力比以前更强。

有一种有效的工作流程，你不与 AI 对抗。你让它拉着你走。这基本上就是"氛围编程（vibe coding）"做得好时的样子：顺着工具的纹理走，快速移动，接受模型有方向并利用那个动力。

它可以很有用，同时不会把你引导到你想去的确切地方。然后取决于情况，这是好还是不好。

## 专业知识的悖论

AI 在你最不了解的领域往往感觉最令人印象深刻，我认为这是市场中不和谐的最大贡献者。对于普通观察者来说，AI 能力被描述和理解为无限的。

在你深刻理解的领域，你会看到差距。输出接近但不完全正确。它错过上下文，发明细节，选择明显的路径，或者在变得有用之前需要大量引导。

在你了解较少的领域，同样的输出感觉像魔术，因为你缺乏判断力来看到缺少什么。这是一种规模化的盖尔-曼失忆症和邓宁-克鲁格效应。

悖论是，专业知识使 AI 更难使用，但如果你知道如何以适当的方式运用，它也更有价值。专家看到了劣质输出，但他们也知道如何引导、约束和评估模型。

因此，AI 不会消除专业知识的价值。它使专业知识更多地关于方向、判断力，以及了解什么是好的。

## 编程

与行业中的客户和公司交谈，很清楚 Agent 编程现在相当标准。但有一个谱系。几乎没有人私下说他们的 agent 写了 100% 的代码。我也没听到很多真实公司运行大规模独立 agent 群的真实故事。

工程师仍然非常多地参与引导 agent，通常也许只能同时运行几个 agent，以及几个在后台做更简单修复的云 agent。超过那个数量，你就活在了《哈里森·伯杰隆》的世界里，某些东西不断地 ping 你并打断你的思考过程。

我们在 Linear 也看到了这一点。我们大多数付费工作区现在都安装了编程 agent，使用这些 agent 的活动在几个月内增加了 5 倍以上。我不确切知道每家公司内部如何使用云 agent，但方向很清楚：这些工具是正常开发工作流程的一部分。

在 Linear，工程师使用各种 agent，也看到了我们的步伐加快。仅凭我们自己的云编程 agent，我们现在每月修复超过 1,000 个问题，而且还在快速增加。我看到的最大影响是带宽增加。我们可以比以前处理更多较小的问题。那些可能太小、太烦人或太耗时的工作现在变得更容易完成。

但困难的问题仍然困难。Agent 编程并没有像叙述通常暗示的那样大幅加速这些问题。它在边缘处有所帮助，包括脚手架、重构、调查和较小的修复。但困难的部分仍然需要理解系统、做出权衡，以及知道应该存在什么。

在专业编程工作流程中，价值通常不是"AI 写代码，你接受它"。价值是围绕工作的加速。它可以探索方法、搭建实现框架、调试错误、重构代码、编写测试或大型迁移，以及处理较小的修复。

专家仍然提供品味、约束和最终判断。

## 设计

新的图像生成和设计能力好多了，但我仍然觉得它们很有挑战性。

图像生成似乎迭代次数越多就越容易崩溃。很难让 AI 以一种特定的方式更改一件特定的事情。它往往会同时更改很多东西。每次迭代也似乎在整个图像上添加某种滤镜，你可能需要用新的对话、新的描述和新文件重新开始来修复它。

我希望看到更好的工具将 AI 限制在特定区域，而不是每次都让它重新解释整个输出。这在写作中也会发生，要求一处更改往往会导致模型重塑整篇文章。我希望图像生成有更好的控件和工作流程。

## 设计与代码

我不断思考让设计工具直接在生产代码库上操作的价值。

许多设计+代码工具显示一个 React 组件，比如一个按钮，然后让你用设计工具更改它。

但当我在设计时，我可以想象按钮是如何工作的。在那个阶段，它是矢量形状还是 React 按钮对我来说几乎没有区别。

我做的很多设计工作不是生产设计。我不是在尝试实现最终版本或测试每个边缘情况。大多数设计工作是关于做决定、理解问题、找到契合点。那个过程产生很多变体和混乱的想法。

我不希望每次更改都承担生产代码的速度或 token 成本。我不想因为我的提示没有完成而陷入错误。我也不想主要通过文字或代码来设计。设计是一个视觉领域，对我来说它在基于画布的可视化工具中效果最好。

如果我们把关于 UI 是否仍然需要的辩论放在一边。设计仍然是塑造问题和解决方案的一种方式。大多数设计工作是关于理解一个功能如何适合现有系统，它如何与其他概念互动，它的限制是什么，以及它为什么而优化。

即使产品通过 MCP、CLI 或 API 使用，概念和工作流程仍然需要有某种意义。

我觉得更有用的是高级视觉工具，或语义 UI 设计工具，在那里你可以更直接地定义布局和模式。不是画一个矩形，而是画一个模态框，它可以继承系统的一些方面。AI 可以帮助填充屏幕、生成变体和探索方向。价值在于模式，而不是那些模式是否由真实的生产代码支持。

然后，一旦我有了我真正想要实时原型的东西，应该有一个清晰的工作流程，将设计文件提炼给 AI 并将其转换为代码。

在 AI 世界里，设计对我来说就像规划。这是 `/gstack` 在运行，向你提问，但实际上是你自己的大脑在提问，在屏幕上找到解决方案。

## 领域很重要

对于几乎任何关于公司如何使用 AI 所说的话，我都会尝试问：他们是谁，他们的领域是什么，他们处于什么阶段，他们的产品是什么，他们有客户吗，那些客户是谁？

我一直认为每个产品需要不同级别的稳定性、信任、安全性，甚至产品设计，每个产品需要不同级别和类型的设计。

像电子邮件这样频繁使用的、触感重要的工具需要大量 UX 打磨，因为用户感受到每一个小摩擦。他们不断地使用它。小摩擦会累积。

后端服务不同。价值可能在于后端逻辑。UI 可以更有限，甚至更粗糙，产品仍然可以有价值。

我认为许多 AI 公司更像后端公司那样运作。能力就是模型。上面的工具和框架正在被迭代，但到目前为止，它们通常在幕后。这帮助他们快速移动，因为每个功能都可以是系统中的另一个工具，没有多少视觉或概念足迹。

感觉更接近经典的 UNIX 系统，在那里程序是基于文本的，可以独立创建并在运行时组合。操作系统的可组合性比现代 macOS 更容易。

---

最后的论点总是"这些是它们有史以来最差的时候"。

我同意，但我试图接近当前的能力，因为我不确切知道它们会变成什么。这就是我的谨慎乐观的来源。我觉得做一个末日主义者没有太多价值，但我也觉得一厢情愿没有太多价值。

更有用的立场是仔细观察、尝试事物，并保持判断力完整。AI 显然在改变事物，但我不认为我们可以今天只是通过更坚定地相信来构建。我们需要更现实，注意现在什么是可能的，决定什么值得做，并随着现实变化进行调整。

就像那句老谚语一样，如果潜力是收入，所有这些资本支出早就变成利润了。
