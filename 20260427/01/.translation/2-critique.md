# 直译稿问题清单

## 标题
- "将月收入5千美元的创业者与月收入5万美元的创业者区分开来的11个AI习惯" — 太长、太平，原文标题有冲击感。"Separate A From B"是一刀切开，译文应保留这种力道。

## 习惯1
- "构建上下文层，而非提示词库" — "上下文层"技术感强，普通读者不直观。原文 context layers 的意思是"让AI每次干活前都能读到的背景信息体系"，翻成"上下文体系"更清楚。
- "上下文产生的输出效果比任何巧妙的提示词都好10倍" — 拖沓。原文 "Context produces 10x better output than any clever prompt" 很干脆，译文要短。

## 习惯4
- "商机管道价值" — pipeline value 的直译，中文没有这个说法，应译为"在谈项目总额"或"销售漏斗价值"。

## 习惯7
- "他们在每次销售电话前用书面议程预先定框架" — 绕口。"pre-frame"是提前锁定谈话框架，可以译为"提前发议程、锁定谈话框架"，或者直接重写为"每次销售电话前，他们都提前发一份书面议程"。

## 习惯8
- "12个月内每天坚持小量创作所带来的复利效应，是大多数5万美元/月收入者整个客户获取渠道的来源" — 一句话太长，结构复杂，读起来费力。需要拆开。

## 习惯11
- "毫不留情地保持小规模" — "ruthlessly stays small" 直译，有些生硬。"死守小团队规模"或"死活不扩张"更有原文的那股狠劲。

## 技能栈一节
- "提示词工程用于AI思维。上下文工程用于让AI在商业中发挥作用的系统。Make和n8n用于运行运营的工作流程。" — 结构不对称、不流畅。原文三句平行，译文要保持同样的节奏感。
- 建议改为：提示词工程，管AI怎么想。上下文工程，管AI怎么在业务里用起来。Make和n8n，管工作流程跑起来。

## 整体语气
- 原文节奏短促、有力，大量短句。直译稿整体还原度不错，但部分段落句子偏长，需要在意译时进一步拆短，保持原文的"打点"感。
