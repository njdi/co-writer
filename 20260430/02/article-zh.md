# 如何在 n8n 中搭建一个永不枯竭的内容创意引擎

**作者：** Aaron ([@IAmAaronWill](https://x.com/IAmAaronWill))  
**日期：** 2026年4月29日  
**来源：** [How to build a content idea engine in n8n that never runs dry](https://x.com/Zephyr_hg/status/2049459882844143696)

做个人品牌的人，大多数撑不过 90 天。

不是因为不会写。是因为不知道写什么。

每天早上都是同一场仗：盯着空白页发呆，在 X 上刷帖子找灵感，最后发了个千篇一律的东西，自己也看不下去，明天接着重来。

问题不在于自律。在于你缺一个替你提前把题目想好的系统。

这里是一套完整的 n8n 工作流搭建方案，每周一早上自动给你 30 个随时可以动笔的创意——来自你真实的生活、真实的工作、真实的受众。节点具体，提示词具体，输出具体。

看完这篇文章，你会有一个在你睡觉时就把内容创作最难的 80% 干完的系统。

## 这个系统到底做什么

这个引擎按计划每周跑一次。它从你的创意实际存在的每个地方拉取原始素材，带着一份了解你的声音和受众的提示词交给 Claude，然后把 30 个完整成形的内容创意汇进一份文档，你整周都可以从里面取用。

你不再需要坐下来"想内容"了。你坐下来写列表上第三个创意就行。

就这一件事。干掉内容创作里最让人崩溃的那个环节。

这个系统有五件事要做：

1. 从你这周收集原始素材
2. 整理成 Claude 可以使用的结构
3. 带着一份了解你声音的提示词发给 Claude
4. 把输出解析成可用的内容简报
5. 把这些简报写进你真正会去打开的文档或数据库

每件事对应工作流的一个部分。按顺序搭，一个下午就能跑起来。

## 你要从哪五个来源取数据

大多数创意生成器输出垃圾，原因很简单——它们是在从空无一物里提取东西。你让 ChatGPT 给你出内容创意，它给你的是一份所有人都能拿到的回收列表，因为它对你一无所知。

你的引擎需要上下文。真实的上下文。来自那些本来就藏着好内容原材料的地方。

真正重要的来源：

**你自己写过的东西。** 笔记、语音备忘录、没写完的草稿、凌晨两点在手机上打下的想法。这是你最好的想法诞生、又在这里消失的地方。大多数人丢掉了其中 90%，只因为没有一套把它们重新翻出来的机制。

**你的客户工作。** 这周你解决了什么问题。通话里聊了什么。多个客户之间重复出现的规律。客户一直在问的问题。这些东西别人没法写，因为他们没经历过。这些东西也最容易转化，因为它直接连着你在卖的服务。

**你的受众。** 私信、评论、回复、你反复被问到的问题。这周有三个人问了你同一件事，那就是一篇帖子。这个月五次销售电话上你都回答了同一个问题，那也是一篇帖子。你真正去听，受众自然会告诉你写什么。

**你收藏的内容和书签。** 你收藏的文章、帖子、话题串、推文，因为它击中了你。你当时收藏是有原因的。大多数人再也没回去看过。你的引擎应该把它们重新翻出来，作为新角度的原材料。

**你过去表现好的内容。** 之前爆过的内容，换个角度还能再爆。引擎需要知道你哪些帖子发得最好，这样才能把它们作为参照——你的受众对什么真正有反应。

在 n8n 里，每个来源对应一个独立节点，从你实际存数据的地方拉取。Notion、Google Docs、Twitter 导出、用 Apify 抓你自己的主页、你的 CRM、你的邮件，随便什么。重点不是用哪个工具。重点是你给这个系统喂的是你真实生活里的信号，而不是让 AI 凭空发明想法。

如果你现在这些东西根本没有集中记录，那这才是第一步。花一周时间把它们都捕捉起来。一个简单的 Notion 数据库，五张表——笔记、客户问题、受众问题、收藏内容、顶级帖子——就够了。搭好之后随手填，引擎每周自动去取。

## n8n 搭建流程，逐节点拆解

下面是工作流从上到下的结构。

**1. 定时触发器。**

设置为每周一早上 6 点触发。系统在你睡觉时运行。等你坐下来喝咖啡，创意已经在那里等你了。

**2. 数据收集节点。**

每个来源一个节点，每个节点只拉取过去 7 天的数据。

Notion 来源，用 Notion 节点，加上"最后编辑时间在过去 7 天内"的过滤器。这样只拉新内容，不是每次把整个数据库都取一遍。

Google Docs 或 Sheets，用对应的节点加日期过滤器。逻辑一样。

过去的 Twitter 内容，可以用 Apify 的 Twitter 抓取节点，或者手动维护一张 Google Sheet 存你的顶级帖子，让节点从里面拉。

客户问题，最简单的做法是维护一个 Notion 或 Sheets 数据库，每次通话结束记一行。引擎拉过去一周的条目。

私信和受众问题，很难做到完全自动化。务实的做法：平时遇到有意思的私信、评论、问题，随手粘贴进一个 Notion 页面。引擎周一来取这个页面。

每次只拉少量、近期的数据。你不需要全部东西。你需要的是这周的信号。给 Claude 喂六个月的数据，输出只会越来越差，不会更好。

**3. 合并与结构节点。**

从各个来源收完数据后，你有五条独立的数据流。用 Merge 节点把它们合在一起，再用 Set 节点给每块数据清晰地打上标签。

Set 节点的输出大概长这样：

```
PERSONAL NOTES THIS WEEK:
[content from notes pull]

CLIENT PROBLEMS THIS WEEK:
[content from client log]

AUDIENCE QUESTIONS THIS WEEK:
[content from DM/comment log]

SAVES THIS WEEK:
[content from bookmarks pull]

TOP PERFORMING POSTS (LAST 90 DAYS):
[content from analytics pull]
```

这个标签写法至关重要。没有它，Claude 把所有东西当成一团混在一起处理，输出就会变差。有了它，Claude 能区分"这是我自己记下来的"和"这是我受众问的"，生成创意时权重就对了。

**4. Claude API 节点。**

魔法在这里发生。用 HTTP Request 节点配置好调用 Claude API，或者如果你装了专用插件，用 Anthropic 节点也行。

模型：想要最好的输出质量用 claude-opus-4-7，想要速度和更低成本用 claude-sonnet-4-6。对于每周跑一次的任务，成本差异可以忽略不计。用 Opus。

最大 tokens：8000。你要 30 个详细的创意，空间得够。

系统提示词和用户提示词的结构：下一节详细说。

**5. 解析节点。**

Claude 返回的是一整块文本。你需要把它拆成 30 条独立的创意条目，才能逐行写进数据库。

最简单的做法：在提示词里告诉 Claude 以 JSON 格式返回。然后用 n8n 的 Code 节点把 JSON 解析成独立的条目，每条变成输出里的一行。

如果 JSON 让你觉得不踏实，另一个办法是让 Claude 用特定分隔符比如 `---IDEA---` 隔开每个创意，再用 Split 节点把响应切成块。

**6. 输出节点。**

把解析好的创意发到你周一早上真正会去打开的地方。

我推荐的设置：一个 Notion 数据库，列有钩子、角度、格式、要点、来源和状态。引擎每周一创建 30 行新记录。你整周处理它们，把每一条标记为已写、已排期或已跳过。

不用 Notion 的话，把创意格式化成编号列表放在 Google Doc 里也行。或者让它发邮件给你。选你真正会打开的地方就行。

整套搭建就这些。六个功能模块。熟悉 n8n 的用户三到四小时搞定。第一次上手的用一天。

## 让它出好结果的提示词

烂提示词会产出 30 个"提升生产力的 5 个技巧"。好提示词会产出 30 个你真正想写的创意。

提示词需要四件事，按这个顺序：

**声音上下文。**

两三段描述你怎么写作、你不做什么、你的语气是什么样的。要具体。

差的："用随意的、对话式的语气写。"

好的："直接，不拐弯。段落短，一到三句话。没有企业腔，没有流行语，没有咨询范儿的废话。语境合适时偶尔骂几句。总是用缩写。没有破折号。写法像在酒吧跟人解释一件事，不是在会议室做演讲。"

声音描述越具体，输出听起来越像你自己写的。

**受众上下文。**

谁在看。他们想要什么。他们看腻了什么。他们处于什么阶段。

差的："创业者和企业主。"

好的："在搭自动化业务的独立创业者和小型代理商主理人。他们已经懂 n8n 基础了。他们想要的是商业角度、定价逻辑、具体的搭建方案。他们看够了'什么是 API'这种入门内容。默认他们有技术背景。默认他们想赚钱。"

**效果上下文。**

你有史以来最顶的三到五条帖子，原文逐字粘进去。有互动数据的话一起附上。

这是最大的杠杆。它教 Claude 你的受众实际上对什么有反应，而不是你认为他们应该对什么有反应。

手动提取一次，硬编码进提示词。每隔几个月有新的爆款出来，更新一下。

**输出规格。**

精确地告诉 Claude 你要什么。不只是一个话题。每个创意需要四个字段：

- **钩子**：帖子或文章的开场白
- **角度**：是什么让这个创意和同一话题的普通版本不一样
- **格式**：帖子、话题串、文章或视频
- **要点**：读者离开时知道或者能做到的具体事情

30 个创意，每个创意四个字段，以 JSON 格式返回。

提示词有了这四个部分，输出就不再千篇一律了。开始有点像你状态最好那天自己想出来的创意——只是从 Claude 30 秒的处理，替代了你自己三个小时的冥想苦想。

## 怎么维护这套输入而不让它变成额外负担

引擎好不好，取决于喂进去的数据。来源是空的，输出就是空的。

诀窍是让维护输入这件事没有任何摩擦。

**笔记**：整周随手录语音备忘录。有一个 Notion 页面或 Apple Notes 文档，想到什么就往里扔。不用整理，不用润色，只管记录。引擎处理的就是这些粗糙的原材料。

**客户工作**：每次通话结束，花 60 秒记下客户的核心问题和你怎么解决的。数据库里一行。就这样。

**受众问题**：有人在私信或评论里问了你一个好问题，把它复制粘贴进你的 Notion 记录里。五秒钟的事，回报巨大。

**收藏内容**：用平台原生的收藏功能就行。引擎每周去取一次。

**顶级帖子**：每季度更新一次。拉一下你的数据，找出前五，粘进提示词。完成。

整套输入维护，每天摊下来大概 10 分钟，分散在整周里。换来的是每周 30 个内容创意，完全不需要你在创意上动脑子。这就是这笔交换。

## 为什么这套方法比其他方法都强

你不是在让 AI 替你有创意。你是在让一个比你休息得更好、整理得更清楚的「自动版自己」上线，它能调用你有时间好好坐下来想的时候本来就会用到的所有素材。

信号是你的。声音是你的。对受众的了解是你的。Claude 只是在做那一层综合——你周一早上 9 点那颗疲惫的脑袋做不好的那层。

所以输出不像 AI 写的垃圾。基础不是 AI，是你真实的生活，经过一套系统被组织成内容。

这套方法有效还有另一个原因：它干掉了内容创作里最难受的部分——冷启动。从零到第一稿，是大多数人放弃的地方。周一早上有 30 个创意在等着，你每次都直接跳过了那一步。

## 今天就去做

打开 n8n。新建一个工作流。

加一个定时触发器。设成周一早上 6 点。

先接一个数据源。最容易的那个。大概是你平时就在往里记东西的 Notion 页面或 Google Doc。加上 7 天过滤器，让它正常拉取。

加上 Claude API 节点。写一版包含四个部分的提示词草稿。先别追求完美，让整条链路先跑通。

手动跑一次。看看输出。大概能做到 60%。

加第二个数据源。再跑一次。输出会好一些。

把五个来源都接上，根据输出里缺少或不对的地方调整提示词。不同的周分别跑几次，调整几轮。

三周之内，你会有一个比你自己单打独斗想出来的创意质量更好的系统。除了在日常生活里顺手记录原材料，不需要任何额外的持续投入。

用 n8n 搭东西，整个意义就在于把你不想做的工作干掉，让你有精力做更多你愿意做的。

想内容创意，就是大多数人最不想做的那件事。搭一次，从此没了。
