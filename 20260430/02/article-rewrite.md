# 如何在 n8n 中搭建一个永不枯竭的内容创意引擎

**作者：** Aaron ([@IAmAaronWill](https://x.com/IAmAaronWill))  
**日期：** 2026年4月29日  
**来源：** [How to build a content idea engine in n8n that never runs dry](https://x.com/Zephyr_hg/status/2049459882844143696)

做个人品牌的人，大多数熬不过 90 天。

不是因为不会写。是因为想不出写什么。

每天早上的状态大概是这样的：打开电脑，盯着空白页发一会儿呆，去 X 上刷一圈找灵感，最后随便发了个差不多的东西，自己都觉得没劲，明天接着来一遍。

真正的问题不是缺乏自律。是缺少一套在你坐下来之前就已经把题目想好了的系统。

这篇文章要介绍的，就是这样一套 n8n 工作流。它每周一早上自动给你准备好 30 个可以动笔的创意——来自你真实的生活、真实的工作、真实的读者。节点具体，提示词具体，输出具体。

看完这篇，你会有一个在你睡觉时就把内容创作最难的 80% 处理好的系统。

## 这个系统到底在做什么

说起来也不复杂。

这个引擎每周按计划跑一次。它去你的创意实际存在的几个地方，把原始素材拉回来，再带着一份了解你写作风格和读者的提示词，交给 Claude 去处理。最后，30 个完整成形的内容创意会汇进一份文档，你整周都可以从里面取用。

你不需要再坐下来"想内容"了。你坐下来，打开列表，写第三条就行。

整件事的关键就在这里：省掉内容创作里最让人犯难的那个起步环节。

这个系统要做五件事：

1. 从你这周的生活和工作里收集原始素材
2. 整理成 Claude 可以读懂的结构
3. 带着一份了解你声音的提示词发给 Claude
4. 把输出解析成一条条可用的内容简报
5. 把这些简报放进你真正会去打开的文档或数据库

每件事对应工作流的一个部分。按顺序搭，一个下午就能让整条链路跑起来。

## 从哪里取素材

很多创意生成工具效果差，原因说穿了很简单——它们是在从空无一物里挖东西。你让 ChatGPT 给你出内容创意，它给你一份所有人都能得到的通用列表，因为它对你的情况一无所知。

你的引擎需要真实的上下文。来自那些本来就藏着好内容原材料的地方。

真正值得接入的来源有五个：

**你自己写过的东西。** 笔记、语音备忘录、没写完的草稿、凌晨两点随手打下的想法。你最好的思考都在这里产生，也在这里消失。大多数人丢掉了其中 90%，只因为没有一套把它们重新翻出来的机制。

**你的客户工作。** 这周你解决了什么问题。通话里聊了什么。几个客户之间反复出现的规律。客户一直在问的问题。这些东西别人没法写，因为他们没经历过。而且往往最容易带来转化，因为它直接连着你在做的事情。

**你的读者。** 私信、评论、回复、反复被问到的问题。这周有三个人问了你同一件事，那就是一篇帖子的题材。这个月在五次通话里你都解释了同一个问题，那也是。你真正去听，读者自然会告诉你写什么。

**你收藏的内容。** 你存过的文章、帖子、推文，当时觉得有意思就存下来了，然后再也没回去看过。大多数人都是这样。你的引擎可以把它们重新翻出来，作为新角度的参考素材。

**你过去反响好的内容。** 之前好的东西，换个角度通常还能再好。引擎需要知道你哪些内容发出去效果不错，这样才能把它们作为参照，了解你的读者对什么真正感兴趣。

在 n8n 里，每个来源对应一个节点，从你实际存数据的地方取就好。Notion、Google Docs、Twitter 导出、用 Apify 抓你自己的主页、你的 CRM、你的邮件，用哪个都行。重点不是工具，而是你给这个系统喂的是真实生活里的信号，不是让 AI 凭空发明想法。

如果你现在这些东西根本没有集中记录，那这是第一步。花一周时间把它们整理起来。一个简单的 Notion 数据库，五张表——笔记、客户问题、受众问题、收藏内容、高效帖子——就够了。搭好之后随手填，引擎每周自动去取。

## n8n 搭建流程，逐节点拆解

工作流从上到下，大概是这个样子。

**1. 定时触发器。**

设置为每周一早上 6 点自动触发。系统在你睡觉时跑完。等你坐下来喝咖啡，创意已经在那里等着了。

**2. 数据收集节点。**

每个来源一个节点，每个节点只取过去 7 天的数据。

Notion 来源，用 Notion 节点，加上"最后编辑时间在过去 7 天内"的过滤器。这样只取新内容，不是每次把整个数据库都捞一遍。

Google Docs 或 Sheets，用对应的节点加日期过滤器。逻辑一样。

Twitter 的历史内容，可以用 Apify 的 Twitter 抓取节点，也可以手动维护一张 Google Sheet 存你的高效帖子，让节点从里面取。

客户问题，最简单的做法是维护一个 Notion 或 Sheets 数据库，每次通话结束记一行。引擎取过去一周的条目就好。

私信和读者问题，这部分很难做到完全自动化。比较实际的做法是：平时遇到有意思的私信、评论、问题，随手粘进一个 Notion 页面。引擎周一来取这个页面。

每次只取少量、近期的数据，不用追求全量。你需要的是这周的信号。给 Claude 喂六个月的数据，输出反而会变差，不会更好。

**3. 合并与结构节点。**

各个来源的数据收进来之后，你有五条独立的数据流。用 Merge 节点把它们合在一起，再用 Set 节点给每块数据打上清楚的标签。

Set 节点的输出大概长这样：

```
本周个人笔记：
[笔记来源的内容]

本周客户问题：
[客户记录的内容]

本周受众问题：
[私信/评论记录的内容]

本周收藏内容：
[书签来源的内容]

近 90 天高效帖子：
[数据分析来源的内容]
```

这个标签写法很重要。没有它，Claude 会把所有东西当成一团来处理，输出质量就会下降。有了它，Claude 能区分"这是我自己记下来的"和"这是读者问的"，生成创意时就有了不同的侧重。

**4. Claude API 节点。**

这一步是整个流程的核心。用 HTTP Request 节点配置好调用 Claude API，或者安装了专用插件的话，用 Anthropic 节点也行。

模型方面：想要最好的输出质量，用 claude-opus-4-7；想要速度和成本更低，用 claude-sonnet-4-6。每周跑一次，两者的成本差别不大。用 Opus 就好。

最大 tokens 设为 8000。你要 30 个详细的创意，得给够空间。

系统提示词和用户提示词的具体写法，下一节详细说。

**5. 解析节点。**

Claude 返回的是一整块文本，你需要把它拆成 30 条独立的创意条目，才能逐行写进数据库。

最简单的做法：提示词里告诉 Claude 以 JSON 格式返回。然后用 n8n 的 Code 节点把 JSON 解析成独立条目，每条变成输出里的一行。

如果你觉得 JSON 不够可靠，可以让 Claude 用固定分隔符比如 `---IDEA---` 把每个创意隔开，再用 Split 节点把响应切成块。

**6. 输出节点。**

把解析好的创意发到你周一早上真正会去打开的地方。

我推荐的设置是：一个 Notion 数据库，列有钩子、角度、格式、要点、来源和状态。引擎每周一生成 30 行新记录，你整周处理它们，把每一条标记为已写、已排期或已跳过。

不用 Notion 也行，把创意格式化成编号列表放在 Google Doc 里，或者直接让系统发邮件给你。选一个你真正会打开的地方就好。

整套搭建就这些。六个功能模块。熟悉 n8n 的用户三到四小时能搞定，第一次上手的话大概要一天。

## 提示词怎么写

话说回来，这套系统能不能出好结果，很大程度上取决于提示词写得怎么样。

写得差，产出的是 30 个"提升生产力的 5 个技巧"。写得好，产出的是 30 个你真正想写的东西。

提示词需要四个部分，按顺序来：

**声音上下文。**

写两三段，说清楚你怎么写作、你不做什么、你的语气是什么感觉。要具体。

模糊的写法："用随意的、对话式的语气。"

具体的写法："直接，不拐弯。段落短，一到三句话。没有企业腔，没有流行语，没有咨询范儿的废话。语境合适时偶尔骂几句。总是用缩写。没有破折号。写法像在酒吧跟人解释一件事，不是在会议室做演讲。"

描述越具体，输出越接近你本人写出来的感觉。

**受众上下文。**

谁在看。他们想要什么。他们看腻了什么。他们处于什么阶段。

模糊的写法："创业者和企业主。"

具体的写法："在搭自动化业务的独立创业者和小型代理商主理人。他们已经懂 n8n 基础了。他们想要的是商业角度、定价逻辑、具体的搭建方案。他们看够了'什么是 API'这种入门内容。默认他们有技术背景，默认他们想赚钱。"

**效果上下文。**

把你有史以来反响最好的三到五条帖子，原文逐字粘进提示词。有互动数据的话一起附上。

这是所有部分里影响最大的一块。它让 Claude 看到你的读者实际上对什么有反应，而不是你猜测他们应该对什么有反应。

手动提取一次，写死在提示词里。每隔几个月有新的高效内容出来，更新一下就好。

**输出规格。**

告诉 Claude 你具体想要什么。不只是一个话题，每个创意需要四个字段：

- **钩子**：帖子或文章的开场白
- **角度**：是什么让这个创意和同一话题的普通版本不一样
- **格式**：帖子、话题串、文章或视频
- **要点**：读者读完能知道或能做到的具体事情

30 个创意，每个四个字段，以 JSON 格式返回。

提示词有了这四个部分，输出就不一样了。开始有点像你状态最好那天自己想出来的东西，只是把三个小时的冥想苦想，压缩进了 Claude 30 秒的处理。

## 怎么维护这套输入，而不让它变成另一件负担

说到这里，有一件事值得单独说清楚。

这套系统的质量，完全取决于你往里放了什么。来源是空的，输出就是空的。

所以关键是让维护这件事没有摩擦感——不是靠自律去坚持，而是让它自然地融进你本来的日常里。

**笔记**：整周随手录语音备忘录。有一个 Notion 页面或 Apple Notes，想到什么就往里扔一下。不用整理，不用润色，只管记录。引擎处理的就是这些粗糙的原始材料。

**客户工作**：每次通话结束，花 60 秒记下客户提出的核心问题和你怎么处理的。数据库里一行。就这些。

**受众问题**：有人在私信或评论里问了你一个有意思的问题，把它复制进你的 Notion 记录里。五秒钟的事，后续的回报很值。

**收藏内容**：用各平台原生的收藏功能就行，不需要额外整理。引擎每周自动去取一次。

**高效帖子**：每季度更新一次。查一下数据，找出前五，粘进提示词。完成。

把这些加在一起，每天大概花 10 分钟，分散在整周里。换来的是每周 30 个内容创意，完全不需要你专门坐下来想点什么。这就是这笔交换。

## 为什么这套方法不一样

有一点值得说清楚：你不是在让 AI 替你想东西。

你是在让一个比你休息得更好、整理得更清楚的"另一个自己"来帮你做综合——它能调用你有时间好好坐下来想的时候本来就会用到的所有素材。

信号是你的。声音是你的。对读者的了解是你的。Claude 只是在做那一层整理，那一层你周一早上九点的脑袋不太做得好的工作。

所以输出不像通常意义上的 AI 生成内容。基础不是 AI，是你真实的生活，经过一套系统被组织成了可以写的东西。

这套方法还有一个让它真正好用的地方：它处理掉了最难的那一步，就是冷启动。从空白到第一稿，是大多数人放弃的地方。每周一早上有 30 个题目在等着，你每次都直接从第二步开始。

## 今天就可以开始

不用一次搭完。先从一个最简单的版本跑起来，然后慢慢加。

打开 n8n，新建一个工作流。

加一个定时触发器，设成周一早上 6 点。

先接一个数据源，选最容易的那个。大概是你平时就在往里记东西的 Notion 页面或 Google Doc。加上 7 天过滤器，让它正常取数。

加上 Claude API 节点，写一版包含四个部分的提示词草稿。先不用追求完美，让整条链路先跑通。

手动跑一次，看看输出是什么样的。大概能到 60% 的状态。

加第二个数据源，再跑一次。输出会好一些。

把五个来源都接上，根据结果里哪里不对、哪里缺少什么，调整提示词。跨不同的周跑几次，慢慢调整。

三周之内，你会有一个比自己单独想出来的创意质量更稳定的系统。日常只需要顺手记录原材料，不需要额外的持续投入。

用 n8n 搭东西，意义就在这里：把你不想做的那些事情交出去，让你有精力做更多你真正想做的。

想内容、找题目，就是大多数做个人品牌的人最头疼的事。搭好这套系统，这件事就不用自己操心了。
