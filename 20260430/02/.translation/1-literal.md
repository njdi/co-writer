# 如何在 n8n 中构建一个永不枯竭的内容创意引擎

**作者：** Aaron ([@IAmAaronWill](https://x.com/IAmAaronWill))  
**日期：** 2026年4月29日  
**来源：** [How to build a content idea engine in n8n that never runs dry](https://x.com/Zephyr_hg/status/2049459882844143696)

大多数运营个人品牌的人会在 90 天内精疲力竭。

不是因为他们不会写。而是因为他们想不出写什么。

每天早晨都是同样的战斗：盯着空白屏幕，在 X 上刷内容找灵感，最终发了些千篇一律的东西，讨厌自己，明天重复。

解决方法不是自律。而是一个在你坐下来之前就已经完成思考的系统。

这里是一个 n8n 工作流的完整搭建方法，它每周一早上给你 30 个随时可以动笔的内容创意，来自你真实的生活、你真实的工作，以及你真实的受众。具体的节点，具体的提示词，具体的输出。

读完这篇文章，你将拥有一个在你睡觉时完成内容创作中最难的 80% 工作的系统。

## 这个系统实际上做什么

这个引擎按计划每周运行一次。它从你的创意已经存在的每个地方提取原始信号，将该信号连同一个了解你的声音和受众的提示词一起交给 Claude，然后将 30 个完整成形的内容创意放入一个你可以整周使用的单一文档中。

你不再坐下来"想内容"了。你坐下来写列表上的第三个创意。

这就是整个游戏。消除内容创作中杀死大多数人的那个部分。

这个系统有五个任务：

1. 从你这周收集原始信号
2. 整理这些信号让 Claude 可以使用
3. 用一个了解你声音的提示词发送给 Claude
4. 将输出解析为可用的内容简报
5. 将这些简报放入一个你真正会打开的文档或数据库中

每个任务是工作流的一个部分。按顺序构建它们，你将在一个下午让整个系统运行起来。

## 你从哪五个来源提取数据

大多数创意生成器输出垃圾内容的原因是它们在从空无一物中提取。你向 ChatGPT 索要内容创意，它给你一个所有人都能得到的同样的回收列表，因为它没有任何上下文。

你的引擎需要上下文。真实的上下文。来自那些已经包含好内容原材料的地方。

真正重要的来源：

**你自己的写作。** 笔记、语音备忘录、写了一半的草稿、你在凌晨 2 点在手机上打出的想法。这是你最好的思考诞生和消亡的地方。大多数人失去其中 90%，因为没有系统来重新浮现它们。

**你的客户工作。** 你这周解决的问题。通话中的对话。多个客户之间的规律。客户一直在问的问题。这是没有人能写的东西，因为他们没有亲身经历过。这也是能转化的东西，因为它直接与你销售的工作相关。

**你的受众。** 私信、评论、回复、你不断被问到的问题。如果这周有三个人问了你同样的问题，那就是一篇帖子。如果你这个月在五次销售电话上回答了同样的问题，那就是一篇帖子。如果你真正倾听，你的受众会告诉你写什么。

**你保存的内容和书签。** 你保存的文章、帖子、话题串、推文，因为某些内容触动了你。你保存它们是有原因的。大多数人再也不会回去看它们。你的引擎应该将它们重新纳入流通，作为新角度的原材料。

**你过去表现好的内容。** 之前爆的内容换个角度还会再爆。引擎需要知道你的顶级帖子是什么，这样它就可以将它们用作你的受众真正响应的参考点。

在 n8n 中，每一个都是一个独立的节点，从你实际存储数据的任何地方拉取。Notion、Google Docs、Twitter 导出、用 Apify 抓取你自己的个人资料、你的 CRM、你的邮件，随便什么。重点不在于具体工具。重点在于你用你生活中的真实信号喂养这个系统，而不是要求 AI 从虚无中发明想法。

如果你目前没有以任何集中的方式追踪这些东西，那是第一步工作。花一周时间捕捉它们。一个有五个表格的简单 Notion 数据库——笔记、客户问题、受众问题、保存的内容、顶级帖子——就足够了。一旦设置好，你随时填写，引擎每周从中拉取。

## n8n 的搭建，逐节点说明

这是从上到下的工作流结构。

**1. 定时触发器。**

设置为每周一早上 6 点触发。系统在你睡觉时运行。当你坐下来喝咖啡的时候，创意已经在等着你了。

**2. 数据收集节点。**

每个来源一个节点。每个节点从该来源拉取过去 7 天的数据。

对于 Notion 来源，使用带有"最后编辑时间在过去 7 天内"过滤器的 Notion 节点。这只拉取新内容，而不是每周拉取你的整个数据库。

对于 Google Docs 或 Sheets，使用相应的节点加上日期过滤器。同样的逻辑。

对于你过去的 Twitter 内容，你可以使用 Apify 的 Twitter 抓取节点，或者从你手动更新顶级帖子的 Google Sheet 中拉取。

对于客户问题，最简单的版本是一个 Notion 或 Sheets 数据库，你在每次客户通话后记录一行笔记。引擎拉取过去一周的条目。

对于私信和受众问题，这很难完全自动化。实用的方法：在周中随时遇到有趣的私信、评论和问题时，保留一个持续更新的 Notion 页面，将它们粘贴进去。引擎周一拉取该页面。

保持每次拉取的量小而新鲜。你不需要所有东西。你需要本周的信号。如果你试图给 Claude 喂六个月的数据，输出会越来越差，而不是越来越好。

**3. 合并和结构节点。**

从每个来源收集数据后，你将有五个独立的数据流。使用 Merge 节点将它们合并，然后使用 Set 节点清晰地标记每个数据块。

Set 节点应该输出类似这样的内容：

```
PERSONAL NOTES THIS WEEK:
[content from notes pull]

CLIENT PROBLEMS THIS WEEK:
[content from client log]

AUDIENCE QUESTIONS THIS WEEK:
[content from DM/comment log]

SAVES THIS WEEK:
[content from bookmarks pull]

TOP PERFORMING POSTS (LAST 90 DAYS):
[content from analytics pull]
```

这种标记至关重要。没有它，Claude 将所有东西视为一个未分化的整体，输出就会变差。有了它，Claude 知道"我记下来的东西"和"我的受众问的东西"之间的区别，并相应地权衡创意。

**4. Claude API 节点。**

这是魔法发生的地方。使用配置为调用 Claude API 的 HTTP Request 节点，或者如果你已经安装了的话，使用专用的 Anthropic 节点。

模型：如果你想要最好的输出质量，使用 claude-opus-4-7；如果你想要速度和更低的成本，使用 claude-sonnet-4-6。对于每周运行，成本差异可以忽略不计。用 Opus。

最大 tokens：8000。你要求 30 个详细的创意。你需要这个空间。

系统提示词和用户提示词结构：在下一节中介绍。

**5. 解析节点。**

Claude 返回一整块文本。你需要将它拆分成 30 个单独的创意条目，这样它们就可以逐行写入数据库。

最简单的方法：在提示词中告诉 Claude 以 JSON 格式返回输出。然后使用 n8n 中的 Code 节点将 JSON 解析为单独的项目。每个项目成为输出中的一行。

如果 JSON 感觉不可靠，另一种方法是要求 Claude 用特定的分隔符如 `---IDEA---` 分隔每个创意，然后使用 Split 节点将响应拆分成块。

**6. 输出节点。**

将解析后的创意发送到你周一早上真正会打开的地方。

我推荐的设置：一个带有钩子、角度、格式、要点、来源和状态列的 Notion 数据库。引擎每周一创建 30 个新行。你整周都在处理它们，将每一个标记为已写、已排期或已跳过。

如果你不用 Notion，将创意格式化为编号列表的 Google Doc 也有效。或者让它通过邮件发给你。选择你真正会打开的地方。

这就是整个搭建。六个功能部分。有经验的 n8n 用户三到四小时就能让它运行起来。第一次接触的用户需要一天。

## 让它生效的提示词

一个糟糕的提示词会产生 30 个版本的"生产力的 5 个技巧"。一个好的提示词会产生 30 个你真正想写的创意。

提示词需要四件事，按顺序：

**声音上下文。**

两到三段描述你如何写作，你不做什么，你的语气听起来是什么样的。要具体。

差的："用随意的、对话式的语气写。"

好的："直接而坦率。一到三句话的短段落。没有企业语言，没有流行语，没有咨询风格的话。偶尔在合适时骂几句。总是使用缩写。没有破折号。写得像在酒吧解释某事的人，而不是在会议上演讲的人。"

声音描述越具体，输出听起来越像你。

**受众上下文。**

谁在读这个。他们想要什么。他们厌倦了看什么。他们在旅程的哪个阶段。

差的："企业家和企业主。"

好的："建立自动化业务的独立创业者和小型机构所有者。他们已经了解 n8n 基础知识。他们想要商业角度、定价逻辑、具体的搭建建议。他们厌倦了'什么是 API'这样的初级内容。假设他们有技术素养。假设他们想赚钱。"

**效果上下文。**

你有史以来最顶级的三到五篇帖子，逐字粘贴到提示词中。如果你有的话附上参与度数字。

这是最大的杠杆。它教 Claude 你的受众实际上响应什么，而不是你认为他们应该响应什么。

你可以手动提取一次并硬编码到提示词中。每隔几个月随着新的顶级表现者出现而更新。

**输出规格。**

精确地告诉 Claude 你想要什么。不只是一个话题。每个创意需要四个字段：

- **钩子**：帖子或文章的开场白
- **角度**：是什么让这个创意与同一话题的普通版本不同
- **格式**：帖子、话题串、文章或视频
- **要点**：读者离开时知道或能做的具体事情

30 个创意，每个创意都有四个字段，以 JSON 格式返回。

当提示词有这四个部分时，输出就不再是千篇一律的了。它开始听起来像你在最好的一天会想到的创意，只是压缩成了 Claude 30 秒的处理时间，而不是你思考的三个小时。

## 如何在不让它变成负担的情况下喂养系统

引擎的好坏取决于输入的数据。如果你的来源是空的，输出就是空的。

诀窍是让输入的维护毫无摩擦。

**笔记**：在整周里把所有东西都录成语音备忘录。有一个单一的 Notion 页面或 Apple Notes 文档，随时把想法倾倒进去。不要整理它们。不要精炼它们。只是捕捉。引擎处理粗糙的材料。

**客户工作**：每次通话结束后，花 60 秒记录客户提出的核心问题以及你如何解决它。数据库中的一行。就这样。

**受众问题**：当有人在私信或评论中问了你一个好问题，把这个问题复制粘贴到你的 Notion 日志中。花五秒钟。回报巨大。

**保存的内容**：使用平台原生的任何保存功能。引擎每周拉取它们一次。

**顶级帖子**：每季度更新一次。拉取你的分析数据，找到前五名，粘贴到提示词中。完成。

整个输入维护例程每天可能需要 10 分钟，分散在整周中。输出是每周 30 个内容创意，不需要你任何创意上的付出。这就是交换。

## 为什么这比其他任何方法都好

你不是在要求 AI 为你有创意。你是在要求它成为你的自动驾驶版本——休息更充分、组织更好——能够访问所有你有时间好好坐下来思考时会使用的输入。

信号是你的。声音是你的。受众知识是你的。Claude 只是在做那个你周一早上 9 点疲惫的大脑无法做好的综合层工作。

这就是为什么输出听起来不像 AI 垃圾。基础不是 AI。而是你真实的生活，通过一个将其组织成内容的系统。

这个方法有效的另一个原因：它消除了内容创作中最糟糕的部分，即冷启动。从零到第一稿是大多数人放弃的地方。每周一早上有 30 个创意在等着，你每次都跳过了最难的那一步。

## 今天就做什么

打开 n8n。创建一个新工作流。

添加一个定时触发器。设置为周一早上 6 点。

先连接一个来源。最容易的那个。可能是你已经在倾倒笔记的 Notion 页面或 Google Doc。用 7 天过滤器让它正确拉取。

添加 Claude API 节点。写一个包含四个部分的提示词粗稿。暂时不要追求完美。只是让某样东西能端到端运行。

手动运行它。看看输出。它会完成 60% 的工作。

添加第二个来源。再次运行。输出会变好。

继续添加来源，直到所有五个都连接好。根据输出中缺少或错误的内容调整提示词。在不同的周里运行三到四次。

三周内，你将拥有一个比你独自想出的创意更好的系统，除了在你度过每一周时捕捉原材料之外，不需要任何持续的努力。

在 n8n 中构建东西的全部意义在于消除你讨厌的工作，这样你就可以做更多你不讨厌的工作。

内容创意是大多数人讨厌的工作。把这个构建一次，它就消失了。
