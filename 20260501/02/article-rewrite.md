# 五十年了，软件第一次活了

**作者：** Ashpreet Bedi ([@ashpreetbedi](https://x.com/ashpreetbedi))  
**日期：** 2026年5月1日  
**来源：** [Dynamic Software](https://x.com/Zephyr_hg/status/2049904901371633815)

五十年了，软件一直是静态的。

你用过的每一个程序，骨子里都是一样的结构：一堆函数，跑在写死的控制流里。If、else、while、for。函数负责干活——读数据库，调 API，转换数据。

进去什么，出来什么，每次都一样。这是五十年的规矩。

然后 2024 年来了。控制流活了。一个新的软件类别就此诞生——活的、动态的、随时响应的软件。

## 软件已死，软件万岁

静态软件像一段录音。你按下播放，听到的就是当初录下的那个版本。同样的音符，同样的顺序，每次都一样。那场演奏发生在某个开发环境里，之后它就永远在重放同一首曲子。

动态软件不一样，它是一场现场演出。

乐谱在，乐器在，乐手在。但今晚这个房间里会发生什么，没有人提前知道——取决于指挥，取决于演奏者，取决于当下这一刻。模型是指挥，工具是乐器，控制流是演奏本身，不是录音。

这就是为什么人们用了一个好的 agent 之后，说不清楚哪里不一样，但就是感觉不同。过去一辈子，他们都在和按钮打交道。现在，他们第一次站在一场现场演出里。软件在回应他们，就在此刻，带着判断，带着在场感，在倾听，在调整。这种"活着"的感觉，是软件从没有过的。

录音是完美的。现场演出不是。乐队会做选择，有时失误，有时出人意料。我们还愿意花钱去音乐厅，就是因为每次的现场都不一样，房间里会发生真实的事。

演奏本身就是意义所在。

动态软件是活的，不确定，不完美。一旦你真的感受过这种不同，再回头看录音，就会明白它一直是什么——凝固的。

我们不是在做更好的录音。我们在构建第一代能够演奏的软件。

## 软件活过来之后，很多假设都站不住了

过去建立在静态软件上的那些假设，软件一旦活过来，一个个都会开始松动。

**确定性没了。** 以前相同的输入一定给出相同的输出。现在不是了。模型会考虑上下文，考虑记忆，考虑之前学到的东西。同一件事，周二下午跑出来的结果，可能和周一上午不一样。文本这块还能多少控制一下，但下一个浪潮是视觉——图表、仪表板、整个页面都按需生成。与其想办法在一个本来就不确定的系统上强加确定性，不如干脆放手，顺着它走。

**状态和时间的逻辑变了。** 静态程序不需要记什么。控制流每次都一样，状态丢在数据库里，CRUD 就够了。动态软件里，状态本身*就是*上下文——过去的对话记录，什么方法有效的历史，对这个领域的积累。数据库不再只是个仓库，它变成了软件运行的地基。

会话的问题随之而来。静态 API 天生是无状态的，每个请求各管各的。动态软件反过来——一个会话是一段连续的上下文，可能跨越几分钟，可能跨越几天，甚至几周。用户回来了，agent 从上次的地方接着走。会话变成了头等大事。

时间也变了。静态软件毫秒级返回，慢点也就几秒。动态软件要推理，要调工具，要等工具返回，然后再推理。一次请求，有时要跑好几分钟。流式传输成了默认，后台执行成了基础能力。原来的 HTTP 请求/响应模型开始吃力，那个 29 秒的负载均衡器超时，也撑不住了。

**软件要学会看住自己。** 静态软件，看代码就知道它干了什么。动态软件，不行。控制流是一个模型，模型是个黑盒。你唯一能知道它做了什么的办法，就是把它做的每一件事都记下来——每一步推理，每一次工具调用，每一次检索。追踪，从一个调试手段，变成了理解软件的唯一途径。

但光记录还不够。静态程序不做决策，没什么需要审批的。动态软件要做决策，决策有后果。有些事可以自己决定，有些要问用户，有些要等管理员点头。软件得把这些说清楚，运行时得真的去执行这些规则。

这些都是实实在在的工程问题。每个认真在做动态软件的团队，都会撞上这些坑。大多数人要花好几个月，从头磨一遍。

## 新的软件类别，需要新的运行时

静态软件有成熟的运行时。写 Django 或者 Express，部署到托管平台，HTTP、会话、扩展、故障恢复，都不用自己操心。基础设施已经铺好了，平台帮你兜着。

动态软件没有这个。你写了一个 agent，然后要在它周围自己搭六个月的基础设施，一个一个手动处理边缘情况。很多坑只有真正大规模跑起来才会遇到——SSE 和 WebSocket 怎么处理，流式传输和后台执行怎么配合，会话如何在重启后恢复，存储怎么做到真的能查，而不是五个供应商凑在一起，审批流程怎么等管理员而不只是用户点确认，怎么做到每个资源、每个工具粒度的权限控制，怎么让 agent 也能在 Slack、Telegram、WhatsApp 上用——因为没人真的想用一个专门的定制 UI。

这就是为什么 80% 的 agent 跑不起来。最后一公里，有太多苦活。

上一次这种量级的转变，是从桌面应用到 Web 应用。那次，Web 软件需要建自己的运行时、自己的协议、自己的基础设施、自己的开发者工具。我们花了二十年才把这些都搭起来。

现在，动态软件来了，还是起点。自己的运行时，自己的协议，自己的基础设施，自己的开发者工具——一切从头开始。

## 未来十年

静态软件用了五十年才成熟。操作系统、数据库、Web 服务器、部署平台、可观测性、身份认证……我们很容易忘了这些东西其实有多新：Heroku 是 2007 年，Kubernetes 是 2014 年，Vercel 是 2015 年。我们今天习以为常的那些基础设施，比大多数用它来做东西的人还年轻。

动态软件，现在是第一年。

谁来构建它的运行时、协议、开发者工具和平台，谁就在定义下一个软件时代的样子。前面的工作很多，也很重。这也是我过去十五年里觉得最有意思的事情。
