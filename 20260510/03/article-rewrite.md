# 到头来，还是网络效应说了算

**作者：** Michael Mignano ([@mignano](https://x.com/mignano))  
**日期：** 2026年5月9日  
**来源：** [It Always Comes Back to Network Effects](https://x.com/Zephyr_hg/status/2053108729504043201)

有一种生意，过去几十年一直很好做：把客户的数据锁进自己的系统。

不是说这些公司居心叵测，只是现实如此。一家企业在某个 CRM 里用了五年，客户信息、销售记录、跟进历史全在里面。要换一套系统？光是把数据搬走这件事，就够让人头疼的了。算一算迁移成本，大多数人最后的选择都是——算了，将就用吧。

这就是所谓的"数据护城河"。用数据的沉没成本，把用户留在原地。

但这条护城河，现在已经守不住了。

AI agent 出现之后，事情变得不一样。agent 可以登进你的系统，把数据一行一行读出来，再写进另一个地方。这个过程确实慢，但 agent 不在乎。它没有下班时间，只要背后的人愿意持续掏钱，它就能一直跑下去。

现在已经有创业公司专门在干这件事：派 agent 去竞争对手的平台上，帮客户把 CRM 数据整个搬走。说实话，这招挺绝的。

而且这不只是 CRM 的问题。聊天机器人里存着的对话记忆、AI 工具里设置好的个性化配置——只要是靠"数据沉没成本"留住用户的地方，迟早都会面对同样的处境。

所以问题来了：什么是 agent 真正搬不走的东西？

答案是：网络效应。从来都是。

agent 能把你 CRM 里的每一行数据复制走，但它没法让你的客户去打开新平台发来的邮件。

agent 能把 Instagram 上的帖子、照片、粉丝关系一条不落地抓走，但它没法让你朋友为了看你发的东西，特意去打开另一个 app。

同样的道理放到大模型上也成立。各家大模型的能力正在趋同，训练数据也可以复制——开源社区已经反复证明了这一点。模型本身，正在变成大路货。

但有一件事没办法复制：正在用你产品的那批用户。他们每一次使用，都在产生真实的反馈，让下一个版本变得更好。这个飞轮，是带不走的。

到头来，还是网络效应说了算。
