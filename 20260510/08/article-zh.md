# 学术研究者的 Claude Code 入门指南

**作者：** Mushtaq Bilal, PhD ([@MushtaqBilalPhD](https://x.com/MushtaqBilalPhD))  
**日期：** 2026年5月7日  
**来源：** [Claude Code 101 for Academic Researchers](https://x.com/Zephyr_hg/status/2052338632426467550)

这是一份五部分的入门教程，专门写给学术研究者。

我尽量用最普通的语言来写，不需要任何技术基础，就能看懂、会用。

## 第一部分：Claude Code 是什么？

Claude Code 是一个装在你电脑上的工具。别被"Code"这个词唬住——你不需要懂任何编程。安装它跟装 Zoom 或 Zotero 没什么两样。

装好之后，你打开自己的项目文件夹（论文、草稿之类的），让 Claude 在里面工作。它能读文件夹里的每一个文件、修改已有文件、创建新文件，而且在你结束对话之后，还能记住你上次做到哪了。

ChatGPT、Gemini 这类网页版 AI，你只能跟它们聊天。Claude Code 不一样，它能真正帮你做事。

### 1.1 你为什么要关注它？

你是一名研究者，电脑里有个文件夹，装着各种 PDF、论文草稿、数据表格、数据集和访谈记录。这些东西都跟你的项目有关，但你需要在文献、数据和笔记之间来回打通——Claude Code 就是为这种情况而生的。

大多数研究者用 AI 的方式都差不多：打开网页，输入问题，看 AI 回答。想让它帮你分析论文？把文件粘进去就行。

Claude Code 的不同在于：不是你把文件搬去给 AI，而是把 AI 请进你的文件夹。

![](images/img-01.jpg)

### 1.2 它对学术工作有什么用？

Claude Code 能成为你得力的研究助手，主要有两个原因。

第一，它记得你。记得你的研究方向、写作风格，以及随时在变的研究需要。你不用每次都从头解释一遍。

第二，它在你的文件夹里工作，可以同时处理多个文件。比如，文件夹里有四十五篇 PDF？它能一次把所有论文的研究目标、研究方法都提取出来，还能把结果整理成一个新文件。

Word、Excel、PDF，各种格式它都能处理。

![](images/img-02.jpg)

### 1.3 具体能怎么用？

如果你做的是质性研究，把访谈记录全部放进文件夹，让 Claude Code 找出每个受访者谈到某个话题时说了什么。然后让它归纳所有访谈里反复出现的主题。

如果你做的是量化研究，把那份乱糟糟的 CSV 或 Excel 丢进去，让它帮你清洗数据、跑描述性统计，或者让它帮你理解审稿人的某条批评意见。

![](images/img-03.jpg)

### 1.4 Claude Code 做不到什么？

Claude Code 很强大，但它代替不了你的专业判断。起草、总结、整理，这些它都行；但什么算论点、什么算证据，那是你的事。

能把 Claude Code 用好的研究者，是把它当助手的人——而不是把脑子整个交出去的人。

![](images/img-04.jpg)

## 第二部分：安装 Claude Code，第一次上手

如果你是新用户，留出 15 到 20 分钟来安装和配置。不需要任何编程基础。

使用 Claude Code 需要 Claude Pro 或 Max 订阅。

### 2.1 安装

Claude Code 支持 Windows 和 Mac。去 claude.com/download 下载对应版本，安装方式跟普通软件一样（Zoom、Zotero 那种）。Windows 装好后在开始菜单里找，Mac 在应用程序文件夹里。

打开应用，第一次会让你用浏览器登录。登录之后，左边是对话历史列，中间是聊天窗口——跟网页版 Claude 或 ChatGPT 的布局差不多。

![](images/img-05.jpg)

### 2.2 打开你的第一个文件夹

点"打开文件夹"，一般在聊天框上方附近（每次更新位置可能略有变动，但大概就在那一带）。

找到你存了 PDF 或草稿的文件夹，打开它。如果你想更谨慎一点，也可以新建一个专门的"Claude 研究助手"文件夹，把材料都放进去再打开。

这样 Claude Code 就进了那个文件夹，里面所有文件和子文件夹它都能访问。

![](images/img-06.jpg)

### 2.3 一次对话是什么样的？

所谓"会话"，就是你跟 Claude 在聊天框里的对话，和用网页版 ChatGPT 基本一样。区别只有一个：现在 Claude Code 能访问你电脑里的文件，也能在你电脑上创建新文件。

进了文件夹之后，给 Claude 一个简单的指令，比如：

> 读取这个文件夹里的所有论文，把它们的核心论点整理成一个单独的文件。

就这样。你跟 Claude Code 打交道，大多数时候就是这么写指令。

Claude Code 会去读这些论文，整理出相关内容，创建一个新文件，保存在你的文件夹里。

在执行过程中，它会请求你的确认。你可以关掉这个确认步骤，但刚开始用的时候，建议保留——你不会想让 Claude 不打招呼就删了什么重要文件。

每次对话都会自动保存在左侧面板里。

![](images/img-07.jpg)

## 第三部分：把 Claude Code 配置成你的研究助手

每次 Claude Code 启动，它都会先找一个叫 **CLAUDE.md** 的文件，读取里面的指令。注意是全大写，这一点别搞错。

创建这个文件有两种方式：手动或自动。

### 3.1 手动创建

在 Windows 上打开记事本，Mac 上打开 TextEdit，用普通语言把你希望 Claude Code 遵守的规则写下来。不需要任何编程语法，普通文字就行。

CLAUDE.md 可以分成几个部分：

- **#Role（角色）** — 告诉它你希望它扮演什么角色，简单介绍你的研究领域和当前项目。
- **#Standards（标准）** — 说明你所在领域的学术规范。比如引用格式、论文结构要求等。
- **#Writing Style（写作风格）** — 告诉它你希望它用什么风格回应你，是学术腔还是更随意的语气。
- **#Critique Style（反馈风格）** — 说明你希望它怎么评价你的工作，是重点审查论点、证据，还是研究方法。

写完之后保存为 CLAUDE.md，放进你的 Claude 主文件夹，就完成了。

不用追求一次写完美，随着项目推进，你可以随时回来改。

![](images/img-08.jpg)

### 3.2 自动创建

不想手动写？直接在 Claude Code 里开一个对话，把你的研究背景、标准、风格偏好告诉它，然后让它自己生成 CLAUDE.md 文件。

它会创建好并保存到你的 Claude 文件夹里。

![](images/img-09.jpg)

### 3.3 自动记忆

在你做项目的过程中，Claude Code 会把跟你工作相关的要点记下来，存在它自己的文件夹里，供自己使用。这些笔记你看不到，也不用管它们。

每次新开一个会话，Claude Code 都会先读 CLAUDE.md 和这些笔记，再来回答你的问题。

时间久了，你给的指令加上它自己积累的笔记，它对你和你的研究就有了足够的了解，真正成为一个可靠的助手。

你也可以问问它记了什么，直接说："告诉我你记忆里存了什么。"如果某条信息过时了（比如引用格式变了），就让它更新一下。

![](images/img-10.jpg)

### 3.4 什么不该放进 CLAUDE.md？

不要把任何机密信息或你不想让 AI 使用的内容放进去。

也要定期检查，别让 CLAUDE.md 里全是过时的内容——那会直接影响它的输出质量。建议每隔几周更新一次。

![](images/img-11.jpg)

## 第四部分：处理你的研究文档

放进 Claude 文件夹的文件，不用整理得井井有条。这不是说你可以塞一堆不相关的东西进去——而是说，不用专门花时间去分类，Claude 自己能搞定。唯一要注意的是，别给文件起让人摸不着头脑的名字，比如"论文（终稿）（终稿2）（用这个）"。

把相关的 PDF、Word 文件、数据集、访谈记录统统丢进去。

假设你有二十篇跟研究主题相关的论文，想找它们共同讨论的议题；或者你有个论点，想看看哪些文章在支持它、哪些在反对它。

在 Claude Code 里打开这个文件夹，输入：

> 读取这个文件夹里的每一个 PDF，告诉我哪些文章不同意以下论点：[把论点粘贴在这里]

Claude 会读完所有 PDF，把相关内容整理出来，可能会以表格的形式呈现。

### 4.1 用来做文献综述

如果你是生物医学研究者或社会科学家，经常要做系统综述，可以下载五十篇论文，放进一个叫"Systematic Review with Claude"的文件夹，在 Claude Code 里打开，把你的纳入/排除标准告诉它，让它帮你筛选所有文章。

它会按你的标准筛完，结果以表格呈现。

![](images/img-12.jpg)

### 4.2 处理访谈记录

如果你做质性研究，可以把访谈记录放进 Claude 文件夹，让它提取跟某个主题相关的内容。

比如，让它整理每位受访者对某个特定问题的回答。

![](images/img-13.jpg)

### 4.3 让 Claude 干那些烦人的活

把五十来个 PDF 放进文件夹，在 Claude Code 里打开，让它把所有文件按论文标题重新命名。

几分钟，搞定。

![](images/img-14.jpg)

### 4.4 让 Claude Code 创建文件

项目会不断推进，你会需要一些可以随时查阅的整理文件，方便日后回头看，也方便 Claude Code 之后快速找到。

每次让 Claude Code 做比较重要的任务——比如筛选论文、提取访谈信息——都记得让它把结果保存成文件存在文件夹里。

它默认会存成 Markdown（.md）格式，文件小，它自己检索也快。如果你更习惯，也可以让它存成 Word 或 Excel。

![](images/img-15.jpg)

## 第五部分：Claude Code 的"技能"

"技能"是一套让 Claude Code 专门处理某类任务的指令文件。跟 CLAUDE.md 一样，技能也是用普通语言写成的 Markdown 文件，也可以手动或自动创建。

创建好技能之后，Claude Code 会记住在什么情况下使用它，你自己不用操心。当然，你也可以随时用斜杠命令手动调出来。

最省事的方法是让 Claude Code 帮你创建。举个例子：你经常开 Zoom 会议，有会议记录，每次都想从里面提取待办事项。

开一个 Claude Code 会话，告诉它你想要一个"从 Zoom 会议记录里提取待办事项"的技能。

它会问你几个细节问题，然后生成一个技能文件。如果你想让它在输出时遵循特定格式，可以在文件里补充。

技能创建好之后，重启 Claude Code，就可以用了。

### 5.1 CLAUDE.md 和技能有什么区别？

CLAUDE.md 是关于你和你项目的全局说明，给 Claude Code 提供整体背景。

技能则是针对具体任务的，更细、更专。CLAUDE.md、技能和自动记忆三者配合，才能让 Claude Code 给出最好的回应。

![](images/img-16.jpg)

## 不要交给 Claude Code 的事

Claude Code 最适合那些费时、费力、重复性高的活——把这些外包给它。但真正算得上学术成果的东西，它做不到，因为它给不了你真正原创的论点。

它能帮你整合信息、梳理材料，让这些内容为你的论点服务——但提出那个论点本身，还是你的事。
