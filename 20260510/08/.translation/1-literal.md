# 学术研究者的 Claude Code 101

**作者：** Mushtaq Bilal, PhD ([@MushtaqBilalPhD](https://x.com/MushtaqBilalPhD))  
**日期：** 2026年5月7日  
**来源：** [Claude Code 101 for Academic Researchers](https://x.com/Zephyr_hg/status/2052338632426467550)

这是一个五部分的教程，旨在帮助学术研究者开始使用 Claude Code。

我用通俗易懂的语言写了这篇教程，你不需要任何技术背景来理解这个教程或使用 Claude Code。

## 第一部分：什么是 Claude Code？

Claude Code 是一个运行在你电脑上的工具。不要被"code"这个词吓到。你不需要任何编程或程序设计技能来使用它。你可以像安装 Zoom 或 Zotero 这样的应用程序一样，把 Claude Code 安装到你的电脑上。

一旦你的电脑上有了它，你打开你的项目文件夹（论文、文章等），让 Claude 在文件夹内工作。Claude 可以读取文件夹中的每一个文件，编辑现有文件，并创建新文件。它还可以在你结束一个会话后记住你正在做什么。

不像 ChatGPT 和 Gemini 这样的基于浏览器的应用程序只是"对话"，Claude Code 实际上可以为你"做"事情。

### 1.1 为什么你应该关注它？

你是一名研究者，电脑上有一个文件夹，里面有几个 PDF 文件、你的论文草稿、电子表格、数据集和一些访谈记录。所有这些文件都与你的项目相关，但你需要在已发表的文献、收集的数据和你自己的笔记之间建立联系。

这正是 Claude Code 所设计的使用场景。

大多数学者和研究者已经在浏览器中使用过 AI 应用程序（ChatGPT、Claude、Gemini 等）。你打开一个标签页，输入一个提示，AI 就会回复。如果你想问关于你的论文或草稿的问题，你就把它们添加到 ChatGPT 或 Claude 中。

Claude Code 的不同之处在于，你不再是将文件带到浏览器中的 AI 应用程序，而是将 AI 带到包含所有数据的文件夹中。

![](images/img-01.jpg)

### 1.2 为什么 Claude Code 对学术工作很重要

Claude Code 可以作为你可靠而强大的研究助手，有两种基本方式。

首先，Claude Code 记住你、你的研究、你的写作风格以及你不断演变的研究需求。你不必反复解释你的要求。

其次，由于 Claude Code 在你电脑上的文件夹内工作，它可以同时处理多个文件。例如，它可以浏览文件夹中所有四十五个 PDF，并从所有文件中提取相关信息（目标、方法等）。你甚至可以要求它根据提取的信息创建一个新文件。

Claude Code 可以处理所有类型的文件，从 Word 文件到 Excel 表格、PDF 等。

![](images/img-02.jpg)

### 1.3 这在实际中意味着什么？

如果你是一名定性研究者，加载一个装满访谈记录的文件夹，让 Claude Code 给你每一个特定话语的实例，例如每个参与者如何谈论某个特定主题。然后让它寻找跨访谈的整体主题。

如果你是一名定量研究者，在你的文件夹中添加一个凌乱的 CSV 文件或 Excel 表格，让 Claude Code 帮你整理。你可以让它运行描述性统计，或者让它解释审稿人提出的关键意见。

![](images/img-03.jpg)

### 1.4 Claude Code 不是什么

Claude Code 是一个非常强大的工具，但它不能替代你的专业判断。它可以起草、总结、编码，但什么算作论点或证据是你的责任。

可能成功使用 Claude Code 的学者是那些将其视为研究助手的人，而不是那些将所有思考和判断都外包给它的人。

![](images/img-04.jpg)

## 第二部分：安装 Claude Code 应用程序，你的第一个会话

如果你是 Claude Code 的新用户，请留出 15-20 分钟来安装和设置它。你不需要任何编程技能来使用 Claude Code。

你需要 Claude Pro 或 Max 订阅才能在电脑上使用 Claude Code。

### 2.1 安装

Claude Code 适用于 Windows 和 Mac。前往 claude.com/download 并下载与你电脑兼容的版本。按照你安装其他软件（如 Zoom、Zotero 等）的方式运行安装程序。在 Windows 上，它会出现在你的开始菜单中。在 Mac 上，它会出现在你的应用程序文件夹中。

安装后打开应用程序。第一次打开时，Claude 会要求你通过浏览器登录。登录后，你会看到左侧有一列，中间有一个聊天面板。左侧列将显示你所有对话的记录，就像在浏览器中使用 Claude 或 ChatGPT 一样。

![](images/img-05.jpg)

### 2.2 打开你的第一个文件夹

点击"打开文件夹"。它通常位于主面板聊天栏的顶部。每次更新后它的位置可能会稍微变化，但你大多数时候会在聊天栏附近找到它。

导航到你电脑上存储了 PDF 或草稿的任何文件夹。如果你想特别谨慎，可以创建一个专门的"Claude 研究助手文件夹"并导航到那里。

现在 Claude Code 就在那个文件夹里了。它可以访问主文件夹内的每个文件和子文件夹。

![](images/img-06.jpg)

### 2.3 一个会话是什么样子的

会话是你在主聊天面板中与 Claude 进行的对话，就像你在 ChatGPT 等浏览器应用程序中做的那样。

交互方式类似于 ChatGPT 等基于浏览器的应用程序。唯一的区别是，现在 Claude Code 可以访问你电脑上的文件。它还有在你电脑上创建文件的能力。

一旦你进入文件夹，给 Claude 一个简单的提示，如：

> 读取此文件夹中的所有论文，并将它们的主要论点作为单独的文件给我。

就是这样。你与 Claude Code 的大多数交互都是像这样写指令。

Claude Code 将继续从论文中检索相关信息，并开始创建一个新文件。这个文件将出现在你正在工作的文件夹中。

在继续的过程中，它会要求你授予权限。你可以让 Claude Code 绕过权限，但在开始时，让 Claude Code 请求权限是个好主意。你不想让 Claude Code 在没有询问你的情况下删除任何重要文件。

你在 Claude Code 中开始的每个会话都会自动保存在左侧面板中。

![](images/img-07.jpg)

## 第三部分：Claude Code 作为你的研究助手

每次 Claude Code 启动时，它会立即查找一个名为 **CLAUDE.md** 的文件，以获取要遵循的指令集。注意大写字母，这很重要。

有两种方法可以创建 CLAUDE.md 文件：自动和手动。

### 3.1 手动创建

如果你想手动创建，在 Windows 上打开记事本，或在 Mac 上打开 TextEdit，写下你希望 Claude Code 遵循的指令类型。请注意，你只需要用普通语言写这些指令。你不需要使用任何编程语言。

你可以将 CLAUDE.md 文件分成以下几个部分：

- **#角色** — 描述你希望 Claude Code 作为你的研究助手承担的角色类型。给它一些关于你研究领域和当前项目的细节。
- **#标准** — 描述与你研究领域相关的学术标准。例如，你可以告诉 Claude Code 遵循给定的引用样式或研究论文的特定结构。
- **#写作风格** — 告诉 Claude Code 你希望它遵循的写作风格类型，无论你是希望它以学术风格还是非正式风格回复。
- **#批评风格** — 描述你希望 Claude Code 如何批评你的工作，无论你是否希望专注于你的论点、证据或方法论。

将文件保存为 CLAUDE.md 在你电脑上的主 Claude 文件夹中，就完成了。

你不需要有一个完美精心制作的 CLAUDE.md 文件。你可以随时返回并随着项目的发展进行编辑。

![](images/img-08.jpg)

### 3.2 自动创建

如果你不想手动创建 CLAUDE.md 文件，只需在 Claude Code 中启动一个聊天会话，提供关于其角色、标准、写作风格和批评风格的详细信息。然后要求它使用这些信息创建一个 CLAUDE.md 文件。

Claude Code 将创建文件并将其保存到你的 Claude 文件夹中。

![](images/img-09.jpg)

### 3.3 自动记忆

在你进行项目工作的过程中，Claude Code 会写下关于你工作的简短笔记，并将其保存供自己使用。你不会看到它们，也不需要太担心它们。Claude Code 将这些笔记放在一个文件夹中，并自行管理。

每次你开始一个会话，Claude Code 都会读取 CLAUDE.md 和它自己的笔记，并使用它们来回答你的问题。

随着时间的推移，你的指令和 Claude Code 的笔记给了它足够的关于你和你工作的背景，使其成为一个可靠的研究助手。

你可以通过输入"告诉我你在记忆中存储了什么"来询问 Claude Code 关于其记忆的情况。如果某条信息已经过时（例如引用样式），只需要求它用新信息更新其记忆。

![](images/img-10.jpg)

### 3.4 不要放在 CLAUDE.md 中的内容

不要在 CLAUDE.md 中放置任何机密信息或任何你不想让 AI 使用的信息。

不要让 CLAUDE.md 成为一套过时的信息，因为这将对其输出产生相当大的影响。你可能想每隔几周更新一次该文件。

![](images/img-11.jpg)

## 第四部分：处理你的研究文档

Claude 文件夹不必组织得非常整齐。这并不意味着你可以开始在这个文件夹中放置无关的文件。这意味着你不应该花时间去组织它。Claude 自己可以做到这一点。你可能想记住的唯一一件事是不要给你的文件起令人困惑的名字，如"论文（最终版）（最终版2）（使用这个）"。

添加一堆与你项目相关的 PDF、Word 文件、数据集、访谈记录等。

假设你有二十篇与你研究主题相关的研究论文，你想找到贯穿所有论文的共同主题。或者也许你有某个主张或论点，你正在寻找提供支持或与该主张相对立证据的论文。

在 Claude Code 中打开文件夹，写下如下提示：

> 读取此文件夹中的每个 PDF，告诉我哪些文章不同意以下论点：[在此处粘贴论点]

Claude 将读取文件夹中的所有 PDF，并从这些文章中给你提供相关信息。Claude 的回答可能以表格的形式呈现。

### 4.1 文献综述的研究助手

如果你是生物医学研究者或经常进行系统综述的社会科学家，下载五十篇研究文章，并将它们放在一个名为"与 Claude 进行系统综述"的文件夹中。

在 Claude Code 中打开文件夹，给出你的筛选（纳入/排除）标准，并要求它筛选文件夹中的所有论文。

Claude Code 将根据你的标准筛选论文，并以表格形式给出结果。

![](images/img-12.jpg)

### 4.2 处理记录

如果你是定性研究者，你可以在 Claude 文件夹中添加访谈记录，并要求它提取与给定主题相关的信息。

例如，你可以要求它提取每个受访者如何回答给定问题的内容。

![](images/img-13.jpg)

### 4.3 给 Claude 一个繁琐的任务

在文件夹中放入约五十个 PDF，并在 Claude Code 中打开它。然后要求它浏览所有文件，并使用其标题重命名它们。

你会看到 Claude Code 在几分钟内完成了所需的工作。

![](images/img-14.jpg)

### 4.4 要求 Claude Code 创建文件

由于你的项目将不断发展，你会希望 Claude Code 也为你创建某些文件，以便在需要时回头参考。随着项目的发展，它也将帮助 Claude Code 轻松检索。

每当你要求 Claude Code 执行重要任务时，如筛选论文或从记录中提取信息，都要求它将你的回答作为文件保存在文件夹中。

一般来说，Claude Code 会将这些文件保存为 markdown（md）格式，这种格式占用空间很少，Claude Code 从中检索信息也非常容易。但你也可以要求它创建 Word 文件或 Excel 表格。

![](images/img-15.jpg)

## 第五部分：Claude Code 的"技能"

技能是一组使 Claude Code 成为某一任务专家的指令。就像 CLAUDE.md 文件一样，技能也是一个用普通英语写成的 markdown 文件。就像 CLAUDE.md 一样，你可以手动和自动创建技能文件。

一旦你创建了一个技能，Claude Code 就会记住何时使用它。你自己不必记住。但如果你想要，你可以随时通过使用正斜杠命令来调用技能。

创建技能最简单的方法是让 Claude Code 为你创建。假设你有定期的 Zoom 通话，你有这些通话的记录。对于每次通话或记录，你想提取可操作的项目或待办事项。

要创建技能，启动一个 Claude Code 会话，简单地要求它创建一个用于从 Zoom 记录中提取可操作项目的技能。

Claude Code 将开始工作。它会问你后续问题，并创建一个你可以编辑的技能文件。也许你想让 Claude Code 在回复时遵循特定的结构。

技能创建后，重启 Claude Code，技能就可以使用了。

### 5.1 CLAUDE.md 和技能之间的区别

CLAUDE.md 包含关于你和你项目的全局指令。它给 Claude Code 提供了关于你和你研究的大局观。

另一方面，技能是为特定任务设计的。这些文件包含更具体和细粒度的信息。CLAUDE.md、技能和自动记忆协同工作，为你提供最佳响应。

![](images/img-16.jpg)

## 不要委托给 Claude Code 的事项

Claude Code 擅长劳动密集型、耗时和重复性的任务。将这些任务外包给 Claude Code。但 Claude Code 无法创造真正算作学术成果的东西，因为它无法给你提供新的和原创的论点。

它可以综合你可以用来服务于你的论点的信息，但提出原创论点的工作仍然是你作为研究者的职责。
