# 为什么我们应该停止为AI智能体设计套具

**作者：** Kangwook Lee ([@Kangwook_Lee](https://x.com/Kangwook_Lee))  
**日期：** 2026年5月9日  
**来源：** [Why We Should Stop Designing Harnesses for AI Agents](https://x.com/Zephyr_hg/status/2052925157606568217)

*（对于不熟悉"马与车架类比"的读者，请先阅读我最近的文章）*

在这篇文章中，我想解释为什么我们所有人都应该停止手工设计AI智能体的套具。

**摘要：** AI智能体的最佳套具可能不再是人类为它设计的那个，而是智能体自己工程化的那个。如果我们继续为AI智能体设计套具，我们将成为瓶颈。

## 套具创造一个阈值

让我从提出一种关于套具与模型关系的新思考方式开始。

今天大多数人使用的观点是"固定模型，改变套具"。

我想提出的观点是"固定套具，改变模型能力"。

那么，对于一个固定的任务，套具在模型的原始智能上创造了一个单一的阈值。

如果模型比阈值更聪明，它就能用这个套具解决任务。如果模型低于阈值，它就不能。

我们把这个阈值称为θ(h)，相对于套具h来定义。

## 三区域框架

当谈到比较两个套具时，这个观点变得非常有用。

一个是好套具（h_good）。一个是坏套具（h_bad）。

好套具使任务更容易，所以它创造了一个更低的阈值。坏套具要求更聪明的模型来完成同样的任务，所以它创造了一个更高的阈值。

这自然地产生了三个区域。

![](images/img-02.jpg)

**第一区域。** 处于这个区域的模型无法用任何套具解决任务。

**第二区域。** 处于这个区域的模型具有中等智能，它们能够解决任务，但只有在使用好套具的情况下！

**第三区域。** 处于这个区域的模型足够聪明，无论使用什么套具都能解决任务。

注意，套具质量在中间区域最重要，当模型具有中等智能但不是非常聪明的时候。那时好套具才能发挥作用。

我们把这个称为三区域框架。

## 人类制造的套具 vs AI制造的套具

让我们回到马与车架的类比。

马和AI之间有一个关键的区别。

马不能建造自己的套具。AI可以。

现在，考虑两种套具。

- **h_human：** 由人类制造的套具。
- **h_ai：** 由AI制造的套具。

在过去几年里，情况是这样的：

```
h_good = h_human
h_bad  = h_ai
```

人类制造的套具比AI制造的套具更好。

所以在三区域框架中，中间区域意味着："AI模型只能在人类制造的套具下工作。"

我们精心手工工程化的智能体循环、工具使用模式、记忆结构、规划脚手架，表现优于AI自己产生的东西。构建一个好的智能体意味着构建一个好的套具。

个人而言，我在这个时代花了大量时间为AI智能体设计套具。我设计了最先进的聊天机器人套具[ACL'23]、基于VLM的图像聚类智能体[ICLR'24]、PUBG的游戏伴侣智能体Ally，以及用于代码智能体的Terminus-KIRA。

那段时间非常有趣。持续了一段时间。

## 但前提正在改变

AI在编码方面变得更好。在调试方面更好。在系统迭代方面更快。最终，在构建套具方面比我们更好。

这翻转了情况：

```
h_good = h_ai
h_bad  = h_human
```

这意味着什么？根据三区域框架，中间区域现在意味着："AI模型只能在AI制造的套具下工作。"

换句话说，如果我们继续使用"人类制造的套具"，我们将无法看到能力处于中间区域的模型的全部潜力。

事实上，我们最近的工作，Meta Harness（由@yoonholeee主导），已经证明了对于中等规模的模型，AI在迭代套具工程方面优于人类。

"黄色区域"的出现为"被误管理的天才"假说提供了另一种可能的解释。如果我们继续使用手工设计的套具，我们将看不到底层AI模型的全部潜力。

## 结论

当一个模型极其强大时，套具质量几乎无关紧要。简单的智能体循环和正确的工具足以处理几乎任何事情。

当一个模型太弱时，套具质量也几乎无关紧要。没有套具能拯救它。

问题在于中间。当一个模型具有适度的智能时。原则上有能力，但在没有适当支持的情况下失败。那就是套具质量起决定性作用的地方。

直到最近，那个好套具必须来自人类。

不再是了。为了充分利用中间区域的模型，我们需要退出套具工程。

让聪明的马自己建造它们的套具。
