# 直译稿问题梳理

## 风格问题

1. **"每次发布（per-rollout）"** — 保留英文括号注释，读起来有技术文档腔。"rollout" 在 Codex 语境里指"一次会话运行"，直接译成"每次会话"更自然。

2. **"Codex 认领一个启动任务"** — "认领启动任务"是直译编程术语，读者感觉生硬。可以改成"Codex 会启动一个后台任务"或"触发一次后台处理"。

3. **"该子智能体读取候选记忆，决定哪些要合并、哪些要修补、哪些要丢弃"** — 偏机械，可以稍微自然一点："子智能体读取这些候选记忆，判断哪些保留、哪些合并、哪些丢弃"。

4. **"这是设计上的选择：纯文本加子字符串匹配"** — 冒号后面跟短语，语气有点生硬。可以调整为："这是有意为之的设计——纯文本、子字符串匹配"。

5. **"速率限制感知"** — 这个词在中文里完全没有直觉。可以改成"感知速率限制"或直接描述功能："配额感知"。

6. **"老化"** — 这里翻译的是 Aging（配置项含义），但单独用"老化"作标题略显奇怪，可以改成"过期清理"或"记忆老化"。

7. **"生产场景中触及这些限制"** — 这个小节标题下方缺少实质正文内容（原文有图片，正文内容非常简短），直译没有问题，但可以稍微调整句式："下面几个真实团队场景能具体说明这些限制在哪里会踢进来。"

8. **"底层发生了什么变化"** — 原文是 "What changes underneath"，作为引导句偏直译。可以改成"接入 Mem0 之后，底层有几件事变了"或"这几点会有本质不同"。

9. **"使用 Mem0，问"我们的部署命令是什么？"能找到"生产部署通过 `make ship-prod` 进行""** — 这句话结构复杂，读起来要停顿一下才能理解。可以拆开：换 Mem0 之后，就算你问"我们的部署命令是什么？"，它也能找到"生产部署走 `make ship-prod`"——即使这两组词根本不重叠。

10. **"Codex CLI 的记忆层在今日编程智能体领域设计精良"** — 原文是 "carefully designed"，翻成"设计精良"丢了"仔细、用心"的含义，有点像营销话术。可以改成"Codex CLI 的记忆层设计是认真的"或保留原意"Codex CLI 的记忆层在今天的编程智能体里算是设计用心的"。

## 语气问题

11. "这就是整个记忆层" — 原文是 "That's the whole memory layer"，直译还行，但可以更有力："以上就是整个记忆层，仅此而已。"

12. 整篇翻译较为规整，但原文作者的语气是相对直接、带一点克制讽刺的技术人风格（尤其在"Where it stops"部分），中文版稍微偏向了正式技术文档风格，这部分可以在意译阶段调整回来。
