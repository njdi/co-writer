## 1：什么是 NotebookLM？

NotebookLM 最准确的理解是：一个以来源为锚点的工作空间。但但但……这是什么意思？

你上传可信的材料，NotebookLM 帮你提取、整理、比较、综合，并将这些材料转化为结构化的输出。那具体能用来做什么？

- 从上传的资料中提取知识
- 构建研究矩阵和数据表
- 把一批资料转化为课程、指南或 SOP
- 把密集材料转化为学习系统
- 生成 Studio 资产：音频概述、视频概述、报告、数据表、抽认卡、测验、幻灯片、信息图、思维导图
- 为外部 AI 助手制作可复用的知识文件
- 把客户、业务或研究材料转化为精致的输出
- 把杂乱信息整理成可复用、可教授、可分享或可导出的结构化内容

NotebookLM 很强大。它以来源为锚点，支持引用核查，这种特性大大降低了幻觉风险，让输出质量更高。

这样理解它：

NotebookLM 把可信的素材转化为结构化的知识资产。把它当成一个分层系统来用：

- **输入层：** 上传可信资料
- **推理层：** 用聊天功能提取、比较和综合
- **捕获层：** 把有用的输出保存为笔记
- **复利层：** 把有价值的笔记转换回来源
- **资产层：** 生成报告、表格、抽认卡、测验、幻灯片、音频、视频、信息图和思维导图
- **部署层：** 导出到 Docs、Sheets、PDF、PPTX、Markdown、Notion、Obsidian 或其他系统
- **扩展层：** 当项目超出 NotebookLM 原生边界时，接入 Gemini、Claude、自定义 GPT、Zotero、Obsidian 或 Notion

这篇文章适合：

- 想为学习、研究、写作、内容创作和运营建立可重复工作流的中级用户

## 2：它是怎么工作的？

在这篇文章变得极其有用之前，我们得先过一段可能让你想跳过的内容——让你对它的工作原理有个底。如果你已经在打哈欠，可以直接往下滚看干货；但你对一个工具了解得越透，你从它身上得到的就越多。

NotebookLM 有六个工作层。

**第 1 层：来源**

来源是输入层。

这些是你上传的 PDF、文档、网页、视频、音频文件、电子表格、图片或粘贴的文本。

它们划定了这个笔记本所知道的知识边界。

资料烂，输出就烂。
资料好，输出才有用。
精挑细选的资料，永远强过乱塞一堆。

**第 2 层：聊天**

聊天是推理和综合层。

在这里你提问、提取信息、比较资料、测试想法、生成结构化答案。

聊天可以用来：

- 来源清单
- 信息提取
- 内容综合
- 矛盾映射
- 主张核查
- 大纲
- 提示词
- 清单
- 草稿报告
- 核查验证

不要一上来就说"帮我总结一下"。从清单、提取、结构开始。

**第 3 层：笔记**

笔记是捕获层。

它们把有价值的输出保存下来，不让它消失在漫长的聊天记录里。

笔记可以用来存：

- 已保存的大纲
- 提取出的框架
- 来源摘要
- 模块计划
- 经过核查的主张
- 最终草稿
- 项目决策

一条好的笔记不只是保存了一个答案。

它是可以反复调用的项目记忆。

**第 4 层：把笔记转换为来源**

这是复利层。

也是 NotebookLM 最强大的工作流之一。

你可以把一个答案保存为笔记，然后把这条笔记转换成来源。这让你能够分阶段搭建知识资产。

举个例子：

1. 提取所有框架。
2. 把提取结果保存为笔记。
3. 把笔记转换为来源。
4. 要求 NotebookLM 用这个新的框架来源作为结构，来构建一门课程。

这改变了整个玩法。

NotebookLM 不再是一次性问答工具，而是成了分阶段的知识构建系统。

**第 5 层：Studio**

Studio 是资产生成层。

它把以来源为锚点的材料转化为各种输出：

- 报告
- 数据表
- 学习指南
- 抽认卡
- 测验
- 思维导图
- 音频概述
- 视频概述
- 幻灯片
- 信息图

Studio 是 NotebookLM 从阅读助手变成生产工具的地方。

但不要让 Studio 来主导你的策略。

先提取。
再综合。
最后才生成资产。

**第 6 层：导出与部署**

资产做好之后，把它送到该去的地方。

- Google Docs：报告和指南
- Google Sheets：表格和矩阵
- PPTX 或 PDF：幻灯片
- Markdown：用于 Obsidian、Notion、Claude Projects 或自定义 GPT
- MP3：音频概述
- 公开笔记本链接：分享给客户或受众
- html

NotebookLM 很擅长综合。

但它未必是你存储这些信息的最好地方……个人维基会更合适！

### 用例 1：从上传的资料构建完整知识库

**用途**

把大量上传的资料转化为可复用的知识库，最终可以变成指南、课程、教程、操作手册、剧本、系列文章、幻灯片或 AI 助手文件。

**最适合**

研究人员、创作者、顾问、企业、教育者、专家，以及任何想认真构建长期知识资产的人。

**使用的来源类型**

PDF、Google Docs、网页、YouTube 字幕、音频字幕、ePub、Markdown 文件、Google Slides、Google Sheets、CSV 文件和粘贴的笔记。

**前期设置**

- 删掉弱的、过时的或不相关的资料。
- 给文件取清晰的名字。
- 很长的文档按主题拆分成块。
- 如果主题有专业术语，先做一份词汇表文档。
- 把词汇表作为第 1 号来源上传。
- 把笔记本角色设定为"专家知识架构师"。
- 分阶段处理资料，不要一次要求一个巨大的输出。

**工作流**

1. 跑一遍来源清单。
2. 逐来源提取知识。
3. 创建主题映射。
4. 创建概念映射。
5. 创建框架映射。
6. 创建流程映射。
7. 识别矛盾和空白。
8. 构建模块化知识库。
9. 把每个模块保存为笔记。
10. 把有价值的笔记转换为来源。
11. 用这些笔记来源构建最终的指南、课程或剧本。
12. 审核来源覆盖情况，以及是否有无来源支撑的主张。

**直接可用的提示词：**

```
Act as an Expert Knowledge Architect.

Create a complete knowledge base from the selected sources.

Work in this order:

1. Create a source inventory.
2. Extract key ideas, definitions, frameworks, processes, examples, warnings, tools and gaps.
3. Create a master theme map.
4. Create a concept map.
5. Create a framework map.
6. Create a process map.
7. Identify contradictions and missing information.
8. Build a modular knowledge base.
9. Add checklists, templates, prompts and practical exercises.
10. Finish with a source coverage and claim audit.

Write in British English.
Be comprehensive.
Do not invent unsupported examples.
Mark gaps clearly.
```

**预期输出**

一个包含模块、定义、框架、流程、示例、清单、模板、提示词和最终审核的结构化知识库。

**质量把关**

- 每个来源是否都出现在清单里了？
- 输出是否过度依赖某一个来源？
- 完全相同的内容是否已合并？
- 不同但相关的想法是否得到保留？
- 没有来源支撑的推断是否已标注？
- 空白是否已明确说明？

**常见错误**

- 直接要求"帮我总结一切"
- 跳过清单阶段
- 没有把有用的输出保存为笔记
- 让 NotebookLM 把内容压缩得太狠
- 把第一个答案当最终答案

**进阶版本**

用分阶段的笔记转来源工作流：

1. 把来源清单保存为笔记。
2. 把框架映射保存为笔记。
3. 把模块大纲保存为笔记。
4. 把最有价值的笔记转换为来源。
5. 要求 NotebookLM 以笔记来源为结构、以原始来源为证据，构建最终知识库。

**最终产出**

一个可复用的知识库。

### 用例 2：把资料转化为一门课程

**用途**

把上传的材料转化为包含模块、课程、练习、评估和最终项目的结构化课程。

**最适合**

教育者、创作者、顾问、培训师、教练、企业和专家。

**使用的来源类型**

培训手册、YouTube 教程、讲义、PDF、ePub、Google Docs、Google Slides 和流程文档。

**前期设置**

- 上传覆盖完整主题或技能的资料。
- 为重要术语做一份词汇表。
- 把角色设定为"专家课程设计师"。
- 决定课程面向初学者、中级、专家，还是三者都有。

**工作流**

1. 提取核心技能或主题。
2. 定义课程的核心承诺。
3. 确定目标学习者。
4. 制定初学者、中级和进阶轨道。
5. 按顺序构建模块。
6. 在每个模块里添加课程。
7. 添加练习、评估和最终项目。
8. 生成学习指南、抽认卡或测验。
9. 把课程结构导出到 Docs。

**直接可用的提示词：**

```
Act as an Expert Curriculum Designer.

Turn the selected sources into a complete course.

Include:

1. Course title
2. Course promise
3. Target learner
4. Beginner, intermediate and advanced tracks
5. Module structure
6. Lesson titles
7. Learning outcomes
8. Exercises
9. Assessments
10. Final project
11. Common mistakes
12. Required templates or worksheets
13. Suggested study schedule

Use only the selected sources.
Mark gaps where the sources do not provide enough material.
```

**预期输出**

包含模块、课程、练习和评估的完整课程蓝图。

**质量把关**

- 练习是否基于来源材料？
- 课程顺序是否合理？
- 是否从初学者友好的内容开始？
- 是否保留了专家级深度？
- 缺失的示例是否标注为空白？

**常见错误**

- 让 AI 在资料范围之外发明课程内容
- 课程覆盖范围太宽
- 跳过评估环节
- 创建没有练习的被动课程

**进阶版本**

单独生成各类课程资产：

- 学生工作册
- 讲师指南
- 幻灯片大纲
- 题库
- 最终项目评分标准
- 每个模块的音频概述
- 公开的配套笔记本

**最终产出**

一门完整课程。

### 用例 3：创建深度研究或长篇指南

**用途**

把分散的研究整合成一篇有分量的书面指南、文章、电子书章节或技术解说。

**最适合**

作家、研究人员、创作者、创始人、分析师、学生和顾问。

**使用的来源类型**

PDF、ePub、文章、字幕、报告、Google Docs 和网页。

**前期设置**

- 上传相关资料。
- 确定最终读者是谁。
- 把 AI 设定为"高级技术写手"或"专家解说者"。
- 逐节构建指南。

**工作流**

1. 生成核心论点。
2. 生成大纲。
3. 把每一节映射到对应的支撑来源。
4. 逐节起草。
5. 核查主张。
6. 添加示例和注意事项。
7. 构建结论。
8. 导出到 Google Docs。

**直接可用的提示词：**

```
Act as a Senior Technical Writer.

Create a long-form guide from the selected sources.

First produce:

1. Working title
2. Thesis
3. Target reader
4. Section-by-section outline
5. Main argument of each section
6. Supporting sources for each section
7. Contradictions or tensions to address
8. Gaps that need a second pass

Do not write the full guide yet.
First create the architecture.
```

**预期输出**

有来源支撑和论点流程的完整指南大纲。

**质量把关**

- 论点是否有来源支撑？
- 各节是否按逻辑顺序排列？
- 主张是否有根据？
- 矛盾是否有所处理？
- 是否包含了示例？

**常见错误**

- 用一个提示词要求输出完整指南
- 接受通用大纲
- 没有把有来源支撑的主张和综合内容分开
- 忽视矛盾

**进阶版本**

逐节构建：

```
Write Section 1 only.

Use the agreed outline.
Use only the selected sources.

Include:
- clear explanation
- source-backed claims
- examples
- warnings
- practical steps
- gaps
- section summary
```

**最终产出**

一篇长篇指南或深度研究。

### 用例 4：构建研究矩阵或数据表

**用途**

把杂乱的资料转化为结构化的研究数据库。

适用于：

- 主张/证据矩阵
- 文献综述矩阵
- 竞争对手比较表
- 词汇表数据库
- 工具库
- 矛盾映射
- 来源质量审核
- 示例库
- 引用库
- 框架库
- 行动项目表

**最适合**

研究人员、分析师、顾问、学生、合规团队和战略家。

**使用的来源类型**

PDF、学术论文、报告、电子表格、CSV 文件、网页和技术文档。

**前期设置**

- 上传资料。
- 给资料取清晰的名字。
- 生成表格之前先确定好列名。
- 尽可能使用数据表功能。
- 导出到 Sheets 进行分析。

**工作流**

1. 选择表格类型。
2. 定义精确的列。
3. 生成表格。
4. 审核空单元格。
5. 检查引用。
6. 导出到 Sheets。
7. 把表格当作研究数据库使用。

**直接可用的提示词：**

```
Create a Data Table from all selected sources.

Use these columns:

1. Item
2. Category
3. Definition or description
4. Source-backed evidence
5. Practical implication
6. Example
7. Limitation or warning
8. Related concept
9. Confidence level
10. Gap or missing information

Rules:
- Only include information supported by the selected sources.
- Merge exact duplicates.
- Preserve different framings where they add nuance.
- Mark inferred points clearly.
- Leave a cell blank if the source does not provide the information.
```

**预期输出**

一张干净的结构化表格，可以导出到 Sheets。

**质量把关**

- 列是否有实际意义？
- 单元格是否塞得太满？
- 没有来源支撑的主张是否已标注？
- 引用是否可以追溯？
- 完全相同的内容是否已合并？

**常见错误**

- 只说"帮我做一张表"，不定义列
- 列太多，表格太宽
- 信任推断出来的单元格内容
- 不检查来源覆盖情况

**进阶版本**

为以下内容分别做独立的表格：

- 框架
- 主张
- 矛盾
- 定义
- 示例
- 工具
- 常见错误
- 来源质量
- 实施步骤

**最终产出**

研究矩阵或结构化知识数据库。

### 用例 5：文献综述和学术研究工作流

**用途**

处理学术论文，提取主题、比较方法、识别空白，构建文献综述。

**最适合**

学生、研究人员、学者和分析师。

**使用的来源类型**

学术 PDF、期刊文章、研究数据集、方法论论文和综述文章。

**前期设置**

- 按子主题给论文分组。
- 避免一次上传太多大体量 PDF。
- 在 NotebookLM 之外使用引用管理工具。
- 把角色设定为"博士后研究员"。

**工作流**

1. 上传一批有针对性的论文。
2. 创建来源审核表。
3. 提取研究问题和研究方法。
4. 按主题对论文分组。
5. 识别共识和分歧。
6. 比较各论文的研究方法。
7. 提取研究空白。
8. 生成文献综述大纲。
9. 导出到 Docs，并用引用管理工具正确引用。

**直接可用的提示词：**

```
Act as a post-doctoral researcher.

Conduct a literature review of the selected papers.

Include:

1. Major thematic clusters
2. Consensus view in each cluster
3. Dissenting papers or opposing findings
4. Methodologies used
5. Methodological weaknesses
6. Key evidence
7. Research gaps
8. Suggested research questions
9. Literature review outline

Use source-backed claims only.
Mark inferred synthesis clearly.
```

**预期输出**

包含方法比较和空白分析的主题文献综述大纲。

**质量把关**

不要假设修改幻灯片时会自动用到来源。

先在聊天里构建并核查大纲，再生成幻灯片，修改之后再次手动核查事实。

**工作流**

1. 在聊天里创建幻灯片大纲。

**聊天用提示词：**

```
Create a source-backed slide deck outline.

For each slide include:

1. Slide title
2. Core message
3. Source-backed bullet points
4. Supporting source name
5. Any numbers or claims requiring manual verification
6. Suggested visual
7. Speaker note

Do not create the slide deck yet.
First produce the verified outline.
```

**修改用提示词：**

```
Revise the slide deck for clarity and presentation quality.

Only revise:
- wording
- layout
- visual hierarchy
- slide flow
- audience clarity

Do not introduce new factual claims.
Do not add new statistics.
Do not change numerical claims unless I provide the corrected numbers manually.
```

**预期输出**

经过核查的幻灯片大纲和可导出的演示文稿。

**质量把关**

- 数字是否正确？
- 幻灯片是否文字过多？
- 是否先核查了大纲？
- 修改过程中是否引入了新的主张？

**常见错误**

- 没核查大纲就直接生成幻灯片
- 试图同时修事实和改设计
- 把生成的幻灯片当最终版本
- 不检查数字和主张

**进阶版本**

上传品牌规范，让 NotebookLM 调整语气和演讲者注释。仍然手动核查所有事实。

**最终产出**

演示幻灯片。

```
Create a table with:

1. Source name
2. Source type
3. Publication date if available
4. Author or organisation if available
5. Core thesis
6. Usefulness rating
7. Potential bias or weakness
8. What this source is best used for
9. Whether it should be kept, removed or used cautiously

Then identify any contradictions between sources.
```

**预期输出**

一份来源质量报告。

**质量把关**

- 日期是实际存在的还是被推断出来的？
- AI 是否推断了缺失的元数据？
- 薄弱来源是否已删除？
- 识别出的矛盾是否真实存在？

**常见错误**

- 跳过来源审核
- 信任每一个上传的链接
- 保留旧的或质量差的材料
- 不检查矛盾

**进阶版本**

每次生成重要输出后都进行完整的主张核查。

**最终产出**

来源审核报告。

### 用例 16：大型项目和多笔记本工作流

**用途**

管理规模超出单个笔记本上限的项目。

**最适合**

博士项目、书籍写作、企业知识库、研究团队和大型内容系统。

**使用的来源类型**

大体量来源库、PDF、书籍、字幕、报告和项目文档。

**前期设置**

- 接受一个笔记本装不下所有东西这个事实。
- 按主题、项目、受众或输出类型拆分来源。
- 用外部存储做永久记忆。

**工作流**

1. 创建主题笔记本。
2. 分别处理每个笔记本。
3. 生成 Bridge Summary（跨笔记本摘要）。
4. 导出 Bridge Summary。
5. 存到 Obsidian、Notion、Docs 或 Zotero。
6. 需要跨笔记本工作时，用 Gemini 辅助。
7. 把最终草稿带回 NotebookLM 进行核查。

**直接可用的提示词：**

```
Create a Bridge Summary for this notebook.

The summary must represent the most important knowledge from all selected sources.

Include:

1. Core topic
2. Main themes
3. Key frameworks
4. Important evidence
5. Contradictions
6. Gaps
7. Useful examples
8. Recommended next-step questions
9. Source list

This Bridge Summary will be exported into an external knowledge base.
Make it dense and portable.
```

**预期输出**

一份可以在笔记本外部独立使用的高密度摘要。

**质量把关**

- 是否涵盖了所有主要来源？
- 来源名称是否得到保留？
- 重要空白是否包含在内？
- 这份摘要在 NotebookLM 外部是否还能用？

**常见错误**

- 把大 PDF 拼合在一起绕过来源上限
- 做一个装所有东西的超级笔记本
- 没有导出摘要
- 离开 NotebookLM 后丢失引用上下文

**进阶版本**

搭建一套协作网络：

- NotebookLM 负责以来源为锚点的综合
- Zotero 负责管理论文
- Obsidian 或 Notion 负责永久知识存储
- Gemini 负责跨笔记本协调

**最终产出**

跨笔记本知识系统。

### 用例 17：NotebookLM 和 Gemini 联合工作流

**用途**

把 NotebookLM 的来源锚定能力，和 Gemini 的长提示词处理、更广推理与执行能力结合起来。

**最适合**

重度用户、创作者、开发者、战略家、教育者和研究人员。

**使用的来源类型**

NotebookLM 输出、报告、Gemini 对话记录、Google Docs 和导出的笔记。

**前期设置**

- 用 NotebookLM 做以来源为锚点的提取。
- 把输出导出或传给 Gemini。
- 用 Gemini 做长篇生成或更广泛的执行任务。
- 需要时把最终输出带回 NotebookLM 核查。

**工作流**

1. 在 NotebookLM 里完成提取。
2. 保存或导出提取内容。
3. 用 Gemini 把提取内容做成指南、课程、代码简报、电子书或剧本。
4. 用 NotebookLM 来源核查 Gemini 的输出。
5. 回到 NotebookLM 生成 Studio 资产。

**Gemini 提示词：**

```
You are working from a NotebookLM extraction.

Your job is to turn the extracted source-grounded material into a final knowledge asset.

Rules:
- Do not restart extraction.
- Do not invent unsupported claims.
- Separate source-backed material, synthesis and inference.
- Preserve all high-signal frameworks, examples, tactics and warnings.
- Write in British English.
- Structure the output as a practical playbook.

Build:
1. Executive overview
2. Core mental model
3. Feature-to-outcome map
4. Use-case library
5. Prompt library
6. Source-type strategy
7. Verification system
8. Recommended notebook architectures
9. Final knowledge asset pipeline
10. Final audit
```

**预期输出**

长篇指南、课程、剧本或扩展的知识产出物。

**质量把关**

- Gemini 是否加入了来源中没有的网络知识？
- 主张是否仍然可以追溯？
- 它是否改变了来源材料的含义？
- 是否需要回到 NotebookLM 重新核查？

**常见错误**

- 用 Gemini 来做提取，而不用 NotebookLM
- 忘记 Gemini 的来源锚定方式不同
- 让 Gemini 自己发明示例
- 没有把最终输出带回来核查

**进阶版本**

让 Gemini 负责：

- 编码
- 长提示词编辑
- 电子书重构
- 文章生成
- 架构或网站搭建
- 宽范围的创意扩展

让 NotebookLM 负责：

- 来源提取
- 核查
- Studio 输出
- 引用检查

**最终产出**

Gemini 加持的知识资产。

### 用例 19：书籍、电子书或长篇内容工作流

**用途**

把 NotebookLM 用作书籍、电子书和长篇内容的研究与结构引擎。

**最适合**

作者、创作者、学者、教育者和新闻通讯作者。

**使用的来源类型**

ePub、书籍笔记、访谈录音、字幕、草稿、研究论文和大纲。

**前期设置**

- 上传研究资料。
- 上传草稿材料。
- 把角色设定为"高级编辑"。
- 逐章推进。

**工作流**

1. 提取核心论点。
2. 提取书籍的主要论据。
3. 构建章节映射。
4. 把证据分配到各章。
5. 识别空白。
6. 起草章节大纲。
7. 生成练习或工作册材料。
8. 构建配套资产。

**直接可用的提示词：**

```
Act as a Senior Editor.

Use the selected sources to create a book architecture.

Include:

1. Book title options
2. Core thesis
3. Target reader
4. Chapter-by-chapter outline
5. Main argument of each chapter
6. Supporting source material
7. Examples to include
8. Gaps in the source material
9. Exercises or workbook ideas
10. Promotional content angles
```

**预期输出**

包含证据支撑和章节逻辑的书籍架构。

**质量把关**

- 论点是否连贯？
- 章节是否重复过多？
- 示例是否真实？
- 书籍整体是否有递进感？

**常见错误**

- 让 NotebookLM 为整本书代笔
- 使用太多不相关的资料
- 跳过确立论点这一步
- 不检查叙事是否流畅

**进阶版本**

生成：

- 配套工作册
- 章节练习
- 课程版本
- 系列文章
- 发售推广内容
- AI 助手版本

**最终产出**

书籍大纲和研究库。
