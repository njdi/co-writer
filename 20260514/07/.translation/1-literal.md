## 1：什么是 NotebookLM？

NotebookLM 最好理解为一个以来源为根基的工作空间，但但但……这是什么意思？

你上传可信材料。NotebookLM 帮助你提取、整理、比较、综合并将这些材料转化为结构化的输出，所以你应该用它来做什么？

- 从上传的来源中提取知识
- 构建研究矩阵和数据表
- 将来源集合转化为课程、指南和标准操作程序
- 从密集材料中创建学习系统
- 生成 Studio 资产，如音频概述、视频概述、报告、数据表、抽认卡、测验、幻灯片、信息图和思维导图
- 为外部 AI 助手创建可复用的知识文件
- 将客户、业务或研究材料转化为精致的成果物
- 将杂乱信息转化为可以复用、教授、分享或导出的结构化输出

NotebookLM 很强大。它以来源为根基，可以核查引用。这种根基性降低了幻觉风险，改善了你获得的输出。

这样想：

NotebookLM 将可信的来源材料转化为结构化的知识资产。将其用作分层系统：

- **输入层：** 上传可信来源
- **推理层：** 使用聊天功能提取、比较和综合
- **捕获层：** 将有用的输出保存为笔记
- **复利层：** 将强力笔记转换回来源
- **资产层：** 生成报告、表格、抽认卡、测验、幻灯片、音频、视频、信息图和思维导图
- **部署层：** 导出到 Docs、Sheets、PDF、PPTX、Markdown、Notion、Obsidian 或其他系统
- **扩展层：** 当项目超出 NotebookLM 的原生边界时，使用 Gemini、Claude、自定义 GPT、Zotero、Obsidian 或 Notion

## 2：它是如何工作的？

在这篇文章变得极其有用之前，我们必须经历这段稍微枯燥的部分，这样你就知道它在做什么。如果你对这部分打哈欠，想直接看干货，可以向下滚动，但你对一个工具了解得越多，你从中得到的就越多。

NotebookLM 有六个工作层。

**第 1 层：来源**

来源是输入层。

这些是你上传的 PDF、文档、网页、视频、音频文件、电子表格、图片或粘贴的文本。

它们定义了笔记本知识的边界。

糟糕的来源产生糟糕的输出。
好的来源产生有用的输出。
精心策划的来源始终胜过随机堆砌。

**第 2 层：聊天**

聊天是推理和综合层。

在这里你提问、提取信息、比较来源、测试想法并生成结构化答案。

聊天的用途：

- 来源清单
- 提取
- 综合
- 矛盾映射
- 主张审核
- 大纲
- 提示词
- 清单
- 草稿报告
- 核查

不要以"总结一切"开始。从清单、提取和结构开始。

**第 3 层：笔记**

笔记是捕获层。

它们保留有价值的输出，使其不会在漫长的聊天中消失。

笔记的用途：

- 保存的大纲
- 提取的框架
- 来源摘要
- 模块计划
- 经过核查的主张
- 最终草稿
- 项目决策

好的笔记不仅仅是保存的答案。

它是可复用的项目记忆。

**第 4 层：将笔记转换为来源**

这是复利层。

这是 NotebookLM 最强大的工作流之一。

你可以将答案保存为笔记，然后将该笔记转换为来源。这让你能够分阶段构建知识资产。

示例：

1. 提取所有框架。
2. 将提取结果保存为笔记。
3. 将笔记转换为来源。
4. 要求 NotebookLM 使用该新框架来源作为结构来构建课程。

这改变了整个游戏。

NotebookLM 不再是一次性问答工具，而是成为分阶段知识构建系统。

**第 5 层：Studio**

Studio 是资产生成层。

它将以来源为根基的材料转化为如下输出：

- 报告
- 数据表
- 学习指南
- 抽认卡
- 测验
- 思维导图
- 音频概述
- 视频概述
- 幻灯片
- 信息图

Studio 是 NotebookLM 从阅读助手变为生产工具的地方。

但不要让 Studio 来主导策略。

先提取。
再综合。
最后生成资产。

**第 6 层：导出与部署**

资产构建完成后，将其移至正确的目的地。

使用：

- Google Docs 用于报告和指南
- Google Sheets 用于表格和矩阵
- PPTX 或 PDF 用于幻灯片
- Markdown 用于 Obsidian、Notion、Claude Projects 或自定义 GPT
- MP3 用于音频概述
- 公开笔记本链接用于客户或受众共享
- html

NotebookLM 非常擅长综合。

但它并不总是你应该存储这些信息的地方……个人维基会更好！

### 用例 1：从上传的来源构建完整知识库

**用途**

将许多上传的来源转化为可复用的知识库，可以成为指南、课程、教程、操作手册、剧本、系列文章、幻灯片或 AI 助手文件。

**最适合**

研究人员、创作者、顾问、企业、教育者、专家以及任何构建严肃长期知识资产的人。

**使用的来源类型**

PDF、Google Docs、网页、YouTube 字幕、音频字幕、ePub、Markdown 文件、Google Slides、Google Sheets、CSV 文件和粘贴的笔记。

**设置**

- 删除弱、过时或不相关的来源。
- 清晰地重命名文件。
- 按主题分块处理非常长的文档。
- 如果主题使用专业术语，创建一个词汇表文档。
- 将词汇表上传为来源 1。
- 将笔记本角色配置为"专家知识架构师"。
- 分阶段处理来源，而不是要求一个巨大的输出。

**工作流**

1. 运行来源清单。
2. 逐来源提取知识。
3. 创建主题映射。
4. 创建概念映射。
5. 创建框架映射。
6. 创建流程映射。
7. 识别矛盾和空白。
8. 构建模块化知识库。
9. 将每个模块保存为笔记。
10. 如有用，将强力笔记转换为来源。
11. 从这些笔记来源构建最终指南、课程或剧本。
12. 审核来源覆盖情况和不支持的主张。

**可粘贴的提示词：**

```
Act as an Expert Knowledge Architect.
...（保持英文原文）
```

**预期输出**

一个包含模块、定义、框架、流程、示例、清单、模板、提示词和最终审核的结构化知识库。

**质量控制检查**

- 每个来源是否出现在清单中？
- 输出是否过度依赖某一个来源？
- 完全相同的内容是否已合并？
- 不完全相同但相关的想法是否得到保留？
- 不支持的推断是否已标注？
- 空白是否已明确说明？

**常见错误**

- 要求"总结一切"
- 跳过清单阶段
- 未能将有用的输出保存为笔记
- 让 NotebookLM 压缩过多内容
- 将第一个答案视为最终答案

**进阶版本**

使用分阶段的笔记转来源工作流：

1. 将来源清单保存为笔记。
2. 将框架映射保存为笔记。
3. 将模块大纲保存为笔记。
4. 将最强的笔记转换为来源。
5. 要求 NotebookLM 使用笔记来源作为结构、原始来源作为证据来构建最终知识库。

**生成的最终资产**

一个可复用的知识库。

### 用例 2：将来源转化为课程

**用途**

将上传的材料转化为包含模块、课程、练习、评估和最终项目的结构化课程。

**最适合**

教育者、创作者、顾问、培训师、教练、企业和专家。

**使用的来源类型**

培训手册、YouTube 教程、讲义、PDF、ePub、Google Docs、Google Slides 和流程文档。

**设置**

- 上传覆盖完整主题或技能的来源。
- 为重要术语创建词汇表。
- 将角色配置为"专家课程设计师"。
- 决定课程是面向初学者、中级、专家还是三者都有。

**工作流**

1. 提取核心技能或主题。
2. 定义课程承诺。
3. 识别目标学习者。
4. 创建初学者、中级和进阶轨道。
5. 按顺序构建模块。
6. 在每个模块中添加课程。
7. 添加练习、评估和最终项目。
8. 生成学习指南、抽认卡或测验。
9. 将课程结构导出到 Docs。

**可粘贴的提示词：**

```
Act as an Expert Curriculum Designer.
...（保持英文原文）
```

**预期输出**

包含模块、课程、练习和评估的完整课程蓝图。

**质量控制检查**

- 练习是否基于来源材料？
- 课程顺序是否合理？
- 是否从初学者友好的内容开始？
- 是否保留了专家级深度？
- 缺失的示例是否标注为空白？

**常见错误**

- 让 AI 在来源材料之外发明课程
- 使课程过于宽泛
- 跳过评估
- 创建没有练习的被动课程

**进阶版本**

生成独立的课程资产：

- 学生工作册
- 讲师指南
- 幻灯片大纲
- 题库
- 最终项目评分标准
- 每个模块的音频概述
- 公开伴随笔记本

**生成的最终资产**

一门完整课程。

### 用例 3：创建深度研究或长篇指南

**用途**

将分散的研究转化为严肃的书面指南、文章、电子书章节或技术解说。

**最适合**

作家、研究人员、创作者、创始人、分析师、学生和顾问。

**使用的来源类型**

PDF、ePub、文章、字幕、报告、Google Docs 和网页。

**设置**

- 上传相关来源。
- 确定最终读者。
- 将 AI 配置为"高级技术写手"或"专家解说者"。
- 逐节构建指南。

**工作流**

1. 生成论点。
2. 生成大纲。
3. 将每节映射到支持来源。
4. 逐节起草。
5. 审核主张。
6. 添加示例和警告。
7. 构建结论。
8. 导出到 Google Docs。

**可粘贴的提示词：**

```
Act as a Senior Technical Writer.
...（保持英文原文）
```

**预期输出**

有来源支撑和论点流程的强力指南大纲。

**质量控制检查**

- 论点是否得到来源支撑？
- 各节是否按逻辑顺序排列？
- 主张是否有根基？
- 矛盾是否得到处理？
- 是否包含示例？

**常见错误**

- 用一个提示词要求完整指南
- 接受通用大纲
- 未将来源支撑的主张与综合内容分开
- 忽视矛盾

**进阶版本**

逐节构建：

```
Write Section 1 only.
...（保持英文原文）
```

**生成的最终资产**

长篇指南或深度研究。

### 用例 4：构建原生研究矩阵或数据表

**用途**

将杂乱的来源转化为结构化的研究数据库。

用于：

- 主张/证据矩阵
- 文献综述矩阵
- 竞争对手比较表
- 词汇表数据库
- 工具库
- 矛盾映射
- 来源质量审核
- 示例库
- 引用库
- 框架库
- 行动项目表

**最适合**

研究人员、分析师、顾问、学生、合规团队和战略家。

**使用的来源类型**

PDF、学术论文、报告、电子表格、CSV 文件、网页和技术文档。

**设置**

- 上传来源。
- 清晰命名来源。
- 在生成表格之前确定列。
- 尽可能使用数据表。
- 导出到 Sheets 进行分析。

**工作流**

1. 选择表格类型。
2. 定义精确列。
3. 生成表格。
4. 审核空单元格。
5. 检查引用。
6. 导出到 Sheets。
7. 将表格用作研究数据库。

**可粘贴的提示词：**

```
Create a Data Table from all selected sources.
...（保持英文原文）
```

**预期输出**

可以导出到 Sheets 的干净结构化表格。

**质量控制检查**

- 列是否有用？
- 单元格是否过度填充？
- 不支持的主张是否已标注？
- 引用是否可用？
- 完全相同的重复是否已合并？

**常见错误**

- 要求"一张表格"但不定义列
- 使表格过宽
- 信任推断的单元格
- 不检查来源覆盖情况

**进阶版本**

为以下内容创建单独的表格：

- 框架
- 主张
- 矛盾
- 定义
- 示例
- 工具
- 错误
- 来源质量
- 实施步骤

**生成的最终资产**

研究矩阵或结构化知识数据库。

### 用例 5：文献综述和学术研究工作流

**用途**

处理学术论文以提取主题、比较方法、识别空白并构建文献综述。

**最适合**

学生、研究人员、学者和分析师。

**使用的来源类型**

学术 PDF、期刊文章、研究数据集、方法论论文和综述文章。

**设置**

- 按子主题分组论文。
- 避免一次性上传太多大型 PDF。
- 在 NotebookLM 外部使用引用管理器。
- 将角色配置为"博士后研究员"。

**工作流**

1. 上传一批有针对性的论文。
2. 创建来源审核表。
3. 提取研究问题和方法。
4. 按主题分组论文。
5. 识别共识和异议。
6. 比较方法。
7. 提取空白。
8. 生成文献综述大纲。
9. 导出到 Docs 并使用引用管理器正确引用。

**可粘贴的提示词：**

```
Act as a post-doctoral researcher.
...（保持英文原文）
```

**预期输出**

包含方法比较和空白分析的主题文献综述大纲。

**质量控制检查**

- AI 是否混淆了相关性和因果关系？
- 它是否夸大了共识？
- 它是否正确识别了方法？
- 它是否创造了假研究空白？
- 引用是否已手动检查？

**常见错误**

- 将 NotebookLM 视为引用管理器
- 一次上传太多论文
- 立即要求完成的论文章节
- 忽视方法论差异

**进阶版本**

创建：

- 方法论矩阵
- 证据质量表
- 矛盾映射
- 研究空白表
- 论文章节大纲
- 口头答辩问题库

**生成的最终资产**

文献综述大纲和研究矩阵。

### 用例 6：考试复习和主动学习系统

**用途**

将被动学习材料转化为主动回忆、测验、抽认卡、苏格拉底式辅导和复习计划。

**最适合**

学生、考试候选人、职业学习者以及任何学习密集材料的人。

**使用的来源类型**

讲义、教科书 PDF、历年考题、笔记、YouTube 讲座和学习指南。

**设置**

- 一次只上传一个模块或主题的材料。
- 将 NotebookLM 配置为"严格的苏格拉底式导师"。
- 生成学习指南、抽认卡和测验。
- 使用聊天进行主动回忆。

**工作流**

1. 上传学习材料。
2. 要求概念映射。
3. 生成学习指南。
4. 生成抽认卡或测验。
5. 运行苏格拉底式测试模式。
6. 追踪薄弱领域。
7. 生成复习计划。
8. 每周重复。

**可粘贴的提示词：**

```
Act as a strict Socratic tutor.
...（保持英文原文）
```

### 用例 19：书籍、电子书或长篇内容工作流

- 上传草稿材料。

3. 构建章节映射。

5. 识别空白。

```
Act as a Senior Editor.
```
