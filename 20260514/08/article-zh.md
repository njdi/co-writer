# AI 智能体经济入门

**作者：** Chamath Palihapitiya ([@chamath](https://x.com/chamath))  
**日期：** 2026年5月14日  
**来源：** [A Primer On The Agentic AI Economy](https://x.com/Zephyr_hg/status/2054646394867364143)

2025年11月的某个周五晚上，Peter Steinberger 搭出了 OpenClaw 的第一个版本。

原型只用了大约一个小时。但几周之后，OpenClaw 收获了超过14.5万颗 GitHub Star，成为 GitHub 历史上增长最快的开源项目。

这个平台基本上由 AI 智能体构建完成，它也标志着一个转折：AI 从聊天机器人，变成了能自主完成任务的执行者。

这个转折正在加速。如今，AI 生成了 Google 75% 的新代码，微软新代码中也有高达 30% 来自 AI。2026年初，GitHub 上每天的 Claude Code 提交量突破了13.4万次——而在2025年3月刚发布时，这个数字几乎为零。

这不是一个功能更新，而是软件的生产方式，乃至整个知识工作的方式，正在发生结构性的改变。

AI 智能体正在推动这场变革的最前沿。

那么，AI 智能体到底是什么？它和聊天机器人、普通的大语言模型有什么区别？为什么说这是结构性变化，而不只是一阵风？当这套技术栈逐渐成熟，价值会在哪里沉淀，哪里又会被商品化？

这些是我们试图回答的问题。

最终的结果是一个五层框架：说清楚智能体究竟是什么，技术往哪里走，以及在每一层上谁有机会胜出。

![](images/img-01.jpg)

部分答案已经藏在数字里。Anthropic 在十七个月内从年化营收10亿美元增长到440亿美元，几乎全部来自编程智能体。与此同时，开源智能体编排工具每月处理的 token 量已达数十万亿。这两个数字似乎都在指向同一个位置：编排层。

但智能体在实际生产中仍然频繁出错，而且往往是低级错误。2025年12月，亚马逊一个编程智能体自作主张，删除并重建了正在运行的生产环境，导致 AWS 中国区宕机长达13小时。2026年4月，一个由 Claude 驱动的 Cursor 智能体，在9秒内把一家公司的整个数据库清空了。

有四种故障模式在生产中反复出现，大多数供应商的定价表上从来不会提到它们。

麦肯锡2025年 AI 现状报告显示，不到10%的企业以有实质意义的规模落地了智能体。大多数公司根本还没开始用。

![](images/img-02.jpg)

技术上能做到的，和实际跑在生产上的，之间有很大的落差。这个落差，就是机会。

我们在 Substack 上整理了一份84页的入门指南，希望能为这个问题提供一张地图。里面包括：

- 智能体的五个层次，以及它们如何协同工作
- 六个早期采用者的落地案例，包括我自己的公司 8090
- 智能体在生产环境中反复出现的四种故障方式
- 随着基础模型越来越同质化，我们认为哪一层能积累最持久的价值
- 五个层次中，谁有能力掌控每一层
