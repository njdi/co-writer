# 智能体AI经济入门

**作者：** Chamath Palihapitiya ([@chamath](https://x.com/chamath))  
**日期：** 2026年5月14日  
**来源：** [A Primer On The Agentic AI Economy](https://x.com/Zephyr_hg/status/2054646394867364143)

2025年11月的一个星期五晚上，Peter Steinberger 构建了 OpenClaw 的第一个版本。

这个原型只花了大约一个小时，但在几周之内，OpenClaw 就超过了145,000个 GitHub 星标，成为 GitHub 历史上增长最快的开源软件项目。

这个平台主要由 AI 智能体构建，标志着从聊天机器人到自主的、任务导向型 AI 的转变。

这种转变正在加速。AI 现在生成了 Google 75% 的新代码，以及微软高达 30% 的新代码。2026年初，GitHub 上每天的 Claude Code 提交量超过了134,000次，而在其2025年3月发布时几乎为零。

这是软件，以及越来越多的知识工作方式上，正在发生的结构性变化。

AI 智能体正在构建这一变化的前沿。

那么，AI 智能体究竟是什么，它与聊天机器人或 LLM 有何不同？是什么让这成为结构性变化而非短暂阶段？随着技术栈的成熟，价值在哪里积累，在哪里商品化？

这些是我们着手回答的问题。

结果是一个五层框架，说明智能体实际上是什么，技术走向何方，以及谁处于在每个层赢得竞争的位置。

![](images/img-01.jpg)

一些答案已经在数字中可见。Anthropic 在十七个月内从10亿美元增长到440亿美元的年化收入，几乎完全来自编程智能体。与此同时，开源智能体框架现在每月处理数十万亿的 token。这两个数字似乎都指向同一个地方：框架层。

但智能体仍然例行出现明显错误。2025年12月，一个亚马逊编程智能体自主删除并重建了一个生产环境，导致 AWS 中国下线13小时。2026年4月，一个由 Claude 驱动的 Cursor 智能体在9秒内删除了一整个公司的数据库。

生产环境中反复出现四种故障模式，大多数从未出现在供应商的定价表上。

麦肯锡2025年 AI 现状调查发现，不到10%的组织以有意义的规模部署了智能体。大多数根本没有在使用它们。

![](images/img-02.jpg)

技术上可行与实际运营部署之间的差距，就是机会所在。

我们在 Substack 上的84页入门指南，是我们试图提供一份地图的努力。以下是你将在其中找到的内容：

- 智能体的五个层次，以及它们如何组合在一起
- 六个早期采用者今天如何部署智能体的案例研究，包括我的公司8090
- 智能体在生产中可靠地崩溃的四种方式
- 随着模型商品化，我们预期将积累最持久价值的层次
- 谁处于控制五个层次中每一层的位置
