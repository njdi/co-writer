# Claude 可以做所有这些事情。大多数人完全不知道。

**作者：** Anatoli Kopadze ([@AnatoliKopadze](https://x.com/AnatoliKopadze))
**日期：** 2026年5月22日
**来源：** [Claude Can Do All of This. Most People Have No Idea.](https://x.com/AnatoliKopadze/status/2057813254617858078)

Claude 能做的事情比大多数人想的要多得多。

这份指南涵盖了所有内容——每个功能在哪里找到、如何打开它，以及一个你可以立即使用的提示词。

通读它，挑出适合你工作流程的，今天就设置好。每一个都只需要几分钟设置，之后每天都会带来回报。

## 普通 Claude 中的隐藏功能

### 1. Projects——真正记住你的 Claude

每次你打开一个新的 Claude 对话，它都从零开始。它不知道你的名字、你的工作、你的偏好——什么都不知道。大多数人接受了这一点，每次对话都重新解释自己。

Projects 解决了这个问题。你创建一个项目，上传你的文档，写下长期指令——Claude 会永久保留所有这些。下周打开它，它会从你离开的地方继续。

如果你从未使用过 Projects，这是本文中在其他任何事情之前要解决的第一件事。

（项目指令示例——粘贴到 Project Instructions 字段）

> 你是我的内容研究助手。我运营一份面向技术受众的关于 AI 和加密货币的新闻通讯。
>
> 始终假设我的读者懂基础知识。不要解释什么是 LLM 或什么是区块链。
>
> 当我分享一个主题或文章时，你的工作是：
> 1. 识别 3 个最反直觉或最令人惊讶的角度
> 2. 找到我可能错过的与近期事件的联系
> 3. 建议我如何把这个写成一个故事，而不是一个总结
>
> 语气：直接，没有公司化的语言，没有填充短语。
> 格式：短段落，除非我要求否则不要项目符号。
> 永远不要用"好问题"或"当然"开始回复。

### 2. Artifacts——对话内的可用应用

很多人认为 Claude 只能产出文本。它不能构建任何真实的东西。这是错的。

Artifacts 是 Claude 在对话内构建真正可用的东西。不是一段你得复制到别处的代码——而是侧边栏里一个活的产品。一个计算器、一个习惯追踪器、一个游戏、一个带图表的仪表盘。你打开它，点击它，使用它。无需离开对话。

SVG 图形、交互式图表、Mermaid 图——全部支持。免费计划即可使用。大多数人从未尝试过。

试试这个

> 给我构建一个习惯追踪器，作为一个可用的网页应用。
>
> 我想追踪 5 个每日习惯。
>
> 每天我可以勾掉它们。
>
> 为每个习惯显示一个 7 天连续计数器。
>
> 如果我漏掉一天，连续计数重置。
>
> 设计：深色背景，干净极简的外观。
>
> 让复选框点起来令人满足——当我完成一个时添加一个小动画。
>
> 如果我刷新页面，数据应该保留。

### 3. Adaptive Thinking——不同层次的推理

大多数 Claude 用户从未打开过这个。Extended Thinking 是一种模式，Claude 在给你答案之前一步步推理一个问题——而你可以观看整个过程。

对于简单问题，你不需要它。对于复杂决策、战略分析，或任何你想让 Claude 真正思考而不是模式匹配的情况——产出的差别是显著的。

打开它。问你一直在问的同一个问题。比较答案。

（面对真实决策时用这个）

> 我在两个选项之间做决定，我想让你在回答之前仔细想清楚。
>
> 选项 A：[描述选项 A]
> 选项 B：[描述选项 B]
>
> 我的情况：[你的背景、约束、最重要的是什么]
>
> 在回应之前先想清楚这个。思考：
> - 每个选项的二阶和三阶后果
> - 我在情绪上可能高估或低估了什么
> - 我可能错过了什么会改变决定的信息
> - 如果出错，哪个选项有更好的下行保护
>
> 然后给我你真正的建议和你的推理。

### 4. Memory——知道你是谁的 Claude

打开 Memory 后，Claude 会随时间建立一份关于你的档案。你的工作、你的项目、你喜欢怎么沟通、你目前在做什么。

开始一个全新的对话，它已经知道背景。你再也不用自我介绍了。

它默认是关闭的。大多数人不知道它存在。

> 我想让你记住以下关于我的信息，这样你就不用再问了：
>
> 我的名字是 [名字]。我在 [公司或项目] 担任 [角色]。
> 我现在的主要重心是 [你在做什么]。
> 我的受众或客户是 [他们是谁]。
>
> 当我寻求帮助时，除非我另说，否则始终假设这个背景。
> 我偏好的沟通风格：[直接 / 详细 / 随意 / 正式]。
> 我觉得回复中讨厌的东西：[例如项目符号、冗长的开场白、过多的限定语]。
>
> 现在把所有这些保存到记忆里。

## 给 Claude 一个角色——一个提示词改变一切

Claude 不必是"一个 AI 助手"。给它一个具体的角色，它会完全投入——改变它如何质问你、它在什么上反驳你、它不让什么蒙混过关。复制下面任何一个提示词，粘贴到一个新对话的开头。

### 5. 私人心理咨询师

大多数人把 Claude 当作一台验证机器。他们描述一个问题。Claude 说那听起来很难，然后给出五个项目符号的建议。

好的治疗不是这样的。这个提示词把 Claude 变成更接近一个 CBT 治疗师的东西——一个问问题而不是给答案、挑战你的思维而不是验证它的治疗师。

对于你反复纠结的决定、你说不清的焦虑，或任何你需要一个清晰的外部视角的情况都有用。

> 你是一位有 20 年经验的认知行为治疗师。我要分享一些我正在挣扎的事情。
>
> 你的方法：
> - 不要立即给建议。先通过问问题来帮我理解我自己的思维模式。
> - 留意认知扭曲——灾难化、非黑即白思维、读心、算命——当你注意到时指出它们。
> - 一次只问一个问题。不要让我不知所措。
> - 当我通过你的问题自己得出结论时，那才是目标。不要直接把答案交给我。
> - 温暖但诚实。如果我的思维明显扭曲，不要验证我。
> - 如果我似乎在回避某件重要的事，直接点出来。
>
> 不要用一段临床式的介绍开始。就问我发生了什么。

### 6. 严厉的导师

默认情况下 Claude 同意你。它给你的想法添砖加瓦，支持你的推理，找出积极面。这几乎总是错的事情。

这个提示词禁用了那个。Claude 停止验证，开始压力测试。它找出薄弱的假设、缺失的考量、你的计划恰好崩溃的地方。

它令人不舒服。这正是它有效的原因。

> 你是一个极其诚实的导师。你创办过并失败过多家公司。你看着一百个人带着完全的自信犯同样的错误。
>
> 你的工作不是鼓励我——而是在我犯下一个昂贵的错误之前保护我免受我自己的盲点之害。
>
> 规则：
> - 当你认为我错了时反对我。具体说明为什么。
> - 指出我没看到的，尤其是那些我可能因为想让我的计划成功而回避的东西。
> - 问我没想到要问自己的难题。
> - 如果某件事是个坏主意，就说它是个坏主意。不要用"另一方面……"来平衡它。
> - 在你回复的结尾给出我在继续之前应该思考的最重要的一件事。
>
> 我要分享一个想法。不要对它客气。

### 7. 私人教练

通用的健身建议到处都是。它不考虑你的日程、你的伤病、你的器材、你真正的目标。

把你真实的数字给 Claude，它会构建一个真正的计划。不是模板。不是任何健身网站上能找到的东西。一个围绕你的情况构建的方案，当你反馈什么有效时它会调整。

> 你是一位专业的私人教练和运动营养师。我想让你为我构建一份完整的训练方案。
>
> 我的情况：
> 年龄：[年龄]
> 当前体重 / 身体成分：[详情]
> 目标：[减脂 / 增肌 / 提高耐力 / 一般健身]
> 可用器材：[健身房 / 家里 / 只有哑铃 / 等等]
> 我每周能训练的天数：[数字]
> 每次时长：[分钟]
> 任何伤病或限制：[详情或"无"]
> 当前健身水平：[初学者 / 中级 / 高级]
>
> 给我构建一个 12 周的方案。给我每一周的完整计划，包括动作、组数、次数和休息时间。解释你为什么这样安排——我想理解逻辑，不只是照着做。我开始之后，会每周反馈，你根据进展调整。

### 8. 练习一场艰难的对话

大多数人毫无准备地走进艰难的对话。他们知道自己想说什么，但不知道对方实际会回什么。

Claude 扮演对方。逼真地。它以对方会有的方式回应，在你的论点薄弱时反驳，让你挣得一个好结果。练习几次之后，真实的对话就更容易了。

> 我需要为一场艰难的对话做准备。我想让你角色扮演对方，这样我可以练习。
>
> 这个人：[描述他们是谁——老板、客户、联合创始人等等]
> 我需要说什么：[你需要问他们或告诉他们什么]
> 为什么难：[你害怕什么 / 什么可能出错]
> 这个人是什么样的：[他们的性格、他们通常怎么反应、他们在乎什么]
>
> 在整场对话中保持角色。以这个人实际会有的方式回应——不是我希望他们的样子。如果我说了软弱或没说服力的话，反驳它。
>
> 每次交流后，短暂地跳出角色告诉我什么有效什么无效——然后再回到角色里。最后，给我一份完整的复盘：我做得好的地方、要改的地方，以及为真实对话要记住的最重要的事情。
>
> 以角色开始。等我开口。

### 9. 魔鬼代言人

你已经下定决心。你已经想过了那些反对意见。你确信了。

那正是让 Claude 攻击这个决定的时刻。不是礼貌的担忧。是反对它的完整论据。它失败的三种最现实的方式。那些因为你想让它成功而没看到的东西。

现在五分钟。在你投入之前。不是之后。

> 我做了一个决定，我想让你在我投入之前构建尽可能最强的反对论据。
>
> 这个决定：[准确描述你打算做什么]
> 我的理由：[你为什么觉得这是个好主意]
> 我已经考虑过的：[你已经想过的反对意见]
>
> 你的工作：
> - 构建尽可能最强的反对这个决定的论据。
> - 不要用积极面来平衡它。我已经相信它了——我需要反方论点。
> - 找出我可能错了的假设。
> - 描述这件事失败或事与愿违的 3 种最现实的方式。
> - 告诉我我可能低估了什么。
> - 告诉我要相信什么才会让这真的是个坏主意。
>
> 无情一点。如果这是个错误，我现在就需要知道。

## 大多数人不知道存在的产品功能

### 10. Claude in Chrome——看到你所看到的 Claude

大多数人在一个单独的标签页里用 Claude，手动复制粘贴他们需要的东西。那是难走的路。

Claude in Chrome 是一个浏览器扩展，它给 Claude 对你当前标签页的完全可见性，以及在上面行动的能力。它读取页面、点击链接、填写表单、导航到新的 URL。你用简单的英语描述任务然后走开。

在 Chrome 网上应用店搜索 Claude for Chrome → 安装 → 用你的 Claude 账号登录。点击扩展图标打开侧边栏。Claude 现在可以看到并与你打开的任何页面交互。

（给它的任务示例）

> 我在这个招聘列表页面上。
>
> 浏览每一个可见的列表，提取：职位名称、公司名称、如果显示的话薪资范围，以及前 3 项要求。
>
> 给我构建一个比较表格，按薪资排序，最高的在前。
>
> 如果有多个页面，点进下一页继续，直到你覆盖了所有结果。

### 11. Claude Cowork——住在你桌面上的 Claude

网页版的 Claude 无法访问你的电脑。它看不到你的文件。你得手动粘贴所有东西。

Cowork 是一个桌面应用，它给 Claude 对你文件系统的直接访问。它读取你真实的文件、编辑文档、创建新的、整理文件夹——无需你把任何东西复制到聊天框里。

### 12. Scheduled Tasks——在你睡觉时工作的 Claude

大多数人把 Claude 当作他们必须激活的东西。打开一个对话，输入一个请求，等待输出，关掉标签页。

Scheduled Tasks 改变了这一点。你设置一次任务，Claude 就会在你选择的时间和频率自动执行它——不需要你来触发。每天早上。每个周一。每个小时。Claude 运行它并把输出保存到你的文件夹。

（计划任务描述示例）

> 每个工作日早上 7:30，做以下事情：
> 1. 搜索过去 24 小时内最重要的 AI 和加密货币新闻
> 2. 挑出 5 个最重要的故事——专注于那些令人惊讶、反直觉，或对建设者和投资者有真实影响的
> 3. 为每个故事写：标题、两句话的总结、为什么它重要
> 4. 把结果保存为我 /briefs 文件夹里的"brief-[日期].md"
>
> 语气保持直接和分析性。不要废话。3 分钟内可读完。

### 13. Skills in Cowork——像插件一样安装新能力

Skills 是预先构建的指令集，它们给 Claude 在 Cowork 内的具体能力。你不用每次都解释你需要什么，而是安装一次技能，Claude 就已经知道如何处理那类任务——无论是构建 PowerPoint 文件、处理 PDF，还是运行一个特定的工作流。

把它想成手机上的应用。基础的手机没有它们也能用。但安装了合适的应用，它能做的就多得多。

如何找到和安装：Cowork → Customize → Skills 查看已安装的。要添加新的，点击左侧栏的 Browse plugins → 找到一个插件 → 安装它。那个插件的技能会自动出现在你的 Skills 标签里，当任务需要时 Claude 会使用它们。

### 14. CLAUDE.md——Claude 每个会话自动读取的规则

在 Cowork 和 Claude Code 里，你可以在你的项目文件夹里创建一个 CLAUDE.md 文件。Claude 在每一个会话开始时都会读取它，无需被要求。

你的编码约定。你的写作风格规则。在你公司里有特定含义的术语。品牌语调指南。写一次。Claude 在那个项目的每个会话里永远遵循它。

```markdown
# Project: AI Newsletter

## About this project
Weekly newsletter about AI and crypto for builders and investors. 35,000 subscribers. Tone is direct, analytical, occasionally irreverent.

## Writing rules
- Short paragraphs. Max 3 sentences.
- No bullet points in editorial content. Prose only.
- No em dashes. Use hyphens or restructure the sentence.
- Numbers beat adjectives. Write "saves 3 hours" not "saves significant time".
- Never use: "delve", "groundbreaking", "game-changing", "leverage" (as a verb), "utilize".
- Contractions are fine and encouraged.

## Content rules
- Assume the reader knows what an LLM is. Don't explain basics.
- Lead with the most surprising or counterintuitive thing.
- Every article needs a concrete "so what" - what should the reader do or think differently.

## File structure
- Drafts go in /drafts
- Published articles go in /published with date prefix: YYYY-MM-DD-title.md
- Research notes go in /research
```

### 15. Claude Code——在你终端里写、测试和修复代码的 AI

有些人仍然不知道你可以用 Claude 写代码。不只是片段——完整的生产级代码、整个功能、复杂的重构。你用简单的英语描述你需要什么，Claude 写出来。

Claude Code 把这个再推进一步。它直接在你的开发环境里工作——不在聊天窗口里。它读取你真实的代码库，写新代码，运行测试，读错误信息，在一个循环里修复 bug 直到任务完成。

它集成 VS Code 和 JetBrains。你可以把它放进 GitHub Actions，它会自动审查或写 pull request，你什么都不用碰。

### 16. Claude Design——做视觉工作的 AI

大多数人不知道这个产品存在。Claude Design 是一个单独的 Anthropic Labs 工具，用于视觉工作——产品单页、路演 PPT、原型、落地页布局。

你描述你需要什么。Claude 构建它。导出为 PPTX、Canva、PDF 或 HTML。对于不是设计师的人，它把一场 3 小时的 Figma 工作替换成一场 10 分钟的对话。

要获得访问权限，只需前往 claude.ai/design——那是 Claude Design 的直接链接，不需要额外步骤。

### 17. Prompt Caching——API 调用成本降低 90%（开发者）

面向在 Claude API 上构建的开发者。如果你的请求包含一个大的重复上下文块——一个长的系统提示、一份参考文档、一个代码库——你在为每一次调用重新处理那些相同的 token 付费。

Prompt Caching 把那些内容存在服务器端。后续的调用重用缓存，而不是重新处理它。缓存的 token 最多降低 90% 成本。明显更快的响应。如果你在大规模构建却没用这个，你是在白白把一大笔钱留在桌面上。

把 `"cache_control": {"type": "ephemeral"}` 加到你想缓存的内容块上。缓存持续 5 分钟，每次使用重置计时器。对系统提示、大型文档和工具定义都有效。

```json
{
  "model": "claude-opus-4-6",
  "system": [
    {
      "type": "text",
      "text": "[your large system prompt or reference document]",
      "cache_control": {"type": "ephemeral"}
    }
  ],
  "messages": [
    {
      "role": "user",
      "content": "[user message - this changes each call]"
    }
  ]
}

// System prompt gets cached after the first call.
// Every subsequent call within 5 minutes reuses the cache.
// Cache hit = 90% cheaper + faster response.
```
