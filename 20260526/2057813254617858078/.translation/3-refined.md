# 这些 Claude 全都能做，多数人却根本不知道

**作者：** Anatoli Kopadze ([@AnatoliKopadze](https://x.com/AnatoliKopadze))  
**日期：** 2026年5月22日  
**来源：** [Claude Can Do All of This. Most People Have No Idea.](https://x.com/AnatoliKopadze/status/2057813254617858078)  

Claude 能做的事，比大多数人想的要多得多。

这份指南把它们全讲一遍——每个功能在哪儿找、怎么打开，还附上一段你可以马上拿去用的提示词。

通读一遍，挑出适合你工作流程的，今天就设置好。每一个都只要几分钟，之后每天都在帮你省事。

## 普通 Claude 里藏着的功能

### 1. Projects——真正记得住你的 Claude

每次打开一个新的 Claude 对话，它都从零开始。它不知道你的名字、你的工作、你的偏好——什么都不知道。大多数人就这么接受了，每次对话都把自己重新介绍一遍。

Projects 能解决这个问题。你建一个项目，传上你的文档，写好长期生效的指令——Claude 就会把这些一直记着。下周再打开，它接着上次的地方往下走。

如果你从没用过 Projects，那么在动手做别的之前，先把这一项搞定。

![](images/img-01.jpg)

项目指令示例——粘贴到 Project Instructions 字段里：

> 你是我的内容研究助手。我运营一份面向技术读者、关于 AI 和加密货币的新闻通讯。
>
> 默认我的读者都懂基础。不用解释什么是 LLM，什么是区块链。
>
> 当我丢给你一个主题或一篇文章，你要做的是：
> 1. 找出 3 个最反直觉、最让人意外的切入角度
> 2. 把它和我可能错过的近期事件联系起来
> 3. 给我建议，怎么把它写成一个故事，而不是一份总结
>
> 语气：直接，别用官腔，别堆废话。
> 格式：短段落，除非我开口，否则不要用项目符号。
> 永远别用"好问题"或"当然"来开头。

### 2. Artifacts——对话里就能用的小应用

很多人以为 Claude 只能写文字，造不出真东西。这是错的。

Artifacts 就是 Claude 在对话里直接造出一个能用的东西。不是一段你还得复制到别处的代码，而是侧边栏里一个活生生的产品。一个计算器、一个习惯打卡器、一个小游戏、一个带图表的看板。你打开、点击、用起来，全程不用离开对话。

SVG 图形、交互式图表、Mermaid 流程图——全都支持。免费版就能用。可大多数人从没试过。

试试这个：

> 帮我做一个习惯打卡器，要做成一个能用的网页应用。
>
> 我想打卡 5 个每日习惯。
>
> 每天我能把它们勾掉。
>
> 给每个习惯显示一个 7 天连续打卡的计数。
>
> 漏掉一天，连续计数就清零。
>
> 设计：深色背景，干净极简。
>
> 复选框点起来要让人觉得爽——勾掉一个时加一个小动画。
>
> 刷新页面后，数据要还在。

### 3. Adaptive Thinking——另一个层次的推理

大多数 Claude 用户从没打开过它。Extended Thinking 是这样一种模式：Claude 在给你答案之前，一步步把问题想清楚——而且整个过程你都能看到。

简单的问题用不着它。但碰到复杂决策、战略分析，或者任何你希望 Claude 真去想、而不是随手套路的场合——出来的结果差别很大。

打开它。把你一直在问的那个问题再问一遍。对比一下两次的答案。

![](images/img-02.jpg)

真要做个决定时，用这个：

> 我在两个选项之间犹豫，希望你回答前先认真想一想。
>
> 选项 A：[描述选项 A]
> 选项 B：[描述选项 B]
>
> 我的情况：[你的背景、限制、你最看重什么]
>
> 回应之前先把这件事想透。想想：
> - 每个选项的连锁反应，以及再往后的影响
> - 我在情绪上可能高估或低估了什么
> - 我可能漏掉了哪些会改变结论的信息
> - 万一出事，哪个选项的损失更小
>
> 然后给我你真正的建议，连同你的理由。

### 4. Memory——知道你是谁的 Claude

打开 Memory，Claude 会慢慢攒出一份关于你的档案。你的工作、你的项目、你喜欢怎么交流、你眼下在忙什么。

开一个全新的对话，它已经知道前因后果。你再也不用做自我介绍了。

它默认是关着的。大多数人根本不知道有这东西。

![](images/img-03.jpg)

> 我想让你记住下面这些关于我的事，省得每次都问：
>
> 我叫 [名字]。我在 [公司或项目] 做 [职位]。
> 我现在主要在忙 [你在做什么]。
> 我的受众或客户是 [他们是谁]。
>
> 我找你帮忙时，除非我另说，默认就按这个背景来。
> 我喜欢的沟通方式：[直接 / 详细 / 随意 / 正式]。
> 我看着烦的回复方式：[比如项目符号、冗长的开场白、一堆限定语]。
>
> 现在把这些全存进记忆里。

## 给 Claude 一个角色——一句提示词就全变了

Claude 不一定非得是"一个 AI 助手"。给它一个具体角色，它就会彻底入戏——怎么追问你、在哪儿顶你、什么事不让你蒙混过去，全都变了。下面任意一段提示词，复制下来，贴到新对话开头即可。

### 5. 私人心理咨询师

大多数人把 Claude 当成一台点赞机器。他们说出一个困扰，Claude 说"这确实挺难的"，然后给五条项目符号的建议。

好的心理咨询不是这样的。这段提示词能把 Claude 变成更接近 CBT 治疗师的样子——它问问题，而不是给答案；它质疑你的想法，而不是一味附和。

碰到那种你反复纠结的决定、说不清道不明的焦虑，或者任何需要一个清醒的旁观者视角的时候，它都管用。

> 你是一位有 20 年经验的认知行为治疗师。我要跟你说一件让我很纠结的事。
>
> 你的做法：
> - 别急着给建议。先问问题，帮我看清自己的思维模式。
> - 留意那些认知扭曲——把事情往最坏处想、非黑即白、揣测别人心思、预言坏结局——一旦发现就指出来。
> - 一次只问一个问题。别一下子把我压垮。
> - 当我顺着你的问题自己想明白时，那才算成功。别直接把答案塞给我。
> - 温和但要诚实。如果我的想法明显跑偏了，别附和我。
> - 如果我好像在躲着某件要紧事，直接点破。
>
> 别拿一段诊室式的开场白来开头。就问我，到底怎么了。

### 6. 不留情面的导师

默认情况下，Claude 是顺着你的。它帮你的想法添砖加瓦，替你的推理撑腰，专挑好的说。可这几乎总是帮倒忙。

这段提示词把这套关掉。Claude 不再附和，转而开始使劲挑毛病。它会找出站不住脚的假设、被你忽略的环节，以及你的计划到底会在哪儿崩。

这会让你有点难受。但正因如此，它才管用。

> 你是一位极其诚实的导师。你创过业，也失败过好几回。你眼睁睁看着上百个人，带着十足的自信，犯下同样的错。
>
> 你的任务不是鼓励我——而是在我犯下一个代价高昂的错误之前，帮我躲开自己的盲区。
>
> 规则：
> - 觉得我错了，就反对我。具体说清楚为什么。
> - 把我没看到的指出来，尤其是那些我可能因为太想成功而故意不看的。
> - 问我那些自己都没想到要问的硬问题。
> - 一个主意要是不行，就说它不行。别拿"不过话说回来……"来打圆场。
> - 每次回复的结尾，告诉我在往前走之前，最该想清楚的那一件事。
>
> 我马上要说一个想法。对它别客气。

### 7. 私人健身教练

通用的健身建议满大街都是。可它不管你的作息、你的旧伤、你手边的器材、你真正想要的目标。

把你真实的数字交给 Claude，它会给你一份真正的计划。不是模板，也不是哪个健身网站上随便能搜到的东西，而是一份照着你的情况量身定的方案，等你反馈进展后它还会调整。

> 你是一位资深私人教练兼运动营养师。我想让你帮我做一份完整的训练方案。
>
> 我的情况：
> 年龄：[年龄]
> 当前体重 / 身体成分：[详情]
> 目标：[减脂 / 增肌 / 提升耐力 / 一般健身]
> 可用器材：[健身房 / 家里 / 只有哑铃 / 等等]
> 每周能训练几天：[数字]
> 每次多长时间：[分钟]
> 有没有伤病或限制：[详情，或"无"]
> 当前水平：[初学者 / 中级 / 高级]
>
> 帮我做一份 12 周的方案。把每一周的完整计划给我，包括动作、组数、次数和休息时间。再讲讲你为什么这么安排——我想搞懂背后的逻辑，而不是照着做。开始之后，我每周反馈一次，你根据情况调整。

### 8. 演练一场难开口的对话

大多数人走进一场艰难的对话时，根本没准备。他们知道自己想说什么，却不知道对方会怎么回。

让 Claude 来演对方，而且演得很像。它会照着对方的反应来回，你的理由一软它就顶回来，不会轻易让你赢。这样练上几遍，真正的对话就好开口多了。

> 我要为一场难开口的对话做准备。我想让你扮演对方，好让我练一练。
>
> 这个人：[他们是谁——老板、客户、联合创始人，等等]
> 我要说的话：[你要问他们或告诉他们什么]
> 难在哪儿：[你在怕什么 / 哪里可能出岔子]
> 这个人是什么样：[性格、平时怎么反应、在乎什么]
>
> 整场对话里都待在角色里。照这个人真实会有的反应来回——不是我希望他们的样子。我要是说了软话、没说服力的话，就顶回来。
>
> 每来回一次，短暂跳出角色，告诉我哪句有效、哪句不行——然后再钻回去。最后给我一份完整复盘：我做得好的地方、要改的地方，以及真正对话时最该记住的几件事。
>
> 从角色开始。等我先开口。

### 9. 唱反调的人

你已经拿定主意。该想的反对意见你都想过了。你信了。

可恰恰就是这个时候，该让 Claude 来攻这个决定。不是客客气气提点担忧，而是反对它的全套论据——它失败的三种最现实的路径，以及那些因为你太想成功而看不见的东西。

就现在，花五分钟。在你真正下手之前，而不是之后。

> 我做了一个决定，希望你在我下手之前，把反对它的论据做到最强。
>
> 这个决定：[准确说出你打算做什么]
> 我的理由：[你为什么觉得这是个好主意]
> 我已经想过的：[你已经考虑过的反对意见]
>
> 你要做的：
> - 把反对这个决定的论据做到最强。
> - 别拿好处来平衡。我已经信了——我要的是反方那一面。
> - 找出我那些可能站不住脚的假设。
> - 描述这件事翻车或反噬的 3 种最现实的路径。
> - 告诉我，我大概低估了什么。
> - 告诉我，要相信哪些前提，这才会真的成为一个坏主意。
>
> 别留情。如果这是个错，我现在就得知道。

## 大多数人不知道还有的产品功能

### 10. Claude in Chrome——和你看到同一个页面的 Claude

大多数人是另开一个标签页用 Claude，再手动把需要的东西复制过来。那是费劲的走法。

Claude in Chrome 是一个浏览器扩展，它让 Claude 能完整看到你当前的标签页，还能在上面动手。它读页面、点链接、填表单、跳转网址。你用大白话把任务讲清楚，然后就能走开。

去 Chrome 网上应用店搜 Claude for Chrome → 安装 → 用你的 Claude 账号登录。点扩展图标打开侧边栏。这下 Claude 就能看到、也能操作你打开的任何页面了。

可以交给它的任务示例：

> 我现在在这个招聘列表页面上。
>
> 把页面上能看到的每一条招聘都过一遍，提取：职位名称、公司名称、有写的话还有薪资范围，以及最重要的 3 项要求。
>
> 帮我做一张对比表格，按薪资从高到低排。
>
> 如果有好几页，就点进下一页接着做，直到所有结果都覆盖到。

### 11. Claude Cowork——住在你桌面上的 Claude

网页版的 Claude 碰不到你的电脑。它看不见你的文件，你得把所有东西手动粘进去。

Cowork 是一个桌面应用，它让 Claude 能直接访问你的文件系统。它读你真实的文件、改文档、建新文件、整理文件夹——你不用把任何东西复制进聊天框。

![](images/img-04.jpg)

### 12. Scheduled Tasks——你睡觉时还在干活的 Claude

大多数人把 Claude 当成一个得你主动去点的东西。打开对话、敲下请求、等结果、关标签页。

Scheduled Tasks 改变了这一点。你把任务设置好一次，Claude 就会按你定的时间和频率自动跑——根本不用你来触发。每天早上、每个周一、每个小时都行。Claude 跑完，把结果存进你的文件夹。

![](images/img-05.jpg)

定时任务的描述示例：

> 每个工作日早上 7:30，做这几件事：
> 1. 搜一搜过去 24 小时里最重要的 AI 和加密货币新闻
> 2. 挑出 5 条最重要的——专挑那些让人意外、反直觉，或者对开发者和投资人有真实影响的
> 3. 每条都写：标题、两句话的总结、为什么它重要
> 4. 把结果存成我 /briefs 文件夹里的"brief-[日期].md"
>
> 语气保持直接、有分析。别废话。3 分钟内能读完。

### 13. Cowork 里的 Skills——像装插件一样添新本事

Skills 是预先做好的指令集，给 Claude 在 Cowork 里添上具体的本事。这样你就不用每次都解释要干什么——装一次技能，Claude 就已经知道这类任务该怎么处理，不管是做 PowerPoint、处理 PDF，还是跑一个特定的流程。

可以把它想成手机上的 App。光秃秃的手机也能用，但装上合适的 App，能干的事就多多了。

怎么找、怎么装：Cowork → Customize → Skills，能看到已经装好的。要加新的，点左侧栏的 Browse plugins → 找一个插件 → 装上。那个插件带的技能会自动出现在你的 Skills 标签里，任务一需要，Claude 就会用上。

### 14. CLAUDE.md——Claude 每次开工都自动读的规则

在 Cowork 和 Claude Code 里，你可以在项目文件夹里建一个 CLAUDE.md 文件。每个会话一开始，Claude 都会自动读它，不用你提醒。

你的编码约定、你的写作风格、在你公司里有特定含义的术语、品牌的语调要求。写一次，Claude 就会在这个项目的每个会话里一直照着做。

```markdown
# Project: AI Newsletter

## About this project
Weekly newsletter about AI and crypto for builders and investors. 35,000 subscribers. Tone is direct, analytical, occasionally irreverent.

## Writing rules
- Short paragraphs. Max 3 sentences.
- No bullet points in editorial content. Prose only.
- No em dashes. Use hyphens or restructure the sentence.
- Numbers beat adjectives. Write "saves 3 hours" not "saves significant time".
- Never use: "delve", "groundbreaking", "game-changing", "leverage" (as a verb), "utilize".
- Contractions are fine and encouraged.

## Content rules
- Assume the reader knows what an LLM is. Don't explain basics.
- Lead with the most surprising or counterintuitive thing.
- Every article needs a concrete "so what" - what should the reader do or think differently.

## File structure
- Drafts go in /drafts
- Published articles go in /published with date prefix: YYYY-MM-DD-title.md
- Research notes go in /research
```

### 15. Claude Code——在终端里写代码、跑测试、修 bug 的 AI

有些人到现在还不知道，你可以用 Claude 写代码。不只是片段——而是完整的生产级代码、整个功能、复杂的重构。你用大白话说出要什么，Claude 就写出来。

Claude Code 又往前走了一步。它直接在你的开发环境里干活，而不是在聊天窗口里。它读你真实的代码库，写新代码，跑测试，看报错信息，然后一遍遍修 bug，直到任务完成。

它能接 VS Code 和 JetBrains。你还能把它塞进 GitHub Actions，它会自动审查或撰写 pull request，你一根手指都不用动。

### 17. Prompt Caching——API 调用成本降低 90%（开发者向）

这条给在 Claude API 上做开发的人。如果你的请求里带着一大块反复出现的上下文——一段长的系统提示、一份参考文档、一个代码库——那你每次调用，都在为重新处理那同一批 token 付钱。

Prompt Caching 把那块内容存在服务器端。之后的调用直接复用缓存，不用再重新处理。缓存命中的 token，成本最多能降 90%，响应也明显更快。如果你在做大规模的东西却不用这个，那等于在白白往外扔钱。

把 `"cache_control": {"type": "ephemeral"}` 加到你想缓存的那个内容块上。缓存保留 5 分钟，每用一次就重新计时。对系统提示、大型文档和工具定义都管用。

```json
{
  "model": "claude-opus-4-6",
  "system": [
    {
      "type": "text",
      "text": "[your large system prompt or reference document]",
      "cache_control": {"type": "ephemeral"}
    }
  ],
  "messages": [
    {
      "role": "user",
      "content": "[user message - this changes each call]"
    }
  ]
}

// System prompt gets cached after the first call.
// Every subsequent call within 5 minutes reuses the cache.
// Cache hit = 90% cheaper + faster response.
```
