# 直译稿的问题

## 整体问题

- 大量英文句式残留，比如"模型真正被赢得的地方""模型胜负所在"，中文不这么说，应是"真正拉开差距的地方"。
- 被动语态太多："被训练出来""被工程化出来""被赢得"——中文偏好主动表达。
- 标题太长太硬，"确切流水线"生硬，可改"完整流程"。

## 逐处

- 标题：副标题"分 5 个阶段：……的确切流水线"读起来卡。改为更顺的说法。
- "把这个收藏起来。保存下来。"——两句意思重复且机械，合并成一句。
- "架构是阶段 1 里的一个脚注"——"脚注"是英文比喻，中文可保留但加解释会更顺，或说"只占一小段"。
- "教模型语言本身"——动宾不顺，改"让模型学会语言本身"。
- "选标准工具然后继续往前走"——口语化可以，但"继续往前走"略冗，改"用现成的标准件就行"。
- "把它变成训练材料是一个残酷的多步过滤"——"残酷"用词偏刺激（违反风格），改"要经过一道道严苛的筛选"。
- "值得刻进脑子的口头禅"——"口头禅"不准，原文 refrain 指反复强调的那句话，改"有句话值得记牢"。
- "守护得最严的秘密"——可保留，较自然。
- "让开放数据集相形见绌"——书面，可保留但稍调。
- "计算最优"→"算力上最划算"更白话。
- "惨痛的教训"——bitter lesson 是专有说法（Rich Sutton），保留并标注。
- "把一个预测器变成一个助手"——"预测器"生硬，改"会预测下一个词的模型"。
- "为偏好而不是模仿优化"——语序硬，改"优化的目标是人的偏好，而不是单纯模仿"。
- "测量某种开放式的东西"——很翻译腔，改"怎么衡量一个没有标准答案的东西"。
- "在多少个 token 之间犹豫"——保留犹豫的比喻，挺好。
- "让训练在物理上成为可能"→"让训练在现实里真能跑起来"。
- "天真地训练"——naively 直译，改"不做任何优化地训练"。
- "让 LLM 项目沉没的错误"——sink 直译，改"会让 LLM 项目翻车的几个坑"。
- 结尾"那就是差异所在"——可保留，但前面"深入研究它"略硬，改"往深里钻"。

## 风格底线

作者语气是"懂行的人在给你划重点、反复强调一个反直觉的观点（架构最不重要）"。译文要保住这种笃定和略带挑衅的劲儿，不能翻得太平。
