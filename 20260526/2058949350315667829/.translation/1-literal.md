# 我如何把 Claude Code 设置成我的投资研究分析师

**作者：** leopardracer ([@leopardracer](https://x.com/leopardracer))  
**日期：** 2026年5月26日  
**来源：** [How I Set Up Claude Code as My Investment Research Analyst](https://x.com/leopardracer/status/2058949350315667829)

如果你是搞金融的，你很可能看过 Claude Code 一眼，然后就移开了视线。

一个黑色的终端。一些毫无意义的命令。这是给工程师用的东西，不是给一个整天泡在电子表格和年报里的人用的。

我现在已经在 Claude Code 上跑我的整个投资研究工作流六十天了。筛选、抓取、建模、起草报告、监控投资组合。终端大概在第三天左右就不再让我感到害怕了。到第十天，我已经完全不再打开 Claude 来做研究了。

这里有件没人告诉过我、而我想告诉你的事：如果你想写代码，Claude Code 是用来写代码的。如果你想做饭，它是用来做饭的。如果你想购物，它是用来购物的。而如果你真的热衷于金融，Claude Code 也是用来做金融的。

名字里的"code"是这个产品最具误导性的部分。

这篇文章就是我希望第一天有人能递给我的那份设置指南。读完它，你就会装好 Claude Code，拥有一个让你做的每一份研究都不断累积复利的知识库，以及外接的三个工具，让 Claude Code 能够伸手进入实时网络、驱动浏览器、查询你自己的笔记本。

如果你看过配套的视频，你会知道它大概的样子。这篇文章在提示词、文件结构以及让整个东西运转起来的规则上讲得更深。

让我们从那个改变了我对这一切看法的想法开始。

## Claude 对比 Claude Code：没人解释过的那一件事

Claude 和 ChatGPT 是聊天机器人。你问，它们答。你再问，它们再答。有用，但有限。对话本身就是产品。

Claude Code 不一样。Claude Code 是有手的 Claude。

想想看，当你让 ChatGPT 帮你做一个投资研究工作流时会发生什么。你可以让它帮你筛选几只股票。你可以把财务数据粘贴进去，让它做个 DCF。你可以把一份草稿给它，让它清理一下语言。但这些里的每一个都是单独一步，而你夹在中间，在聊天和你其余的工作之间复制粘贴。

现在想象一个真实的研究工作流。你筛选整个股票池。你按几个参数列出候选名单。你把公司内部的框架套用到候选名单上。你挑出两只。你拉取这两家的财务报表。你建一个模型。你对模型做压力测试。你得出一个内在价值。你起草一份股票研究报告。一位资深分析师审阅它。你发布。

聊天机器人一次只能帮一步。Claude Code 能跑完整条链。

你给它最终目标。需要的话它会打开浏览器。需要的话它会抓取数据。它写文件。它读文件。如果任务允许，它会并行跑五件事。只有在真正需要你的时候它才向你要输入。工作在你做别的事情的时候发生。

这就是"手"的意思。ChatGPT 是一个你跟它说话的大脑。Claude Code 是一个跑你工作流的大脑。

有一句话我总是反复想起。一个聪明、勤奋的人靠做得更多来完成更多。一个智慧的人靠消除得更多来完成更多。Claude Code 就是那个消除者。我们称之为"研究工作"的东西，大部分其实是收集数据、排版、复制、粘贴、切换标签页。Claude Code 把这些从你盘子里拿走，只留下那个真正需要你的部分。

这就是解锁点。这篇文章里其余的一切，只不过是怎么把它设置好。

如果你更想在屏幕上看着整个设置过程发生，包括下一节里 Playwright 和 Firecrawl 的安装，我录了一段 16 分钟的演示，从头到尾覆盖了所有内容：

去 Anthropic 网站上的 Claude Code 页面。你会看到两条安装命令，一条给 Mac，一条给 Windows。复制适合你机器的那一条。

官方 Claude Code 设置文档：https://code.claude.com/docs/en/quickstart

![](images/img-01.jpg)

如果你用 Mac，打开 Terminal。如果你用 Windows，打开 PowerShell。粘贴命令，按回车，等一分钟。这就是安装。

现在输入 `claude` 并按回车。Claude Code 的提示符就打开了。你可以开始用纯英文输入。问它任何事。"嗨，我是新手，我该怎么开始？"这样就行。它会告诉你。

就这样。这就是安装。

![](images/img-02.jpg)

在你继续往下之前，还有一件事要做：下载 Cursor。

官方 Cursor 页面：https://cursor.com/home

Cursor 不是终端的替代品。它是这套系统其余部分生活的地方。当你在下一节设置 Obsidian 知识库时，你会把你的 Obsidian 仓库作为一个文件夹在 Cursor 里打开。你的 raw、wiki 和 output 文件夹都在侧边栏里。Claude Code 在它们旁边的一个面板里运行。你可以在同一个窗口里实时看着文件被创建、编辑和索引。从这里往后你构建的一切都发生在这一个地方。

如果你想，你仍然可以用普通终端。我有时候会用，尤其是当我想看着 Claude Code 啃一些复杂的东西时。但 Cursor 才是日常工作真正发生的地方，因为它是唯一一个知识库、聊天和工具能同时被看到的地方。

Claude Code 单独用是有用的，但它不是让这篇文章里的系统运转起来的东西。安装是两分钟。真正改变你研究的设置才是接下来要讲的。

### 可选，仅当你不想要 Claude 订阅时

继续之前快速说一句。从这里往后的一切都假设你有一个 Claude 订阅，Pro 或 Max。如果你没有，又不想为了试验而专门买一个，有一个能用的替代品叫 MiniMax M2，它能作为替换引擎接入 Claude Code，成本大约是七分之一……大约百分之七。设置好它，然后回来从下一节接着读。接下来的一切运作起来都完全一样。

## 第二步：把 Obsidian 设置成 Claude Code 的记忆

Claude Code 单独是没有记忆的。每次会话都是从头开始。你可以在周一跟它进行一场精彩的对话，关掉终端，到周二它什么都不记得。对大多数编程任务来说这没问题。对研究来说，这是个问题。研究是累积复利的。你三月份对一家半导体公司记下的笔记，到十一月应该还能找到，连同这期间你对这个行业学到的一切。

这正是 Obsidian 解决的问题。Obsidian 是一个免费应用，它把你电脑上的一个文件夹变成一个个人研究图书馆。每条笔记都是一个纯文本文件。你可以把笔记互相链接起来，把它们组织进主题文件夹，并瞬间搜索整个图书馆。没有数据库，没有云，没有锁定。你的研究就放在你的机器上，在你能用任何文本编辑器读取的文件里。

诀窍在于 Claude Code 能读写那同一个文件夹。一旦你设置正确，Obsidian 就成了 Claude Code 的长期记忆。你把图书管理员这个角色交给 Claude Code。它把你收集的东西归档，把你读过的东西总结出来，跨主题把相关的想法链接起来，并且每次你提问时都从图书馆里拉取。随着时间推移，图书馆变得越来越聪明，因为 Claude Code 给你的每个答案都被重新归档了回去。

![](images/img-03.jpg)

下面是怎么搭建它。

### 安装 Obsidian 并创建你的仓库

从 obsidian.md 下载 Obsidian。它个人使用免费，能在 Mac、Windows 和 Linux 上运行。

打开 Obsidian，点击"创建新仓库"。仓库就是你电脑上 Obsidian 存放笔记的一个文件夹。随便给它取个名字（我的就叫 Vault 本身）。选一个你能记住的位置，放在你的 Documents 文件夹里某个地方就行。Obsidian 会打开这个空仓库。

![](images/img-04.jpg)

现在在 Cursor 里打开那同一个仓库文件夹。文件 → 打开文件夹 → 选择你的仓库。你应该看到文件夹名字出现在 Cursor 的侧边栏里，现在是空的。从这里往后一切都发生在这里。

![](images/img-05.jpg)

### 创建文件夹结构

在 Cursor 里打开 Claude Code（侧边栏面板，或者终端，随你）。一字不差地粘贴这条提示词：

```
为我的知识库创建这个文件夹结构：

- raw/ — 这是我的收件箱，我会把源材料丢进这里
- wiki/ — 这是你的地盘，你会在这里写和维护一切
- 在 wiki/ 里创建一个 _master-index.md，标题为"Knowledge Base Index"，
    并附一条说明，写着"主题将在创建时列在这里。"
- output/ — 这是查询结果和报告存放的地方

只创建这些文件夹和那一个文件。其他暂时什么都不要做。
```

Claude Code 会创建三个文件夹和一个文件。打开 Obsidian，你会在侧边栏里看到它们。这就是你的骨架。

![](images/img-06.jpg)

三个文件夹，三项职责。raw/ 是你的收件箱，你把任何想让系统学习的东西丢进去：文章、文字记录、截图、笔记。wiki/ 是图书管理员的地盘，Claude Code 在这里写结构化的总结，把它们组织进主题文件夹，并维护一份关于一切的主索引。output/ 是完成的工作落地的地方：研究报告、查询答案、你想跟图书馆本身分开存放的分析。

### 规则手册（CLAUDE.md）

这是整个系统里最重要的文件。CLAUDE.md 是每次你在这个仓库里打开 Claude Code 时它都会读的规则手册。你把规则写一次，Claude Code 就永远遵循它们。没有这个文件，Claude Code 只是个聪明的助手。有了它，Claude Code 就是个有纪律的图书管理员。

在你仓库的根目录创建一个叫 CLAUDE.md 的文件（不在那三个文件夹里面，而是跟它们并排）。把这个粘贴进去：

```
这个文件告诉你如何在每次会话中维护这个知识库。

## 知识库规则

- 这是一个由 LLM 维护的知识库。你是图书管理员。
- wiki/ 文件夹是你的地盘 —— 你写和维护它里面的一切。
  我很少直接编辑 wiki 文件。
- raw/ 是收件箱。当我把文件丢进这里，你在一个"compile"步骤里
  把它们处理进 wiki。
- wiki/_master-index.md 是入口点。它列出每个主题文件夹，
  附一行描述。始终保持它最新。
- 每个主题在 wiki/ 里有自己的子文件夹（例如 wiki/ai-agents/），
  带自己的 _index.md，列出该主题下所有文章并附简短描述。
- 始终用 [[wiki links]] 来连接跨主题的相关概念。
- 编译原始材料时：
  1. 读取原始文件
  2. 决定它属于哪个主题（或创建一个新主题）
  3. 写一篇带关键要点和相关链接的 wiki 文章
  4. 更新该主题的 _index.md
  5. 更新 wiki/_master-index.md
  6. 如果一个原始文件跨多个主题，在两边都创建文章并交叉链接
- 文章保持简洁 —— 用要点而非段落。
- 每篇 wiki 文章里都包含一个 ## Key Takeaways 小节。
- output/ 用于查询结果和生成的报告。
- 回答问题时，先读 _master-index.md 来导航，然后
  钻进相关主题的 _index.md，再读具体文章。
- 当我说"compile"时，把 raw/ 里所有还没编译的东西
  处理进 wiki。
- 当我说"audit"或"lint"时，审查 wiki 里的不一致、
  断链、空缺，并提出改进建议。
```

保存这个文件。从现在起，每次会话都从 Claude Code 读这份规则手册开始。

### 每日输入：Web Clipper

Web Clipper 页面：obsidian.md/clipper

你需要一种方式把文章弄进你的 raw/ 文件夹，而不用复制粘贴。在你的浏览器上安装 Obsidian Web Clipper 扩展（Chrome、Brave、Edge、Firefox 都支持）。

打开 Web Clipper 设置。添加你的仓库。然后去 Templates，编辑 Default 模板：

```
Vault: 选择你的仓库
Note location: raw
Note name: {{date|date:"YYYY-MM-DD"}}-{{title|safe_name}}
```

现在你在网上读的任何文章，点一下就变成你收件箱里一个干净的 markdown 文件。一份财报电话会议记录、一篇 FT 的文章、一段 10-K 摘录、一篇 Substack 帖子，都用同样的方式被剪藏。

一个可选的附加项：在 Obsidian 里面安装 Local Images Plus 插件（设置 → 社区插件 → 浏览 → 搜索"Local Images Plus"）。当你打开一条剪藏的笔记时，这个插件会把文章里的每一张图片下载到你的仓库里，这样即使原始页面挂掉了，你的图书馆也保持完整。

### 每日工作流：四个动词

一旦一切都设置好了，工作流就是四个动词。

**剪藏（Clip）。** 你看到值得留下来的东西。在浏览器里点一下，它就落进 raw/。你不用决定它去哪，你不用写总结，你只管剪藏它。

**编译（Compile）。** 每周一次，或者每当你的收件箱感觉满了，打开 Claude Code 输入：

```
把 raw/ 里的所有东西编译进 wiki。对每个文件：
1. 读它并识别核心主题
2. 在 wiki/ 里创建或找到合适的主题文件夹
3. 写一篇带总结、关键要点和 [[wiki links]] 的 wiki 文章，
   链接到任何相关概念
4. 更新该主题的 _index.md
5. 更新 wiki/_master-index.md

在相关的地方跨主题交叉链接。
```

Claude Code 读取每篇剪藏的文章，决定它属于哪个主题，写一条结构化的总结笔记，把它归档到合适的文件夹，并更新索引。等你喝完咖啡，你的收件箱空了，你的图书馆变大了。

**查询（Query）。** 向图书馆提问。简单的："上个月我剪藏的那份贝恩半导体报告里的关键发现是什么？"交叉引用的："中国汽车出口的放缓，跟我一直在读的关于大宗商品需求的东西有什么关系？"把答案归档回图书馆的综合型问题："基于 wiki 里的所有内容，印度特种化学品的主要多头和空头理由是什么？把你的答案存成一篇新的 wiki 文章，并链接到你引用的来源。"

最后那个查询就是让系统产生复利的那一招。你问的每个问题都变成一条新笔记。每条笔记都链接到它的来源。每次你用它，图书馆都变得更聪明。

**审计（Audit）。** 每月一次，输入："审计这个 wiki。找出不一致、断链、缺失的交叉引用，以及覆盖上的空缺。先别改动，只给我一份报告。"Claude Code 会走一遍整个图书馆，告诉你哪里开始变模糊了。你来决定改什么。

这就是这套系统。收件箱、图书管理员、图书馆、输出。四个文件夹，一份规则手册，四个动词。

这件事比它看上去更重要的原因是：系统就是纪律。任何人都能装 Obsidian 和 Claude Code。你的研究累积复利而不是漏掉，原因在于规则手册强制了一致性。每次剪藏都用同样的方式归档。每份总结都遵循同样的格式。每个查询都能访问你读过的一切。跑了这套系统六十天后，我可以问 Claude Code"这个季度我关于仓位管理学到了什么"，并在一分钟内得到一个真正的答案，取自一些我都快忘了曾剪藏过的文章。

现在我们加上让 Claude Code 伸到你图书馆之外、进入实时网络的部分。

## 第三步：给 Claude Code 在网络上的手（Playwright + Firecrawl）

你刚搭建的知识库很强大，但它也是封闭的。它只知道你喂给它的东西。对真正的研究工作来说，你需要 Claude Code 伸到实时网络里去：从一个筛选器里拉一张资产负债表，抓取一个投资者关系页面，搜索你正在追踪的公司的最新备案文件。这就是接下来这两个工具做的事。

Playwright 让 Claude Code 驱动一个真实的浏览器。Firecrawl 让 Claude Code 大规模搜索和抓取网络。两个都是免费且开源的。两个都大约两分钟装好。它们合在一起，就是一个只读你带回家的东西的图书管理员，和一个还能走到城那头的图书馆把你需要的东西带回来的图书管理员之间的区别。

### Playwright：浏览器

Playwright 是微软出的一个浏览器自动化工具。一旦装好，Claude Code 就能打开浏览器，导航到任何页面，点击按钮，填写表单，登录账户，并读取屏幕上的内容，全程不用你碰鼠标。

对金融工作来说，这在数据藏在交互背后的地方很重要。一个要你设十个筛选条件才显示结果的筛选器。一个把季度数字藏在下拉菜单背后的投资者关系页面。一个要你接受免责声明才能下载备案文件的监管门户。任何因为页面期待一个人类而让普通抓取器失败的地方，Playwright 让 Claude Code 成为那个人类。

要安装它，在你的仓库里打开 Claude Code 并粘贴：

```
我想安装微软的 Playwright CLI，这样你就能代表我驱动浏览器。
仓库在 https://github.com/microsoft/playwright-cli。
请为我的 [Mac / Windows] 机器一步一步带我走完安装，
对新手友好，并运行你需要运行的命令。
```

Claude Code 会安装它，带你走过任何提示，并在准备好时确认。问个简单的东西来测试它："打开一个浏览器，去 screener.in，告诉我主页上有什么。"如果 Playwright 装对了，一个浏览器窗口会打开，导航到页面，Claude Code 会回报它看到的东西。

### Firecrawl：网络抓取器

Firecrawl 做的事不一样。Playwright 是一个亲手操作的浏览器，而 Firecrawl 是一种快速的、程序化的方式，从任何网页拉取干净的文本，或者跑一次网络搜索并从排名靠前的结果里拉取文本。它把找到的任何东西转换成 Claude Code 能直接读取的干净 markdown 文件。

金融人会立刻认出的用例：你想从 Finviz 拉最近两周的内幕交易数据。你想从一家公司的 IR 页面抓取一份年报。你想搜索"印度特种化学品的最新新闻"，并把排名前十的文章拉进你的知识库。Firecrawl 这三件事各用一条命令就做到了。

要安装：

```
我想安装 Firecrawl CLI，这样你就能代表我搜索和抓取网络。
官方文档在 https://docs.firecrawl.dev/sdks/cli。
请为我的 [Mac / Windows] 机器一步一步带我走完安装，
对新手友好，并给我每一条我需要运行的命令。
```

一旦装好，Firecrawl 就成了 Claude Code 能直接从终端调用的工具。你不用每次手动调用它。当一个任务需要实时网络时，Claude Code 认出它有 Firecrawl 可用，并自动使用它。

### 这在实践中是什么样子

这是我上周跑的一条真实提示词。背景是：我想把资金部署到五只标的上，而我候选名单上只有两只。我需要再找三只。

```
我在筛选要部署资金的标的。去 finviz.com，拉取美国上市公司
最近两周的内幕交易数据。分析这些数据，找强烈买入信号。
给我一份前 10 的观察名单，每只标的都附上理由。
也标出危险信号和要避开的标的。
```

Claude Code 请求使用 Firecrawl 的权限。我说是。它去了 Finviz，拉了数据，解析了它，跑了买入信号逻辑，交叉核对了基本面，然后带着一份十只标的的排名列表回来了。每一只都有一个论点、一份内幕购买的清单（谁买的、什么时候、买了多少）、关键风险，以及一份总结评分卡。从头到尾三分钟。

如果我手动做那件事，至少要九十分钟，而且我会在 Finviz 和 Excel 之间复制数据时引入错误。Claude Code 没离开终端就做完了。

### 关于权限的一件事

每次 Claude Code 想用一个工具，比如 Firecrawl 或 Playwright，或者修改一个文件，或者运行一条命令，它都会请求权限。默认情况下，这意味着每隔几步就有一个是/否的提示。学习的时候有用。一旦你信任这个系统，就烦人了。

有一个叫 `--dangerously-skip-permissions` 的标志，它把提示完全关掉。Claude Code 这时就能运行任何工具、修改你仓库里的任何文件、执行命令，而不用每次跟你确认。这个名字听起来吓人，它也应该吓人。你是在把对你工作目录的完全访问权交给 Claude Code。

我开着这个标志跑了六十天。什么都没坏。没有文件被误删，没有不该运行的命令被运行，没有安全问题。改变的是，我不再坐在终端前等着点"是"。我开始给 Claude Code 一个任务然后走开。等我端着咖啡回来，任务做完了。

这能行得通的原因是规则手册。CLAUDE.md 告诉 Claude Code 它的工作到底是什么、边界在哪。再加上 Claude Code 只能访问你打开的那个仓库文件夹、而不是你整台机器这个事实，风险比这个标志的名字暗示的要小得多。我会建议头一周开着权限跑，这样你能看到 Claude Code 在做什么。一旦你信任了这个模式，就把权限关掉，让它干活。

那是个人选择。有些读者会永远开着权限，那也没问题。系统两种方式都能行。

## 六十天教会了我什么

上面的设置大约花一个小时。它创造出的系统，是我跑了六十天还在学着用的东西。

有几件事让我吃惊。

第一件是终端多快就不再感觉像个编程工具了。到第一周结束，我往 Claude Code 里打字的方式就跟我往 ChatGPT 里打字一样了。纯英文，没有命令，没有语法。人们警告的那种害怕因素在第一天是真实的，到第三天就没了。如果你是个搞金融的读者，正在犹豫，那你犹豫的是一个撑不过第一次会话的问题。

第二件是当你不再做一个聊天机器人用户、而开始做一个工作流操作者时会发生什么。这个转变很微妙，但下游的一切都变了。你不再让 Claude Code"帮你"做一个任务，而开始把整个任务交给它。筛选这些标的，把它们过一遍这个框架，给活下来的拉财务数据，建模型，给每个起草一页纸。你给它最终目标然后走开。工作在后台发生。你回来时面对的是一个装满成品的文件夹。

第三件是研究过去漏掉了多少。在这套系统之前，我的研究散落在 Notion、Apple Notes、浏览器书签、五个不同文件夹里下载的 PDF，还有一个我很少打开的 Substack 草稿文件夹里。我会把同一个洞见重新发现两次，有时三次。Obsidian 知识库结束了这种情况。我现在读的一切都落到一个地方，被 Claude Code 归档进正确的主题，并在六周后我问一个相关问题时被拉回来。什么都不漏。信息累积复利。

还有一个值得知道的工具：NotebookLM。它是谷歌托管的研究工具，它解决的问题跟 Obsidian 不一样。Obsidian 是你的图书馆。NotebookLM 是一个针对某个特定研究目标的聚焦工作区：一家公司、一个行业、一批备案文件。你上传来源，NotebookLM 就在那些文档里给你有依据、带引用的答案。有一个 Python 封装，让你能从 Claude Code 内部直接查询一个笔记本，这意味着你可以构建一个工作流，让 Claude Code 在同一条提示词里既从你的 Obsidian 图书馆拉取，又从一个聚焦的 NotebookLM 来源集拉取。
