# 我如何把 Claude Code 设置成我的投资研究分析师

**作者：** leopardracer ([@leopardracer](https://x.com/leopardracer))  
**日期：** 2026年5月26日  
**来源：** [How I Set Up Claude Code as My Investment Research Analyst](https://x.com/leopardracer/status/2058949350315667829)

如果你是搞金融的，你很可能看过 Claude Code 一眼，然后就把视线移开了。

一个黑色的终端。一串毫无意义的命令。这是给工程师用的东西，不是给一个整天泡在电子表格和年报里的人用的。

我现在已经把整个投资研究工作流跑在 Claude Code 上六十天了。筛选、抓取、建模、起草报告、监控投资组合，全都在上面。大概到第三天，终端就不再让我害怕了；到第十天，我已经完全不再打开 Claude 来做研究了。

有件事没人告诉过我，而我想告诉你：想写代码，Claude Code 就是用来写代码的；想做饭，它就是用来做饭的；想购物，它就是用来购物的。而如果你真的热衷于金融，Claude Code 也是用来做金融的。

名字里的"code"，是这个产品最具误导性的部分。

这篇文章，就是我希望第一天有人能递给我的那份设置指南。读完它，你就装好了 Claude Code，拥有一个让你做的每份研究都不断叠加的知识库，还外接了三个工具，让 Claude Code 能伸手进入实时网络、驱动浏览器、查询你自己的笔记本。

如果你看过配套视频，你大概知道它长什么样。这篇文章会在提示词、文件结构，以及让整套东西运转起来的那些规则上，讲得更深一些。

我们先从那个改变了我整体看法的想法说起。

## Claude 与 Claude Code：没人解释过的那一件事

Claude 和 ChatGPT 是聊天机器人。你问，它们答；你再问，它们再答。有用，但有限。对话本身就是产品。

Claude Code 不一样。Claude Code 是有手的 Claude。

想想看，你让 ChatGPT 帮你做投资研究时会发生什么。你可以让它筛几只股票。你可以把财务数据贴进去，让它做个 DCF。你可以把草稿给它，让它润一润语言。但这些都是单独一步，而你夹在中间，在聊天框和你其余的工作之间来回复制粘贴。

现在想象一个真实的研究工作流。你筛选整个股票池，按几个参数列出候选名单，把公司内部的框架套到名单上，挑出两只，拉取这两家的财务报表，建模型，对模型做压力测试，得出内在价值，起草一份股票研究报告，交一位资深分析师审阅，然后发布。

聊天机器人一次只能帮一步。Claude Code 能跑完整条链。

你给它最终目标。需要的话，它会打开浏览器；需要的话，它会抓取数据。它写文件，它读文件，任务允许的话它会并行跑五件事。只有真正需要你的时候，它才回头要你的输入。这些工作，是在你忙别的事情时悄悄完成的。

这就是"手"的意思。ChatGPT 是一个你跟它说话的大脑；Claude Code 是一个替你跑工作流的大脑。

有一句话我总是反复想起：聪明、勤奋的人靠做得更多来完成更多，而智慧的人靠消除得更多来完成更多。Claude Code 就是那个"消除者"。我们嘴上说的"研究工作"，其实大部分是收集数据、排版、复制、粘贴、切换标签页。Claude Code 把这些从你的盘子里端走，只留下真正需要你的那一部分。

这就是关键所在。这篇文章里其余的一切，无非是怎么把它设置好。

如果你更想在屏幕上看着整个设置过程跑一遍，包括下一节里 Playwright 和 Firecrawl 的安装，我录了一段 16 分钟的演示，从头到尾覆盖了全部内容：

去 Anthropic 网站上的 Claude Code 页面。你会看到两条安装命令，一条给 Mac，一条给 Windows。复制适合你机器的那一条。

官方 Claude Code 设置文档：https://code.claude.com/docs/en/quickstart

![](images/img-01.jpg)

如果你用 Mac，打开 Terminal；如果你用 Windows，打开 PowerShell。粘贴命令，按回车，等一分钟。这就是安装。

现在输入 `claude` 再按回车，Claude Code 的提示符就打开了。你可以直接用大白话输入，想问什么都行。"嗨，我是新手，该怎么开始？"这样就够了，它会告诉你。

就这样。这就是安装。

![](images/img-02.jpg)

往下走之前，还有一件事要做：下载 Cursor。

官方 Cursor 页面：https://cursor.com/home

Cursor 不是终端的替代品，而是这套系统其余部分安家的地方。等你在下一节设置 Obsidian 知识库时，你会把 Obsidian 仓库当作一个文件夹在 Cursor 里打开。你的 raw、wiki、output 文件夹都列在侧边栏里，Claude Code 在旁边的一个面板里运行。文件被创建、编辑、索引的全过程，你都能在同一个窗口里实时看到。从这往后，你构建的一切都发生在这一个地方。

你想用普通终端也行。我有时候会用，尤其想看着 Claude Code 啃一些复杂的活儿时。但日常工作真正发生的地方是 Cursor，因为只有在这里，知识库、聊天和工具才能被同时看到。

Claude Code 单独用是有用的，但它并不是让这篇文章里这套系统运转起来的关键。安装只要两分钟，真正改变你研究的设置，是接下来要讲的。

### 可选：仅当你不想买 Claude 订阅时

继续之前快说一句。从这往后的内容，都默认你有一个 Claude 订阅，Pro 或 Max 都行。如果你没有，又不想为了试试看就专门买一个，有个能用的替代品叫 MiniMax M2，它能作为替换引擎接入 Claude Code，成本大约只有原来的百分之七。把它设置好，再回来从下一节接着读。后面的一切，用起来都完全一样。

## 第二步：把 Obsidian 设置成 Claude Code 的记忆

Claude Code 本身没有记忆。每次会话都从零开始。你周一跟它聊了一场精彩的对话，关掉终端，到周二它什么都不记得了。对大多数编程任务来说这没问题，但对研究来说，这就是个问题。研究是会越积越厚的。你三月份给一家半导体公司记的笔记，到十一月还应该能找得到，连同这期间你对这个行业学到的一切。

这正是 Obsidian 要解决的。Obsidian 是个免费应用，它把你电脑上的一个文件夹变成一座私人研究图书馆。每条笔记都是一个纯文本文件。你可以把笔记互相链接，按主题归进文件夹，并瞬间搜遍整座图书馆。没有数据库，没有云，不会被绑死。你的研究就放在自己机器上，存在任何文本编辑器都能打开的文件里。

诀窍在于：Claude Code 能读写那同一个文件夹。一旦设置妥当，Obsidian 就成了 Claude Code 的长期记忆。你把"图书管理员"这个角色交给它。它替你把收集的东西归档，把读过的内容总结出来，跨主题把相关的想法链接起来，每次你提问时都从图书馆里翻找。时间一长，图书馆越来越聪明，因为 Claude Code 给你的每个答案，又都被归档了回去。

![](images/img-03.jpg)

下面是怎么把它搭起来。

### 安装 Obsidian 并创建你的仓库

从 obsidian.md 下载 Obsidian。它个人使用免费，Mac、Windows、Linux 都能跑。

打开 Obsidian，点击"创建新仓库"。仓库说白了就是你电脑上存放 Obsidian 笔记的一个文件夹。名字随便取（我的就叫 Vault）。选一个你记得住的位置，放在 Documents 文件夹里某处就行。Obsidian 会打开这个空仓库。

![](images/img-04.jpg)

现在在 Cursor 里打开那同一个仓库文件夹。文件 → 打开文件夹 → 选你的仓库。你应该会看到文件夹的名字出现在 Cursor 的侧边栏里，现在还是空的。从这往后，一切都发生在这里。

![](images/img-05.jpg)

### 创建文件夹结构

在 Cursor 里打开 Claude Code（侧边栏面板或者终端，随你）。一字不差地粘贴这条提示词：

```
为我的知识库创建这个文件夹结构：

- raw/ — 这是我的收件箱，我会把源材料丢进这里
- wiki/ — 这是你的地盘，你会在这里写和维护一切
- 在 wiki/ 里创建一个 _master-index.md，标题为"Knowledge Base Index"，
    并附一条说明，写着"主题将在创建时列在这里。"
- output/ — 这是查询结果和报告存放的地方

只创建这些文件夹和那一个文件。其他暂时什么都不要做。
```

Claude Code 会创建三个文件夹和一个文件。打开 Obsidian，你会在侧边栏里看到它们。这就是你的骨架。

![](images/img-06.jpg)

三个文件夹，三项职责。raw/ 是你的收件箱，任何想让系统学习的东西都往里丢：文章、文字记录、截图、笔记。wiki/ 是图书管理员的地盘，Claude Code 在这里写结构化的总结，把它们归进主题文件夹，并维护一份涵盖一切的主索引。output/ 是成品落地的地方：研究报告、查询答案，以及你想跟图书馆本身分开存放的分析。

### 规则手册（CLAUDE.md）

这是整套系统里最重要的文件。CLAUDE.md 是规则手册，每次你在这个仓库里打开 Claude Code，它都会先读一遍。规则你只写一次，Claude Code 就永远照办。没有这个文件，Claude Code 只是个聪明的助手；有了它，Claude Code 就是个有纪律的图书管理员。

在仓库的根目录创建一个叫 CLAUDE.md 的文件（不是放在那三个文件夹里面，而是跟它们并排）。把下面这段粘贴进去：

```
这个文件告诉你如何在每次会话中维护这个知识库。

## 知识库规则

- 这是一个由 LLM 维护的知识库。你是图书管理员。
- wiki/ 文件夹是你的地盘 —— 你写和维护它里面的一切。
  我很少直接编辑 wiki 文件。
- raw/ 是收件箱。当我把文件丢进这里，你在一个"compile"步骤里
  把它们处理进 wiki。
- wiki/_master-index.md 是入口点。它列出每个主题文件夹，
  附一行描述。始终保持它最新。
- 每个主题在 wiki/ 里有自己的子文件夹（例如 wiki/ai-agents/），
  带自己的 _index.md，列出该主题下所有文章并附简短描述。
- 始终用 [[wiki links]] 来连接跨主题的相关概念。
- 编译原始材料时：
  1. 读取原始文件
  2. 决定它属于哪个主题（或创建一个新主题）
  3. 写一篇带关键要点和相关链接的 wiki 文章
  4. 更新该主题的 _index.md
  5. 更新 wiki/_master-index.md
  6. 如果一个原始文件跨多个主题，在两边都创建文章并交叉链接
- 文章保持简洁 —— 用要点而非段落。
- 每篇 wiki 文章里都包含一个 ## Key Takeaways 小节。
- output/ 用于查询结果和生成的报告。
- 回答问题时，先读 _master-index.md 来导航，然后
  钻进相关主题的 _index.md，再读具体文章。
- 当我说"compile"时，把 raw/ 里所有还没编译的东西
  处理进 wiki。
- 当我说"audit"或"lint"时，审查 wiki 里的不一致、
  断链、空缺，并提出改进建议。
```

保存这个文件。从现在起，每次会话都从 Claude Code 读这份规则手册开始。

### 每日输入：Web Clipper

Web Clipper 页面：obsidian.md/clipper

你需要一种不用复制粘贴、就能把文章弄进 raw/ 文件夹的办法。在你的浏览器上装一个 Obsidian Web Clipper 扩展（Chrome、Brave、Edge、Firefox 都支持）。

打开 Web Clipper 设置，添加你的仓库，然后去 Templates，编辑 Default 模板：

```
Vault: 选择你的仓库
Note location: raw
Note name: {{date|date:"YYYY-MM-DD"}}-{{title|safe_name}}
```

现在，你在网上读到的任何文章，点一下就变成收件箱里一个干净的 markdown 文件。一份财报电话会议记录、一篇 FT 的文章、一段 10-K 摘录、一篇 Substack 帖子，都用同一种方式剪藏下来。

还有一个可选的小插件：在 Obsidian 里装上 Local Images Plus（设置 → 社区插件 → 浏览 → 搜"Local Images Plus"）。等你打开一条剪藏的笔记时，这个插件会把文章里的每一张图片都下载进你的仓库，这样即使原始页面哪天挂了，你的图书馆也还是完整的。

### 每日工作流：四个动词

一切设置好之后，工作流就是四个动词。

**剪藏（Clip）。** 你看到值得留下的东西。在浏览器里点一下，它就落进 raw/。你不用决定它该去哪，也不用写总结，只管剪藏就好。

**编译（Compile）。** 每周一次，或者收件箱一满，就打开 Claude Code 输入：

```
把 raw/ 里的所有东西编译进 wiki。对每个文件：
1. 读它并识别核心主题
2. 在 wiki/ 里创建或找到合适的主题文件夹
3. 写一篇带总结、关键要点和 [[wiki links]] 的 wiki 文章，
   链接到任何相关概念
4. 更新该主题的 _index.md
5. 更新 wiki/_master-index.md

在相关的地方跨主题交叉链接。
```

Claude Code 会逐篇读你剪藏的文章，判断它属于哪个主题，写一条结构化的总结笔记，归进合适的文件夹，并更新索引。等你喝完一杯咖啡，收件箱空了，图书馆又大了一圈。

**查询（Query）。** 向图书馆提问。简单的："上个月我剪藏的那份贝恩半导体报告，关键发现是什么？"需要交叉引用的："中国汽车出口放缓，跟我一直在读的大宗商品需求有什么关系？"还有把答案归档回图书馆的综合型问题："基于 wiki 里的所有内容，印度特种化学品的主要多头和空头理由各是什么？把你的答案存成一篇新的 wiki 文章，并链接到你引用的来源。"

最后那个查询，正是让整套系统越滚越大的那一招。你问的每个问题都变成一条新笔记，每条笔记都链回它的来源。你每用一次，图书馆就更聪明一分。

**审计（Audit）。** 每月一次，输入："审计这个 wiki。找出不一致、断链、缺失的交叉引用，以及覆盖上的空缺。先别动手改，只给我一份报告。"Claude Code 会把整座图书馆走一遍，告诉你哪里开始变模糊了。改什么，你来定。

这就是整套系统。收件箱、图书管理员、图书馆、输出。四个文件夹，一份规则手册，四个动词。

这件事之所以比看上去更重要，是因为：系统本身就是纪律。装 Obsidian 和 Claude Code，谁都会。你的研究之所以能越积越厚、而不是一路漏掉，靠的就是规则手册逼出了一致性。每次剪藏都用同样的方式归档，每份总结都遵循同样的格式，每个查询都能调用你读过的一切。这套系统跑了六十天后，我可以问 Claude Code"这个季度我关于仓位管理学到了什么"，不到一分钟就能得到一个实打实的答案，里面引用的，是一些我都快忘了曾经剪藏过的文章。

接下来，我们再加上让 Claude Code 伸到图书馆之外、进入实时网络的那部分。

## 第三步：给 Claude Code 在网络上的手（Playwright + Firecrawl）

你刚搭好的知识库很强大，但它也是封闭的——它只知道你喂给它的东西。真正的研究工作里，你需要 Claude Code 能伸进实时网络：从筛选器里拉一张资产负债表，抓一个投资者关系页面，搜一搜你正盯着的公司最新的备案文件。接下来这两个工具，做的就是这件事。

Playwright 让 Claude Code 能驱动一个真实的浏览器，Firecrawl 让 Claude Code 能大规模地搜索和抓取网络。两个都免费、都开源，都大约两分钟装好。打个比方：一个图书管理员只会读你带回家的东西，另一个还能自己走到城那头的图书馆，把你要的东西带回来——这两个工具，就是这两者之间的差别。

### Playwright：浏览器

Playwright 是微软出的一个浏览器自动化工具。装好之后，Claude Code 就能打开浏览器，导航到任何页面，点按钮、填表单、登录账户、读取屏幕上的内容，整个过程不用你碰一下鼠标。

对金融工作来说，这在数据藏在交互背后的地方特别有用。一个要你设十个筛选条件才肯出结果的筛选器；一个把季度数字藏在下拉菜单里的投资者关系页面；一个要你先点掉免责声明才让下载文件的监管门户。任何因为页面认定"对面得是个人"而让普通抓取器失手的地方，Playwright 都能让 Claude Code 充当那个人。

要安装它，在你的仓库里打开 Claude Code 并粘贴：

```
我想安装微软的 Playwright CLI，这样你就能代表我驱动浏览器。
仓库在 https://github.com/microsoft/playwright-cli。
请为我的 [Mac / Windows] 机器一步一步带我走完安装，
对新手友好，并运行你需要运行的命令。
```

Claude Code 会装好它，带你走过中间任何提示，准备就绪时告诉你。问个简单的来测一下："打开一个浏览器，去 screener.in，告诉我主页上有什么。"如果 Playwright 装对了，一个浏览器窗口会打开，导航到那个页面，然后 Claude Code 会把它看到的东西反馈给你。

### Firecrawl：网络抓取器

Firecrawl 做的事不一样。Playwright 是一个亲手操作的浏览器，Firecrawl 则是一种快、且程序化的方式：从任何网页里拉出干净的文本，或者跑一次网络搜索、把排名靠前的结果里的文本拉回来。它会把找到的东西转换成 Claude Code 能直接读的干净 markdown 文件。

有个用例，金融人一看就懂：你想从 Finviz 拉最近两周的内幕交易数据；你想从一家公司的 IR 页面抓一份年报；你想搜"印度特种化学品的最新新闻"，把排名前十的文章拉进知识库。这三件事，Firecrawl 各用一条命令就搞定。

要安装：

```
我想安装 Firecrawl CLI，这样你就能代表我搜索和抓取网络。
官方文档在 https://docs.firecrawl.dev/sdks/cli。
请为我的 [Mac / Windows] 机器一步一步带我走完安装，
对新手友好，并给我每一条我需要运行的命令。
```

一旦装好，Firecrawl 就成了 Claude Code 能直接从终端调用的工具。你不用每次手动叫它。一个任务需要实时网络时，Claude Code 会认出自己手边有 Firecrawl，自动用上。

### 这在实践中长什么样

这是我上周跑的一条真实提示词。背景是：我想把资金分到五只标的上，可候选名单上只有两只，还差三只。

```
我在筛选要部署资金的标的。去 finviz.com，拉取美国上市公司
```

Claude Code 请求使用 Firecrawl 的权限，我说行。它去了 Finviz，把数据拉下来、解析好，跑了一遍买入信号的逻辑，又交叉核对了基本面，然后带回来一份排好序的十只标的清单。每只都有一个论点、一份内幕购买记录（谁买的、什么时候买的、买了多少）、关键风险，还有一张总结评分卡。从头到尾，三分钟。

### 关于权限的一件事

每次 Claude Code 想用一个工具（比如 Firecrawl 或 Playwright），或者想改一个文件、跑一条命令，它都会先请求权限。默认情况下，这意味着每隔几步就弹一个"是/否"。刚学的时候有用，一旦你信任了这套系统，就嫌烦了。

有个标志叫 `--dangerously-skip-permissions`，它把这些提示全关掉。这时候 Claude Code 就能运行任何工具、修改你仓库里的任何文件、执行命令，不用每次跟你确认。这名字听着吓人，它也确实该吓人——你这是把对整个工作目录的完全访问权交给了 Claude Code。

我开着这个标志跑了六十天。什么都没坏：没误删过文件，没跑过不该跑的命令，也没出过安全问题。变的只是，我不再坐在终端前等着点"是"，而是丢给 Claude Code 一个任务就走开。等我端着咖啡回来，任务做完了。

这能行得通，靠的是规则手册。CLAUDE.md 把 Claude Code 的活儿是什么、边界在哪都讲清楚了；再加上 Claude Code 只能碰你打开的那个仓库文件夹、碰不到你整台机器，风险其实比这个标志的名字暗示的要小得多。我会建议头一周先开着权限跑，好让你看清 Claude Code 在做什么；等你信任了这个套路，再把权限关掉，让它放手干。

这是个人选择。有些读者会一直开着权限，那也挺好。两种方式，系统都照样转。

## 六十天教会了我什么

上面这套设置大约花一个小时。而它造出来的系统，我跑了六十天，还在慢慢学着用。

有几件事让我意外。

第一件，是终端多快就不再像个编程工具了。到第一周结束，我往 Claude Code 里打字的方式，已经跟往 ChatGPT 里打字没两样了——大白话，没有命令，没有语法。大家警告的那种"害怕",第一天是真的，到第三天就没了。如果你是个搞金融的读者，正卡在犹豫里，那你犹豫的，是一个撑不过第一次会话的问题。

第二件，是当你不再做一个"聊天机器人用户"、而开始做一个"工作流操作者"时，会发生什么。这个转变很微妙，但下游的一切都跟着变了。你不再让 Claude Code"帮你"做某一个任务，而是把整个任务一股脑交给它：筛这些标的，过一遍这个框架，给活下来的拉财务数据，建模型，每个再起草一页纸。你给它最终目标，然后走开。工作在后台进行，你回来时面对的，是一个装满成品的文件夹。

第三件，是我以前的研究漏掉了多少。用这套系统之前，我的研究散在 Notion、Apple Notes、浏览器书签、五个不同文件夹里下载的 PDF，还有一个我很少打开的 Substack 草稿夹里。同一个洞见，我会重新发现两次，有时三次。Obsidian 知识库终结了这种情况。我现在读的一切都落到一个地方，被 Claude Code 归进对的主题，等六周后我问一个相关问题时再被翻出来。什么都不漏，信息越积越厚。

还有一个工具值得知道：NotebookLM。它是谷歌托管的研究工具，解决的问题跟 Obsidian 不一样。Obsidian 是你的图书馆；NotebookLM 是针对某个特定研究对象的聚焦工作台——一家公司、一个行业、一批备案文件。你把来源上传上去，NotebookLM 就只在那些文档里，给你有依据、带引用的答案。有一个 Python 封装，让你能从 Claude Code 内部直接查询一个 notebook，这意味着你可以搭出这样一个工作流：让 Claude Code 在同一条提示词里，既从你的 Obsidian 图书馆里取，又从一个聚焦的 NotebookLM 来源集里取。
