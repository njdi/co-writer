# 翻译问题分析：2026 年每个 AI 构建者都必须使用的 Agent 技术栈

## 问题列表

1. **"那些编码任务应该如何完成的技能"** — 句子结构不清晰，"那些编码任务应该如何完成的技能"读起来很绕，应改为"编码任务的执行规范"或"规定任务如何完成的技能文件"。

2. **"带有自我改写钩子的厚实技能"（thick skills with self-rewrite hooks）** — "厚实技能"是直译，含义模糊。原文 "thick" 与前文 Garry Tan 引用的 "thin harness" 对应，指内容丰富、充实的技能文件，可改为"内容充实、带有自我改写钩子的技能文件"。

3. **"一个每晚压缩 agent 当天所学的梦境循环"（a dream loop that compresses what the agent learned each day）** — "梦境循环"是对 dream loop 的创意翻译，但可能让读者困惑。保留英文或加注说明："一个每晚运行的'梦境循环'（dream loop），将 agent 当天的经验压缩整合"会更清楚。

4. **"这是没有人告诉 AI 构建者的真相"** — 开篇直译尚可，但"构建者"在此处可以更自然地表达为"这是没人告诉你的、关于 AI 开发的真相"，语感更强。

5. **"关键洞察，也是 Garry 的帖子让我清晰看到的，是分离"** — 句式生硬，可改为"最关键的洞察——也是 Garry 那条推文让我豁然开朗的地方——就是分离"。

6. **"适用于 Claude Code、Cursor、OpenClaw，或任何能读取 markdown 的 agent"** — "OpenClaw" 应查明是否为 "OpenHands" 或其他工具的笔误，但作为原文保留亦可。此行无大问题。

7. **"保持在 30 行以内。它是一张地图，不是百科全书。"** — 此为代码块内的翻译提示，应保持在英文代码块中不翻译（仅翻译代码块外的说明文字），检查是否有误将代码内注释翻译的情况——本文代码块内注释均已正确保留英文，无问题。

8. **"这是将失败变成学习而不仅仅是错误的钩子"（the hook that turns failures into learning, not just errors）** — 虽能读通，但可以更流畅："这个钩子让失败成为学习的契机，而不仅仅是一个错误记录"。
