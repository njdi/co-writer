# 写给非程序员的代码课 - 第 4 节：比较运算

> AI 时代，能看懂代码就够了。

## 谁应该学习这门课

你听说过Vibe Coding，已经在用、或者打算用 Claude Code、Codex、Cursor 这些工具做点东西，但可能会卡在：

- AI 写出来的代码，看不懂在做什么；
- 程序报错了，丢给 AI 改，改了一处又坏一处；
- 想加个小功能，描述半天 AI 总是做不对；
- 觉得代码有问题，但说不清问题在哪。

这门课就是为你准备的。

## 这门课教什么

### 1. 看懂代码的"形状"

看到一段代码，你能认出哪里在拿数据、哪里在做判断、哪里在循环、哪里在调用现成的功能。

### 2. 看懂代码的"流向"

数据从哪来，经过了什么处理，最后到哪去。这是调试的基础。

### 3. 看懂报错信息

报错不是"程序坏了"，而是程序在告诉你它哪里不对。教你怎么读懂这些信息。

### 4. 怎么准确描述问题

学会"现象是什么"、"我期望的是什么"、"我已经试过什么"。

### 5. 安全和常识

不要暴露自己的隐私信息，不要轻易执行删除命令，什么时候必须自己检查确认后再点回车。

## 大于、小于

代码里经常需要"判断"——年龄够不够 18、分数有没有及格、价格是不是超预算。这种判断就靠**比较运算**。

| 数学 | 代码 | 意思 |
| --- | --- | --- |
| ＞ | `>` | 大于 |
| ＜ | `<` | 小于 |
| ≥ | `>=` | 大于等于 |
| ≤ | `<=` | 小于等于 |

数学里的 `≥` 和 `≤` 在键盘上打不出来，所以代码里就用两个字符拼出来：`>=`、`<=`。注意顺序，是先大于小于、再等号，反过来写 `=>`、`=<` 程序就不认了。

Python 和 JavaScript 的写法是一样的：

```
18 >= 18
60 < 59
9.9 <= 10
```

每一次比较算出来的结果，就是第 2 节讲的**布尔**——要么 `True`，要么 `False`。上面三行的结果分别是 `True`、`False`、`True`。

## 等于、不等于

判断两个值是不是相等，用**两个等号** `==`：

```
3 == 3
3 == 5
```

第一行的结果是 `True`，第二行是 `False`。

**注意：是两个等号，不是一个。** 这是新手最容易看混的地方。回忆一下第 1 节讲的赋值：

```
age = 28
```

一个等号 `=` 是**赋值**——把右边的 `28` 装进左边的 `age` 里，是个动作。

两个等号 `==` 是**判断**——问"左边和右边是不是相等"，结果是 `True` 或 `False`。

一个在"做事"，一个在"问问题"，意思完全不一样。看代码时一定要数清楚等号的个数。

判断"不相等"用 `!=`，感叹号加等号：

```
3 != 5
3 != 3
```

第一行的结果是 `True`，第二行是 `False`。

## 字符串也能比较

`==` 和 `!=` 用在字符串上，就是判断两段文字是不是一样：

```
"张三" == "张三"
"张三" == "李四"
"abc" != "abc"
```

三行的结果分别是 `True`、`False`、`False`。

判断密码对不对、用户名是不是匹配、城市选的是不是"杭州"，都是这么比的。

`>`、`<` 也能用在字符串上，比的是**字母顺序**——前面的字母小，后面的字母大：

```
"apple" < "banana"
"cat" > "ant"
```

两行的结果都是 `True`。`a` 排在 `b` 前面，所以 `"apple"` 比 `"banana"` "小"；`c` 排在 `a` 后面，所以 `"cat"` 比 `"ant"` "大"。
