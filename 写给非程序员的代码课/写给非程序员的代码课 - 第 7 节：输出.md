# 写给非程序员的代码课 - 第 7 节：输出

> AI 时代，能看懂代码就够了。

## 谁应该学习这门课

你听说过Vibe Coding，已经在用、或者打算用 Claude Code、Codex、Cursor 这些工具做点东西，但可能会卡在：

- AI 写出来的代码，看不懂在做什么；
- 程序报错了，丢给 AI 改，改了一处又坏一处；
- 想加个小功能，描述半天 AI 总是做不对；
- 觉得代码有问题，但说不清问题在哪。

这门课就是为你准备的。

## 这门课教什么

### 1. 看懂代码的"形状"

看到一段代码，你能认出哪里在拿数据、哪里在做判断、哪里在循环、哪里在调用现成的功能。

### 2. 看懂代码的"流向"

数据从哪来，经过了什么处理，最后到哪去。这是调试的基础。

### 3. 看懂报错信息

报错不是"程序坏了"，而是程序在告诉你它哪里不对。教你怎么读懂这些信息。

### 4. 怎么准确描述问题

学会"现象是什么"、"我期望的是什么"、"我已经试过什么"。

### 5. 安全和常识

不要暴露自己的隐私信息，不要轻易执行删除命令，什么时候必须自己检查确认后再点回车。

## 输出是什么

代码算出来的结果，你其实是看不见的——程序自己心里算了，没告诉你。要让它把结果"说出来"，就要用**输出**。

输出还有一个特别实用的用处：**看代码到底跑成了什么样**。在代码各处加一行输出，看看每一步算出来的值对不对，是观察代码运行情况最直接的手段。

## 怎么写

Python 用 `print`，JavaScript 用 `console.log`。两边的写法是一个套路——名字后面跟一对**圆括号**，括号里写你想输出的东西。

Python：

```python
print("你好")    # 输出：你好
```

JavaScript：

```javascript
console.log("你好");    // 输出：你好
```

JavaScript 那个名字有点长——`console.log`，中间有个点。`console` 可以理解成"控制台"，`log` 是"记录一条"，合起来就是"在控制台记一条"。

## 能输出什么

数字、字符串、布尔都能直接输出，也可以输出**变量**和**表达式**。

Python：

```python
print("张三")      # 输出：张三
print(28)          # 输出：28
print(True)        # 输出：True

age = 28
print(age)         # 输出：28

print(3 + 2)       # 输出：5
print(age + 1)     # 输出：29
```

JavaScript：

```javascript
console.log("张三");      // 输出：张三
console.log(28);          // 输出：28
console.log(true);        // 输出：true

let age = 28;
console.log(age);         // 输出：28

console.log(3 + 2);       // 输出：5
console.log(age + 1);     // 输出：29
```

注意 `print(age)` 和 `print("age")` 完全不一样：前者输出 `age` 这个变量装着的值（`28`），后者输出 `"age"` 这三个字母本身。第 2 节讲过——有引号的是字符串，没引号的是变量名。括号里是哪种，决定了输出的是哪种。

括号里也可以塞**表达式**——`3 + 2`、`age + 1` 这种能算出值的写法。程序会先算出结果，再把结果输出。所以括号里也可以放比较、逻辑运算：

Python：

```python
print(age >= 18)                  # 输出：True
print(age >= 18 and age < 65)     # 输出：True
```

JavaScript：

```javascript
console.log(age >= 18);                  // 输出：true
console.log(age >= 18 && age < 65);      // 输出：true
```

## 一次输出多个东西

想一次输出多个值，用**逗号**隔开就行。两边都自动在中间加一个空格。

Python：

```python
name = "张三"
age = 28
print("姓名", name, "年龄", age)    # 输出：姓名 张三 年龄 28
```

JavaScript：

```javascript
let name = "张三";
let age = 28;
console.log("姓名", name, "年龄", age);    // 输出：姓名 张三 年龄 28
```

这比用 `+` 拼字符串方便——拼字符串的话，数字还得先转成字符串，麻烦；用逗号一隔，什么类型都能直接丢进去。

## 输出到哪里去了

输出的东西到底跑到哪里去了？这个取决于代码在哪儿运行：

- 在**终端**（命令行）里跑 Python，输出就出现在那个黑底白字的窗口里。
- 在**浏览器**里跑 JavaScript，输出出现在浏览器的"开发者工具" → "控制台"那一栏里（按 F12 能打开）。
- 在代码编辑器里跑代码，输出会出现在编辑器下方的"终端"或"输出"面板里。

不管在哪儿，输出的内容都是一样的，只是显示的地方不同。
