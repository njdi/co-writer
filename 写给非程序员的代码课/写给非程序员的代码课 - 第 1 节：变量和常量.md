# 写给非程序员的代码课 - 第 1 节：变量和常量

> AI 时代，能看懂代码就够了。

## 谁应该学习这门课

你听说过Vibe Coding，已经在用、或者打算用 Claude Code、Codex、Cursor 这些工具做点东西，但可能会卡在：

- AI 写出来的代码，看不懂在做什么；
- 程序报错了，丢给 AI 改，改了一处又坏一处；
- 想加个小功能，描述半天 AI 总是做不对；
- 觉得代码有问题，但说不清问题在哪。

这门课就是为你准备的。

## 这门课教什么

### 1. 看懂代码的"形状"

看到一段代码，你能认出哪里在拿数据、哪里在做判断、哪里在循环、哪里在调用现成的功能。

### 2. 看懂代码的"流向"

数据从哪来，经过了什么处理，最后到哪去。这是调试的基础。

### 3. 看懂报错信息

报错不是"程序坏了"，而是程序在告诉你它哪里不对。教你怎么读懂这些信息。

### 4. 怎么准确描述问题

学会"现象是什么"、"我期望的是什么"、"我已经试过什么"。

### 5. 安全和常识

不要暴露自己的隐私信息，不要轻易执行删除命令，什么时候必须自己检查确认后再点回车。

## 变量

你的年龄每年加一岁，体重天天不一样，搬家城市就变了，甚至名字也能改。

这些"会变的信息"，在代码里就叫**变量**。

如果让你在纸上记录这些信息，大概是这样：

```
姓名：张三
年龄：28
城市：杭州
```

写成代码——

Python：

```python
name = "张三"
age = 28
city = "杭州"
```

JavaScript：

```javascript
let name = "张三";
let age = 28;
let city = "杭州";
```

两种语言长得有点不一样：JavaScript 前面多了个 `let`，每行结尾多了个分号。但核心结构一样——等号左边是变量的名字，右边是它装着的值。

> JavaScript 第一次给变量起名时要带个 `let`，意思是"新建一个变量"；之后再改它的值，就不用写 `let` 了。Python 没这个讲究，第一次出现直接写就行。

一年后过了生日、搬了家：

Python：

```python
age = 29
city = "上海"
```

JavaScript：

```javascript
age = 29;
city = "上海";
```

`age`、`city` 这些名字没变，变的是它们装着的东西。这就是"变量"的"变"。

## 常量

你的年龄会变，但**出生日期**不会变；你的体重天天变，但**身份证号**一辈子就那一个。

这些"一旦定下来就不会再变的信息"，在代码里就叫**常量**。

Python：

```python
BIRTH_DATE = "1997-03-15"
ID_NUMBER = "330106199703150011"
```

JavaScript：

```javascript
const BIRTH_DATE = "1997-03-15";
const ID_NUMBER = "330106199703150011";
```

两边的写法和变量几乎一样，但有两个小区别：

1. **名字全用大写字母**：这是程序员之间的约定——看到一个全大写的名字，就知道"这是个不该改的东西"。
2. **JavaScript 用 `const` 代替 `let`**：意思是"新建一个常量"。如果后面有人手贱写了 `BIRTH_DATE = "2000-01-01"` 想改它，程序会直接报错，根本不让改。

> Python 比较佛系——它没有"常量"这个语法，全大写只是个君子约定，硬要改也能改。JavaScript 的 `const` 才是真·拦着你不让改。

## 等号

你可能已经注意到，前面写代码用了若干个 `=`：

```python
age = 28
```

**但代码里的等号，和数学里的等号意思完全不一样。**

数学里 `age = 28` 是"age 和 28 相等"，是个判断。

代码里 `age = 28` 是个**动作**——"把右边的 28 装进左边的 age 里"。方向是从右往左。

代码里这个特殊的等号，有个正式的名字叫**赋值**——"赋"就是给予，把右边的值"给"到左边的变量里。
