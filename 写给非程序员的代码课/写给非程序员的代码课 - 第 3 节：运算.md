# 写给非程序员的代码课 - 第 3 节：运算

> AI 时代，能看懂代码就够了。

## 谁应该学习这门课

你听说过Vibe Coding，已经在用、或者打算用 Claude Code、Codex、Cursor 这些工具做点东西，但可能会卡在：

- AI 写出来的代码，看不懂在做什么；
- 程序报错了，丢给 AI 改，改了一处又坏一处；
- 想加个小功能，描述半天 AI 总是做不对；
- 觉得代码有问题，但说不清问题在哪。

这门课就是为你准备的。

## 这门课教什么

### 1. 看懂代码的"形状"

看到一段代码，你能认出哪里在拿数据、哪里在做判断、哪里在循环、哪里在调用现成的功能。

### 2. 看懂代码的"流向"

数据从哪来，经过了什么处理，最后到哪去。这是调试的基础。

### 3. 看懂报错信息

报错不是"程序坏了"，而是程序在告诉你它哪里不对。教你怎么读懂这些信息。

### 4. 怎么准确描述问题

学会"现象是什么"、"我期望的是什么"、"我已经试过什么"。

### 5. 安全和常识

不要暴露自己的隐私信息，不要轻易执行删除命令，什么时候必须自己检查确认后再点回车。

## 加减乘除

代码里的加减乘除，符号和数学课上差不多，只是乘号和除号换成了键盘上能直接打出来的字符：

| 数学 | 代码 | 例子 |
| --- | --- | --- |
| ＋ | `+` | `3 + 2` |
| － | `-` | `3 - 2` |
| × | `*` | `3 * 2` |
| ÷ | `/` | `6 / 2` |

Python 和 JavaScript 的写法是一样的：

```
3 + 2
100 - 68
9.9 * 3
6 / 2
```

## 运算顺序：先乘除，后加减

跟数学课上完全一样：**先算乘除，再算加减**。想改变顺序，用**圆括号**。

```
1 + 2 * 3
(1 + 2) * 3
```

第一行的结果是 `7`，不是 `9`；第二行加了括号，结果才是 `9`。括号可以套括号，套几层都行。

## 取余

除法除不尽的时候，会剩下一个**余数**。小学算过：`7 ÷ 2 = 3 余 1`。

代码里专门有一个符号来取这个余数——百分号 `%`：

```
7 % 2
10 % 3
8 % 4
```

三行的结果分别是 `1`、`1`、`0`。最后一个是 `0`，意思是 `8` 能被 `4` 整除，一点不剩。

`%` 在代码里出现得比你想的多：判断一个数是奇数还是偶数（看 `% 2` 是不是 `0`）、每隔几次做一件事、时间换算（秒数转成分钟和秒），都会用到。

## 字符串也能"加"

`+` 号用在数字上是加法，用在**字符串**上是另一个意思——**拼起来**。

```
"张" + "三"
"你好，" + "张三"
```

第一行的结果是 `"张三"`，第二行的结果是 `"你好，张三"`。字符串之间的 `+`，就是把两段文字接到一起，不做任何"加法"运算。
