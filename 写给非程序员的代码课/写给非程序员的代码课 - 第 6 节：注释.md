# 写给非程序员的代码课 - 第 6 节：注释

> AI 时代，能看懂代码就够了。

## 谁应该学习这门课

你听说过Vibe Coding，已经在用、或者打算用 Claude Code、Codex、Cursor 这些工具做点东西，但可能会卡在：

- AI 写出来的代码，看不懂在做什么；
- 程序报错了，丢给 AI 改，改了一处又坏一处；
- 想加个小功能，描述半天 AI 总是做不对；
- 觉得代码有问题，但说不清问题在哪。

这门课就是为你准备的。

## 这门课教什么

### 1. 看懂代码的"形状"

看到一段代码，你能认出哪里在拿数据、哪里在做判断、哪里在循环、哪里在调用现成的功能。

### 2. 看懂代码的"流向"

数据从哪来，经过了什么处理，最后到哪去。这是调试的基础。

### 3. 看懂报错信息

报错不是"程序坏了"，而是程序在告诉你它哪里不对。教你怎么读懂这些信息。

### 4. 怎么准确描述问题

学会"现象是什么"、"我期望的是什么"、"我已经试过什么"。

### 5. 安全和常识

不要暴露自己的隐私信息，不要轻易执行删除命令，什么时候必须自己检查确认后再点回车。

## 注释是什么

代码里除了让程序执行的指令，还经常夹着一些**给人看的话**——解释这行在做什么、提醒后面要改的地方、或者干脆把某一行临时"关掉"。这些给人看、程序不执行的内容，就叫**注释**。

## 怎么写

Python 用 `#`，JavaScript 用 `//`。这个符号后面到这一行结束为止，**整行都是注释**，程序看到就直接跳过。

Python：

```python
# 这是一行注释，程序不会执行
age = 28
```

JavaScript：

```javascript
// 这是一行注释，程序不会执行
let age = 28;
```

注释也可以写在代码后面，跟在同一行：

Python：

```python
age = 28        # 张三的年龄
age = age + 1   # 过完生日加一岁
```

JavaScript：

```javascript
let age = 28;       // 张三的年龄
age = age + 1;      // 过完生日加一岁
```

`#` 或 `//` 后面写什么都行——中文、英文、数字、符号，程序一概不看。

## 注释能干什么

**1. 解释代码在做什么**

代码本身只告诉你"在算什么"，不告诉你"为什么这么算"。注释就是补充"为什么"的地方：

Python：

```python
shipping = 10           # 默认运费 10 元
shipping = 0            # 满 50 包邮，运费清零
```

**2. 临时关掉一行代码**

写代码时经常要试："这一行是不是它在捣乱？"——这时候不用把它删掉，前面加个 `#` 或 `//`，这行就被"关"了，程序跳过，验证完再把符号去掉就恢复了。这个动作叫**注释掉**。

Python：

```python
age = 28
age = age + 1
# age = age + 10
```

JavaScript：

```javascript
let age = 28;
age = age + 1;
// age = age + 10;
```

本来三行跑完 `age` 应该是 39。把第三行注释掉之后，第三行不算了，`age` 最终装的是 29。把 `#`（或 `//`）拿掉，恢复 39。

## 多行注释

如果想一次注释好几行，最简单的办法就是每行前面都加一个 `#` 或 `//`。

Python 和 JavaScript 还各自有一种**多行注释**的写法：

Python 用一对三个引号 `"""..."""`：

```python
"""
这是一段多行注释
可以写好几行
程序都会跳过
"""
```

JavaScript 用 `/* ... */`：

```javascript
/*
这是一段多行注释
可以写好几行
程序都会跳过
*/
```
