# 写给非程序员的代码课 - 第 5 节：逻辑运算

> AI 时代，能看懂代码就够了。

## 谁应该学习这门课

你听说过Vibe Coding，已经在用、或者打算用 Claude Code、Codex、Cursor 这些工具做点东西，但可能会卡在：

- AI 写出来的代码，看不懂在做什么；
- 程序报错了，丢给 AI 改，改了一处又坏一处；
- 想加个小功能，描述半天 AI 总是做不对；
- 觉得代码有问题，但说不清问题在哪。

这门课就是为你准备的。

## 这门课教什么

### 1. 看懂代码的"形状"

看到一段代码，你能认出哪里在拿数据、哪里在做判断、哪里在循环、哪里在调用现成的功能。

### 2. 看懂代码的"流向"

数据从哪来，经过了什么处理，最后到哪去。这是调试的基础。

### 3. 看懂报错信息

报错不是"程序坏了"，而是程序在告诉你它哪里不对。教你怎么读懂这些信息。

### 4. 怎么准确描述问题

学会"现象是什么"、"我期望的是什么"、"我已经试过什么"。

### 5. 安全和常识

不要暴露自己的隐私信息，不要轻易执行删除命令，什么时候必须自己检查确认后再点回车。

## 与、或、非

第 4 节讲的比较，每次只能判断一件事。但现实里的判断经常要**多个条件凑在一起**：

- 年龄满 18 **而且** 持有有效证件，才能办这个业务。
- 是会员 **或者** 消费满 100，就能享受折扣。
- **不是**周末，就要早起上班。

代码里把这种"而且"、"或者"、"不是"叫**逻辑运算**，对应三个运算符：**与**、**或**、**非**。

这三个运算符在 Python 和 JavaScript 里写法不一样：

| 意思 | Python | JavaScript |
| --- | --- | --- |
| 与（而且） | `and` | `&&` |
| 或（或者） | `or` | `\|\|` |
| 非（不是） | `not` | `!` |

Python 直接用英文单词，JavaScript 用符号。意思完全一样，只是长相不同。

## 与：两个都成立才算成立

**与**的规则：左右两边**都**是 `True`，结果才是 `True`；只要有一个是 `False`，结果就是 `False`。

Python：

```python
True and True
True and False
18 >= 18 and 60 >= 60
```

JavaScript：

```javascript
true && true
true && false
18 >= 18 && 60 >= 60
```

三行的结果分别是 `True`、`False`、`True`。最后一行是两个比较拼在一起——"18 大于等于 18"成立，"60 大于等于 60"也成立，所以整体成立。

## 或：有一个成立就算成立

**或**的规则：左右两边**任意一个**是 `True`，结果就是 `True`；两边都是 `False`，结果才是 `False`。

Python：

```python
True or False
False or False
18 >= 20 or 60 >= 60
```

JavaScript：

```javascript
true || false
false || false
18 >= 20 || 60 >= 60
```

三行的结果分别是 `True`、`False`、`True`。最后一行的左边"18 大于等于 20"不成立，但右边"60 大于等于 60"成立，所以整体成立。

## 非：把真假反过来

**非**的规则最简单：`True` 变 `False`，`False` 变 `True`。

Python：

```python
not True
not False
not (3 == 5)
```

JavaScript：

```javascript
!true
!false
!(3 == 5)
```

三行的结果分别是 `False`、`True`、`True`。最后一行里，`3 == 5` 本身是 `False`，前面加个 `not`（或 `!`），就翻成了 `True`。

## 运算顺序

逻辑运算也有先算谁、后算谁的规则：**非**最优先，然后是**与**，最后是**或**。两种语言的规则一样。

Python：

```python
True or False and False
```

JavaScript：

```javascript
true || false && false
```

这一行的结果是 `True`，不是 `False`。因为**与**先算——`False and False` 是 `False`，然后再算 `True or False`，结果是 `True`。

规则记不住没关系，**拿不准就加括号**，跟数学里 `(1 + 2) * 3` 一个道理。括号里的先算，看代码的人也省心：

Python：

```python
(True or False) and False
```

JavaScript：

```javascript
(true || false) && false
```

加了括号，先算括号里的 `True or False` 是 `True`，再算 `True and False`，结果就是 `False` 了。
